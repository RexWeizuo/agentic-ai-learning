#### 1. Harness Engineering 一期讲透（2026-04-02）
**UP主：code秘密花园**

三层演进路线：
- **Prompt Engineering**：把任务讲清楚 → 塑造局部概率空间
- **Context Engineering**：把信息给对 → 在正确时机把正确信息送进去。IG、Skills 的渐进式披露都是典型实践
- **Harness Engineering**：让模型在真实执行中持续做对 → agent = model + harness

Harness 六层架构：
1. **上下文层**：角色定义、信息裁剪、结构化组织
2. **工具系统**：给什么工具、何时调用、结果如何喂回
3. **执行编排**：理解目标→判断信息→补充→分析→生成→检查→修正
4. **记忆和状态**：当前任务状态 / 会话中间结果 / 长期记忆——三者必须分开管理
5. **评估和观测**：独立评估、自动测试、日志、错误归因
6. **约束校验与恢复**：约束（能/不能做什么）、校验（输出前后检查）、恢复（失败后回滚）

一线实践：
- **Anthropic**：context reset（换干净 agent 交接任务，而不是继续压缩）、planner/generator/evaluator 分离（生产验收分离）
- **OpenAI**：人不写代码，只设计环境。渐进式披露（agent.md 变成目录页）。让 agent 接浏览器、日志系统自己验证。把资深工程师经验写成系统规则
- **核心洞察**：同样模型，不同产品表现差距巨大，因为决定上限的是模型，决定能否稳定交付的是 harness

#### 2. PI Coding Agent 的 Hook 扩展体系（2026-04-16）
**UP主：雷哥AI**

PI (Pi Coding Agent) 核心理念：
- **极简设计**：只有 4 个工具（读/写/编辑/运行命令）。不支持 MCP、sub-agent、todo 等
- **极度可扩展**：生命周期每个节点都是扩展点——session start、before agent start、before tool call、tool call、after tool call、turn end、agent end、压缩、fork session、切换 model……
- 扩展插件可迁移、可组合：`pi -E plugin_a -E plugin_b`
- 支持 Python/TS 的 RPC 对接，本地调好可直接迁移到生产

**为什么重要**：被 OpenClaw、Hermes、Claude Code 等广泛采用。极简 + 透明 + 可扩展的设计哲学，比黑盒框架更适合企业落地。

#### 3. 9 个月构建 150+ Agent 的 26 条教训（2025-03-17）
**UP主：KrillinAI小林**

核心要点提炼：
- **Agent ≠ 员工，Agent = SOP**：一个 agent 处理一个标准作业流程，员工处理 5+ 个 SOP。从良好文档化的 SOP 开始
- **企业主不会自己构建 agent**：即使有了"一句话建 agent"的工具，企业主永远委托给懂的人
- **50% 的客户不知道自己需要什么 agent**：咨询比开发更重要，从客户旅程地图出发
- **每 agent 不超过 4–6 个工具**：超过就开始幻觉、混淆工具用法。多了就拆分
- **模型成本不重要**：相比手动执行，agent 成本可忽略。一个问卷流程从 $300/3天 → $1–2/20分钟
- **客户不关心你用哪个模型**：只要用例有价值，数据隐私用 Azure OpenAI 解决
- **Pydantic 验证是银弹**：验证所有 agent 输入输出，agent 就无法搞砸
- **先确立价值再自动化**：不要自动化一个还不存在的流程
- **用 ROI 公式评估**：(时薪 × 总工时 - 运营成本) / 开发成本
- **部署比构建难**：构建 2-3 天，部署再加 3 天。整合到客户流程才是难的
- **不要围绕模型限制构建**：模型很快会消除这些限制，让你的系统过时
- **瀑布项目不行**：agent 项目太敏捷，3 个月规划不现实
- **关键任务 agent 必须有人类在回路中**：误差空间低的场景，先让人审批，微调后再去掉
- **2025 是垂直 agent 年**：先做几个水平 agent → 找到共性 → 产品化为垂直 agent
- **agent 不会取代人**：它们帮企业扩大规模，员工专注于更高层次的事

#### 4. Claude Code 源码泄露：一次完整对话流程（2026-04-03）
**UP主：飞天闪客**

逆向分析一次问答的完整链路：
1. 用户输入 → handleEnter
2. **第一次大模型请求**（异步）：给对话起短标题 → 显示在选项卡
3. **第二次大模型请求**（主流程）：携带完整的 system prompt
   - 身份声明（You are Claude Code, a CLI tool for Claude）
   - 任务规则、工具使用方法、代码风格、安全规则
   - 可用 skill 列表（用户自定义 + 系统内置）
   - 补充命令（git 状态、最近提交等）
   - 所有可用工具定义（function calling）
   - 实验特性开关
   - max_tokens 上限
4. 大模型响应 → thinking + 文本回复 / tool_use
5. 如果 tool_use → agent 执行工具 → 将结果再次请求大模型 → 循环
6. 直到不再需要工具 → 返回最终答案

**洞察**：每次请求都携带完整 system prompt + skill 列表 + 全部历史，token 消耗极大。这也是为什么需要 context reset 和 compaction。

#### 5. Hermes Agent 详解 & 部署（2026-04-18）
**UP主：堂吉诃德拉曼查的英豪**

Hermes vs OpenClaw 核心差异：
- **Skill 自主沉淀**：对话中自动判断"这个流程是否复杂到值得沉淀为 skill"，自动创建、自动更新，**无需人工确认**。越用越懂你
- **更强的安全性**：skill 自生成 → 来源可控；子 agent 上下文永不返回父 agent → 防提示词注入
- **token 更省**：子 agent 看不到父 agent 对话历史，只专心执行任务；父 agent 只看子 agent 结果，不关心过程
- **更好的记忆架构**：短期上下文 + 向量检索 + 全文检索，为长期陪伴型数字员工设计
- **一行命令安装**，原生支持 prompt caching

## 四、📚 学习方法与思维（10 篇）

### ⭐⭐⭐⭐⭐ 必读

#### 波利亚的四张底牌：1945 年的数学神作，AI 时代程序员的生存法则（2026-03-30）
**UP主：为什么叫QQ**

波利亚《怎样解题》（1945）四步法 → AI 时代的四张底牌：

1. **理解问题 → 锚定边界**
   - AI 最大弱点：在没有定义清楚的地方疯狂产生幻觉
   - 模糊的问题 × AI 的速度 = 光速走错路
   - 顶级程序员在唤醒 AI 前，先把业务语言翻译成确定性的工程约束（QPS、降级策略、超时时间、核心链路 vs 边缘逻辑）

2. **制定计划 → 设计回路**
   - 真正高效的模式不是 vibe coding，是 feedback loop
   - 顶级程序员变成了工作流设计师：拆解成微小上下文 → 人机交替开车
   - 人类在关键十字路口把控方向盘

3. **执行计划 → 灰度验证**
   - 当代码生成成本趋近于零，真正昂贵的是**判断成本**
   - AI 生成的代码：看起来全对，实际上有暗坑
   - 你是验收大楼的监理，砖不是你砌的，但塌了责任全是你的

4. **回顾检查 → 系统折叠**
   - 拉开人与人差距最核心的一步
   - 每次成功解决复杂问题后，把经验折叠成可复用的系统资产（prompt 模板、自动化脚本、架构思路沉淀到知识库）
   - 下次直接调用折叠好的系统模块

**核心结论**：未来最值钱的不是写代码最快的人，而是最能驾驭复杂性的人。不要去当 AI 监工，去当系统建造者。

## 五、💪 健身与身体（2 篇）

### 久坐腰突必练的三个动作（2024-12-14）
**UP主：宋教头_**

1. **伟大拉伸 + 单腿跪姿脊柱伸展**
   - 双脚与肩同宽，向前迈一大步，后腿蹬直
   - 头部带动脊柱向前伸展，头腰臀同线
   - 吸气曲肘沉肩向下，呼气手臂向外打开
   - 重复 8-10 次后，后腿改跪姿，保持 15-20 秒，左右交替

2. **经典侧支撑**
   - 锻炼身体体侧线（筋膜链），维持骨盆和脊柱平衡
   - 防止骨盆侧倾和脊柱侧弯

3. **俯卧对侧交替上抬**
   - 锻炼后侧肌群，纠正含胸驼背
   - 上抬幅度不宜过大，上方保持 3-5 秒
   - 三组，每组 8-12 次

> 核心原则：不能头痛医头，要从整体训练身体。

## 六、💰 理财与赚钱（5 篇）

### 日本第一理财书：普通人实现经济自由的五步（2026-05-31）
**UP主：乌苏苏_DoraSusu**

金钱五环力：**续 → 增 → 赚 → 用 → 守**

**第一环：续（攒钱）**——最重要，占据全书一半篇幅
- 核心：重组六大固定支出（一次搞定，每月自动省）
  1. **自动续费**：做一览表，彻底清理。原则：开一个新的必须解约一个老的。投资思维判价值：30元/月会员 = 年360元，需要9000本金（4%收益）才能赚到
  2. **通信费**：很多人用着多年前的老套餐，花十几分钟换低价套餐
  3. **保险**：只保"概率极低但一旦发生倾家荡产"的事（火灾、车祸）。概率高损失小的（碎屏险）不买
  4. **住房**：租房≠给房东打工，买房有折旧/利率/断供风险，变现困难降低人生灵活性
  5. **车子**：尽量不买，打车比养车便宜。真要买→二手车最划算
  6. **税**：打工人在发工资前就被扣税→开启副业变成主动经营者，合法节税

**第二环：增（投资）**
- 先存生活预备金（上班族 6 个月，自营业者 1 年）
- 记住基础常识：稳健投资平均年收益 5-7%，巴菲特平均约 20%
- 超过 20% 的理财产品 99% 不能碰
- 普通人最适合：指数基金长期定投（如标普 500），拿住 15 年以上

**第三环：赚（增加收入）**
- 最有效涨薪方式：跳槽
- 副业≠下班打工，是建立自己的事业收入 → 享受合法节税
- 不知道做什么→做自媒体（发对别人有用的内容）
- 书中列出 15 个在家可做的低风险副业

**第四环：用（幸福消费）**
- 花钱的核心是交换真正的幸福，不是虚荣
- 好的消费：演唱会、旅行、投资自己、花钱买时间、利他

**第五环：守（保护资产）**
- 防诈骗 + 防自己挥霍

### ⭐⭐⭐⭐ Anthropic 为什么成了美国安全威胁（2026-05-05）
