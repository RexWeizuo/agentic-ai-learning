## Agent Session 全日交互

### 1. Ch.01 遗留问题深挖

**学生提问**：
- Anthropic 用 `tool_use` + `input`，OpenAI 用 `tool_calls` + `arguments`——`input`/`arguments` 是用户最开始的输入吗？
- Adapter 是 API call 上面一层的统一封装吗？
- 真实系统笔记展开讲（OpenCode Tool.define、Hermes ToolEntry、OpenClaw channel adapter 等）

**AI 回答核心点**：
- `input`/`arguments` 是**工具的参数**，不是用户输入。用户最初的话在 message 数组里。
- Adapter 就是把 `tool_use.input` 和 `tool_calls.arguments` 统一成一个 `ToolCall` 对象，换 provider 只改一行。
- OpenCode `Tool.define` 把每次工具调用追踪为类型化生命周期对象（created/dispatched/completed/failed/truncated）
- Hermes `ToolEntry` 绑定 schema + handler + 大小限制 + 错误分类器（可恢复 vs 致命）
- OpenClaw/Paperclip：任何东西都可以是工具——channel adapter、工作流步骤、shell 命令、调用另一个 agent——只要遵守 name+schema+result 合同

### 2. Ch.02 五题回答与评估

**第 1 题**：五阶段在量化场景的映射
- 学生答：Plan 阶段调 get_price 获取茅台市值，Act 阶段执行
- AI 评估 ⚠️：方向对但漏了"模型不是一次调用完成，而是每步决定还需要什么信息"这个关键点

**第 2 题**：model-driven stop vs final_answer 的区别
- 学生答：都是模型主动调用停止
- AI 评估 ✅ 正确，补充：final_answer 保证干净的结构化输出，model-driven stop 可能"以为答完了但没答完"

**第 3 题**：doom loop 为什么只看调用不看结果
- 学生答：相同调用说明重复同一件事；参数在变但查不到是另一种 doom loop
- AI 评估 ✅ 正确，补充了进度追踪和启发式检测两种应对策略

**第 4 题**：错误处理——三次失败同一个工具
- 学生答：换一种请求模式（兼容模型？）或要求用户介入
- AI 评估 ⚠️："compatible model"是换**物理上不同的模型**，不是换请求模式。Hermes 用凭证池+回退模型链处理

**第 5 题**：step boundary 五种能力，实盘最关键
- 学生答：权限检查 + 人类审批（因为真花钱高风险）
- AI 评估 ✅ 正确，补充：这两个必须在 Act 之前挂，因为 Act 后不可逆

**abort token 展开**：学生问"工具调用中途收到中断"那段话的意思，AI 用交易中途 Ctrl-C 的场景举例——优雅退出 vs 杀进程

**真实系统笔记展开**：OpenCode SessionProcessor（事件流+worker pool）、Hermes run_conversation（90 次上限+凭证池+回退链）、OpenClaw 优雅停止（预算差一次给 grace call）、Paperclip 外层循环（心跳+重试+持久化）
