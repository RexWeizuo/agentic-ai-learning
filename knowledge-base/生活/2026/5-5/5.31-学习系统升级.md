# 学习系统升级：实时追踪 + 健康作息

## 背景
上周回顾（5.18-5.24）暴露了核心问题：完成度低、基建太多、周四必滑坡、两三天就疲劳。
根本原因不是不够努力，而是：
- 全天排满 9:30-22:30 无微型休息
- 上午下午都是高强度，大脑没换频道
- 计划只有"学什么"，没有"学到什么程度算完成"
- 系统是事后记录，缺少实时锚点

## 新设计

### 每日节奏：50-10-汇报
09:30 启动 + Codex 定任务 → 50min 深度块 1 → 10min 休息 + 汇报 → 块 2 → ...
一天 6 个深度块 + 1 个弹性缓冲 = 约 5-5.5h 有效学习

### 周节奏
| 周一 | 上午休息，下午启动，无新课（启动日） |
| 周二 | 满负荷，最硬内容 |
| 周三 | 正常 |
| 周四 | **主动降载 4 块**（预设轻量日，承认历史滑坡规律） |
| 周五 | 正常 |
| 周六 | 可加量 |
| 周日 | 5 块 + 周回顾，16:00 收工 |

### 汇报协议（每小时一次）
- "XX done，50min，有个点模糊" → 完成汇报
- "休息 15min，14:20 继续" → 休息汇报（让我知道你没消失）
- "比预想难，只做了 5/10，多用 30min" → 调整汇报

### 休息原则
禁止手机（伪休息）。真休息：站起来走、望远处、喝水吃水果、闭眼停机、写一句困惑、和 Codex 汇报。

### 新增文件
- `memory/daily-sessions/` — 每日计划 + 实际时间线
- `memory/pulse-check.md` — 连续天数、本周完成率、疲劳预警（🟡2天/🔴3天低分触发）

## Codex 角色
从"事后记录员"变成"实时学习搭档"——每次会话打开 = 学习块开始，每次汇报 = 记录实际进展，每次收尾 = 总结状态 + 预告明天。
