## main.py 走读

**AI 提问**：
- 第 1 行 `from __future__ import annotations` 是干什么的？
- 第 21 行 `os.environ.get("DASHSCOPE_API_KEY")` 为什么不用硬编码？
- 第 25 行 `sys.exit(1)` 括号里的 1 是什么意思？
- 第 38 行 `run_loop` 从哪个文件导入的？
- 第 41 行 `result.get('partial')` 如果写成 `result['partial']` 有什么不同？
- 第 47-48 行 `if __name__ == "__main__"` 不写会怎样？

**学生**：不知道；这样 AI 就读不到 key 但它不是还是读得到文件内容嘛；不知道；loop 文件导入的；不知道；不知道。

**AI 逐行解答**：
1. `from __future__` 暂时不用管，删了也能跑。
2. `os.environ.get` 不是防 AI，是防 git push。API key 写在代码里 → commit → push → 全世界看到。放环境变量 → 代码零敏感信息。
3. `sys.exit(1)` 是退出码。0=正常，非0=出错。被别的脚本调用时可以判断"崩了没"。
4. `run_loop` 从第 16 行 `from src.quant_agent.loop import run_loop` 导入 ✅ 学生答对。
5. `result.get('partial')` → 不存在返回 None。`result['partial']` → KeyError 崩。partial 只在 step cap 耗尽时出现，正常结束没有这个 key。
6. `if __name__ == "__main__"` 是挡箭牌：被 import 时不执行 main()。没有这行 → import 什么都触发 agent 跑起来。

**学生追问**：partial 这词用来干嘛的？区别什么嘛。

**AI**：标签。告诉调用者答案是半成品还是完整。loop 里只有 step cap 耗尽那条 return 有 `"partial": True`。main.py 用它显示 `⚠️ PARTIAL` 还是 `✅`。

**学生追问**：`if __name__ == "__main__"` 还是没懂。

**AI 换角度**：Python 给每个文件一个身份证 `__name__`。直接运行 → `"__main__"` → if 成立。被 import → `"main"` → if 不成立。不写这行的话，别人 `from main import run_loop` 就会触发整个 agent 跑起来。

## loop.py StepTrace 走读

**AI 提问**：第 55 行 `tokens_used: int = 0` 的 `= 0` 是什么意思？

**学生**：初始没用 token 初始化为 0。不知道 field 用来干嘛的。

**AI**：`= 0` 答对了。`field()` 是因为 Python 的坑：`= []` 所有实例共享同一列表，`field(default_factory=list)` 每次创建新列表。`timestamp` 同理——`= datetime.now()` 时间冻结在 import 那一刻，`field(default_factory=lambda: datetime.now())` 每次创建才取时间。

**学生**：每个实例有自己的 field 领域是吧。

**AI**：✅ 对。

**AI 提问**：StepTrace 给谁看的？给模型看每一步的信息吗？

**学生**：是给人看的吗？不是给模型看每一步的信息嘛。

**AI**：给你看的。模型完全看不到 StepTrace——它只看 messages 数组。终端打印 `Step 1 | CONTINUE | get_price | 778 tk` 是 `_log_step()` 产生的，纯调试用。这就是 Ch.02 的 observability hook。

**学生**：`final_text: str | None = None` 和 `recent_calls: list[tuple[str, str]] = []` 是用来干嘛的？StepTrace 里面为什么没有？

**AI**：final_text 跨所有步骤，最后一步才填上。recent_calls 跨步骤滚动窗口防 doom loop。类比：StepTrace=每天记账单，traces=记账本，final_text=月底总结，recent_calls=银行卡近三笔消费提醒。

## loop.py 循环体走读

**AI 提问**：`response.choices[0].message`——choices 是什么？为什么取 [0]？空会怎样？

**学生**：就是模型的回复格式中的第一块？空就是模型返回空？

**AI**：choices=候选回复列表。API 支持 n=3（一次返回三个答案你挑一个），agent 设 n=1 永远只有一个。[0] 是防御习惯。空→IndexError 崩，但 API 保证至少返回一个，基本不会发生。

**AI 提问**：`if not msg.tool_calls`——这个判断是 Ch.02 的哪种停止条件？

**学生**：如果没有工具调用就说明已经得到工具执行的最终结果了。

**AI**：✅ 这就是 model-driven stop。

**AI 提问**：为什么要把 assistant 的 tool_calls 写回 messages？不追加会怎样？

**学生**：工具调用和结果都应该写到 msg 让模型看得到执行过程吧。

**AI**：对。模型无状态——每次调 API 都要把整个对话历史重新发。不追加 → 模型不知道上一步自己调了什么 → 看到 tool_result 一脸懵。

**学生**：每次调用我还需要把 prompt 和最开始的用户请求也发给模型吧。

**AI**：对。system prompt 和用户请求在 messages[0] 和 [1]，每次调 API 跟着后面累积的内容一起发。它们是锚。

**学生**：但是如果 AI 执行太多步 msg 中的信息会越来越长这不是浪费 token 嘛。它只保留三轮对话、system_prompt 和最开始的用户请求嘛？

**AI**：目前没截断——这是已知缺口。Ch.05 专门讲 compaction（压缩旧消息）。后续读到会加上。

**AI 提问**：`recent_calls.pop(0)` 和 `pop()` 有什么区别？为什么保留最近三个？

**学生**：pop 就是队列值保留三轮。pop(0) 去掉最早的工具调用只保留最近的三个？我还以为 recent_calls 是模型用来跟踪自己的工具调用步骤的，然后又不能保存太多轮——原来是用来检查是不是模型重复工具调用出问题了。

**AI**：✅ 完全正确。pop(0) 踢最早，pop() 踢最新。只存 (name, args)，不存结果——doom loop 看的是"在重复做同一件事"。

**AI 提问**：`tool_map.get(tool_name)` vs `tool_map[tool_name]`——模型幻觉了一个不存在的工具，分别怎样？

**学生**：`tool_map[tool_name]` 会直接报错然后 loop 就中断了，也没有记录中断问题在哪吧。

**AI**：✅ 完全正确。.get() → None → error 作为 tool_result → 模型看到后换方法。这就是 Ch.02 "errors are also turns"。

**AI 提问**：`tool.handler(**args)` 里的 `**args` 是什么语法？换成 `tool.handler(args)` 行不行？

**学生**：不指定参数个数的传法？一起传给函数？问题是应该是把工具名和调用方式传给函数，返回值是函数返回的 dict，不是传 dict 给函数吧。

**AI**：`**` = 拆字典。`get_price(**{"symbol":"600519"})` = `get_price(symbol="600519")`。不用 `**` → 整个 dict 塞进第一个参数 → symbol 变成 dict → 崩。

**AI 提问**：final_answer 解析失败时 `"answer" in result` 是 True 还是 False？

**学生**：final_answer 怎么会解析失败呢。TF 我不知道。error 肯定继续循环处理。

**AI**：昨天你亲眼看到——Step 2 和 Step 3 的 final_answer 因为 JSON 含控制字符解析失败。`"answer" in {"error":"..."}` → False → 不提前 return → 错误作为 tool_result 追加 → 模型下一轮换方式（直接文本输出）。
