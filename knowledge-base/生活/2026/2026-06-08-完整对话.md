### Ch.01 理论学习：一次工具调用

**阅读**：`course/01-function-calling.md`

**核心概念**：
1. 工具调用的四步循环：描述工具 → 模型发请求 → 你执行 → 结果返回
2. Schema 是合同：名称 + 描述（含"不要用于何时"） + 输入 schema
3. 错误应返回为 tool_result，不抛异常——抛异常杀死进程，模型看不到错误无法修正
4. 工具结果有形状：id 回环必须匹配，大结果要截断

### 量化 Agent 项目搭建（线1 启动）

**技术选型**：三个市场（A 股 + 加密货币 + 美股），一个接口

**环境搭建**：
- `uv init quant-agent` 在 `workspace/quant-agent/`
- 依赖：akshare（A股）+ ccxt（加密货币）+ yfinance（美股）
- Python 3.12.10，uv 0.11.8

**第一个工具**：`src/quant_agent/tools/prices.py`
- `GET_PRICE_SCHEMA`：工具定义（名称、描述含边界条件、参数 schema）
- `_detect_market()`：自动识别市场（6位数字→A股，含`/`→加密货币，字母→美股）
- `_fetch_a_stock()`：直接调东方财富 API（绕开 akshare 的批量爬虫检测）
- `_fetch_crypto()`：ccxt 调币安（国内需 VPN）
- `_fetch_us_stock()`：yfinance 调雅虎财经
- `get_price()`：统一 dispatch，错误全返回 dict 不抛异常

**运行结果**：
- ✅ A 股：贵州茅台 1263 元，跌 0.77%，成交 32.9 亿
- ❌ 加密货币：Binance 被墙（需 VPN）
- ❌ 美股：Yahoo 限速
