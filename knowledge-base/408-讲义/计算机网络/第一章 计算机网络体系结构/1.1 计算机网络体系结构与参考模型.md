### 分层思想的生活实例（物流网络）
物流网络采用了分层结构，分别是地区级、省级、市级、区级等。这样做的好处是，**不同级别的物流中心，只要思考如何解决自己这一层级的任务就好了**。

例如，地区级的物流中心思考的是如何在地区之间传输。省级的物流中心思考的是如何在同地区的省之间传输。市级的物流中心思考的是如何在同省的市之间传输。区级的物流中心思考的是如何在同市的区之间传输。街道级的物流中心思考的是如何在同区的街道之间运输。

“分层”可将庞大而复杂的问题转化为若干个较小的局部问题，而这些较小的局部问题就比较容易研究和处理，即“大事化小，小事化了”，大大减小了各个层级要解决的问题难度。

例如，你在网上购买了一件衣服，具体的传输流程如下：
1.  商家会将商品交付给A街道物流中心（上门取件）。
2.  街道中心会判断目标地址所在的街道是否同区，如果同区直接派送至另一个街道；不同区则交给区物流中心。
3.  区中心会判断目标地址所在的区是否同市，如果同市直接派送至另一个区；不同市则交给市物流中心。
4.  市中心会判断目标地址所在的市是否同省，如果同省直接派送至另一个市；不同省则交给省物流中心。
5.  省中心会判断目标地址所在的省是否同地区，如果同地区直接派送至另一个省；不同地区则交给地区物流中心。
6.  地区物流中心收到快件后，会发往另一个地区物流中心（不考虑更大的地理范围）。

分层思想其实在计算机组成原理以及操作系统中都见到过。

## 1.1 计算机网络体系结构与参考模型

在学习计算机网络原理时，往往会将计算机网络划分成五层，分别为物理层、数据链路层、网络层、运输层以及应用层。例如，进程 P1 将数据交给应用层的的实体（实体是指可发送或接收信息的硬件或软件）处理，随后再交给传输层的实体处理，以此类推，最终才变成光/电信号传输。为了让大家理解分层思想，下面举例说明。

## 知识提要

### （一）物理层

【例 1.1】假设有两台计算机，这两台计算机被一条电缆连了起来，双方通过这根电缆通信，即传输数据。

在计算机的世界中，一切都是用二进制数表示，即 $0/1$ 序列。物理层解决的是**比特传输问题**，不负责**解释比特流的含义**。例如，物理层要考虑如何表示“1”或“0”，可以用高电平表示 1，低电平表示 0。物理层双方约定的规则被称为协议。

此外，电缆等物理传输媒体并不属于物理层。物理层下面的传输媒体可以被看作第 0 层。

### （二）数据链路层

【例 1.2】假设现在有多台计算机通过总线互联成了一个小型网络，如下所示。

物理层解决了比特传输的问题，主机 A、B、C 可以畅快地发送比特流了。假设主机 A 要给主机 B 发送数据，A 发送的数据信号会通过总线传播到总线上的所有主机，那么主机 B 如何确认该数据是主机 A 发送给自己的，主机 C 又如何确认该数据并不是发送给自己的。

数据链路层负责解决这个问题。具体来说，每台主机一般都安装了网卡，网卡主要解决物理层和数据链路层的任务。每个网卡都有一个固定的 MAC 地址（先不纠结 MAC 的含义），类似于网卡的身份证。

网卡收到一串比特流后，**可以划分成一个个数据帧**。一种典型的数据帧由首部、数据以及尾部三部分构成。

| 数据帧1 | 数据帧2 | 数据帧3 |
| :--- | :--- | :--- |
| 01010100011010101 | 01010100011010101 | 01010100011010101 |
| **首部** \| **数据** \| **尾部** | | |

网卡在发送数据时会在首部中加上源 MAC 地址和目的 MAC 地址。

| A | B | ...... |
| :--- | :--- | :--- |
| **首部** | **数据** | **尾部** |

当其他主机收到后，根据目的 MAC 地址和自身 MAC 地址是否匹配，来决定是否接受该数据，还可以通过源地址知道是哪个主机发来的数据。

另外，若主机 A 和主机 C 同时使用总线来发送数据，则会造成**信号碰撞**，即双方的信号互相影响。

因此，数据链路层还需要解决信号的碰撞问题。由于总线的碰撞问题，现在一般使用以太网交换机将多台设备互连。交换机是一种数据链路层设备，不同的端口连接不同的网络设备。它能够识别 MAC 地址，若它已经记录了某个端口对应的网络设备，则仅会从这个端口将数据转发出去。交换机可以同时处理多个发送方发来的数据，进而解决了主机争用总线而产生碰撞的问题。

### （三）网络层

解决了物理层和数据链路层的问题后，就可以实现数据在多台主机构成的网络上传输了。然而，随着计算机网络的发展，出现了许多异构网络，这些异构网络之间所采用的物理层协议和数据链路层协议有所不同，可能无法直接通信。因此，需要网络层来解决这个问题。

【例 3.1】假设网络 1 和网络 2 的架构不同，两个网络表示比特 1 和比特 0 的方法有所不同，需要通过路由器这种网络设备相连。一般用一朵云表示一个网络。

路由器的不同接口连接不同的网络时，可以配置不同的物理层和数据链路层协议。

| 层次 | 左侧主机 | 路由器 (E1 / E2) | 右侧主机 |
| :--- | :--- | :--- | :--- |
| **应用层** | 应用层 | - | 应用层 |
| **运输层** | 运输层 | - | 运输层 |
| **网络层** | 网络层 | 网络层 | 网络层 |
| **数据链路层** | 数据链路层 | 数据链路层 / 数据链路层 | 数据链路层 |
| **物理层** | 物理层 | 物理层 / 物理层 | 物理层 |

注：在讨论计算机网络原理时，往往把单个网络简化成一条链路。

有了路由器后，可以将多个网络相连。例如，人们几乎每天都会使用的互联网，是由若干网络和连接这些网络的路由器组成的，互联网也可用一朵云表示。

不过这也产生了新的问题，全世界的网络数量众多，怎么指明某个计算机属于哪个网络呢？因此还需要对互联网中的各网络以及网络中的各个主机编址。

“IP 地址”便是网络层中的设备地址，每个 IP 地址由网络号和主机号构成，处于同一网络的设备网络号相同，主机号不同，进而非常方便地标识了不同网络以及同一网络中的不同设备。

### （四）传输层
解决了物理层、数据链路层以及网络层的问题后，就可以实现数据在多个网络的不同主机之间传送了。然而，一台主机上往往有多个进程，需要标识主机中不同的进程。因此，需要传输层来解决这个问题。

### （五）应用层
解决了物理层、数据链路层、网络层以及运输层各自的问题后，就实现进程之间的通信了。进程之间仅仅实现通信还不够，每个进程实现的功能不一样，因此还需要根据不同的功能设计不同的应用层协议。例如，针对Web服务提出了HTTP协议，针对电子邮件产生了SMTP协议等（第六章学习）。

## 数据封装与传输过程

## 知识提要

### 1. 主机1对数据的处理过程
① 根据应用层协议（例如 HTTP），给用户数据添加一个应用层首部，构建一个应用报文。随后，应用层将构建好的报文向下交付给传输层。
② 根据传输层协议（例如 TCP），给报文添加传输层首部，构建一个传输层报文段。传输层首部包含了用于区分各个进程的控制信息。例如，可以给各个进程编号，进而实现区分。传输层将封装好的 TCP 报文段向下交付给网络层。
③ 为传输层报文段添加一个 IP 首部，将其封装成 IP 数据报。IP 首部的主要作用是 IP 寻址和路由。网络层将封装好的 IP 数据报向下交付给数据链路层。
④ 为 IP 数据报添加一个首部和尾部，将其封装成帧。帧首部和尾部的主要作用是 MAC 寻址和差错校验。数据链路层将封装好的帧向下交付给物理层。
⑤ 物理层并不认识帧的结构，仅仅将其看作比特流，以便将比特流转换成相应的电信号进行发送。

### 2. 路由器对数据的处理过程
① 将收到的电信号转换成比特流，然后将向上交付给数据链路层。
② 数据链路层去掉帧的首部和尾部后，将 IP 数据报向上交付给网络层。
③ 网络层从 IP 数据报的首部中提取出目的 IP 地址，根据目的 IP 地址决定从哪个接口转发该 IP 数据报。然后将该 IP 数据报向下交付给转发口的数据链路层。
④ 转发口的数据链路层为 IP 数据报添加首部和尾部，将其封装成帧，然后将帧向下交付给物理层。
⑤ 转发口的物理层将看作比特流，转变成相应的电信号发送出去。

### 3. 主机2对数据的处理过程
当这一串的比特流离开路由器到达目的站主机 2 时，就从主机 2 的第 1 层依次上升到第 5 层。最后，把进程 P1 发送的数据交给进程 P3。
虽然进程数据要经过上述复杂过程才能送到终点，但这些复杂过程被屏蔽掉了，进程 P1 觉得好像是直接把数据交给了进程 P3。在文献中会见到术语“协议栈”，这是因为几个层次画在一起很像一个栈的结构。

### （二）协议
协议是为实现网络中的数据交换而建立的规则或约定。

### （三）服务
要实现本层通信，还需要使用下面一层所提供的服务（功能调用），即 $n$ 层向 $n+1$ 层提供服务。其中，第 $n$ 层称为服务提供者，第 $n+1$ 层称为用户。

在同一系统中相邻两层的实体交换信息的接口称为服务访问点（Service Access Point, SAP），类似于上层调用下层提供的函数。上层需要通过与下层交换一些命令来调用下层服务，这些命令被称为服务原语。（了解即可）

【注意】协议是“水平”的，而服务是“垂直”的。实体看得见下层提供的服务，但并不知道实现该服务的具体协议，即下层的协议对上层的实体是“透明”的。

### （四）协议的三要素
计算机网络协议有三个要素，分别是语法、语义以及同步，不妨举个简单的例子来理解其含义。

【例 1.1】我给张三打了个电话，他接到电话后，我告诉他，“我吃了五个包子”。
*   **语法**：主语+谓语+宾语
*   **语义**：这句话对应的现实含义。
*   **同步**：我先打电话，张三随后得接电话，然后我们才能通话。

语法确定了通信双方“如何讲”，语义确定了通信双方“讲什么”，而同步则确定了通信双方“讲话的次序”。

#### 1. 语法
语法指数据与控制信息的结构或格式。例如，以太网数据帧的格式如下所示，语法表明了各个字段的长度和先后顺序。

| :--- | :---: | :---: | :---: | :---: | :---: |
| 长度（单位：字节） | 6 | 6 | 2 | 46~1500 | 4 |

#### 2. 语义
语义指需要发出何种控制信息，完成何种动作以及做出何种响应。例如，发送方和接收方都发出了报文，但是由于报文的控制信息不同，可以分为请求报文以及响应报文。

### （五）PDU 和 SDU
对等层次之间传送的数据单位称为该层的协议数据单元（Protocol Data Unit, PDU）。

例如，物理层对等实体间通信的 PDU 称为比特流；数据链路层对等实体间通信的 PDU 称为帧；网络层对等实体间通信的 PDU 称为分组或 IP 数据报；运输层对等实体间通信的 PDU 一般根据协议而定，若使用 TCP 协议，则称为 TCP 报文段，若使用 UDP 协议，则称为 UDP 用户数据报；应用层对等实体间通信的 PDU 一般称为应用报文。

协议数据单元由协议控制信息（PCI, Protocol Control Information）以及服务数据单元（SDU, Service Data Unit）两部分构成，其中 PCI 为控制信息，SDU 为传输的数据。例如，主机 1 在和主机 2 进行通信时，主机 1 对数据的处理流程如下。
① 首先，将用户所需要传输的 SDU 下发到第 5 层（应用层）。
② 在第 5 层加上了对应的 PCI 后就作为了第 4 层（传输层）的 SDU。
③ 同样地，该 SDU 在第 4 层加上了对应的 PCI 后继续往下层下发。

注意，当到达第 2 层（数据链路层）的时候，PCI 可以有两部分，分别附加到数据单元的首部和尾部。
因此，在各层间传输数据时，一般将第 $n$ 层的 SDU 记为 n-SDU，第 $n$ 层的 PCI 记为 n-PCI，
$$ \text{n-SDU} + \text{n-PCI} = \text{n-PDU} = (n-1)\text{-SDU} $$
另外，可以将多个 SDU 合并成一个 PDU，也可以将一个 SDU 划分成几个 PDU。

### 五、OSI 模型（理解并记忆）
国际标准组织 ISO 于 1977 年成立了专门机构研究计算机网络如何分层问题。他们提出了著名的开放系统互连基本参考模型 OSI/RM（Open Systems Interconnection Reference Model），简称为 OSI。OSI 参考模型是一个七层协议的体系结构，自下而上依次是物理层、数据链路层、网络层、运输层、会话层、表示层以及应用层。下面逐一介绍 OSI 模型中每个层次的作用。

| :--- | :--- |

| 表示层 | 表示层 |

| 物理层 | 物理层 |

### （一）物理层
物理层负责在传输媒体上透明地传输比特流。

### （二）数据链路层
数据链路层的任务是在相邻节点（主机、交换机、路由器等节点）之间的链路上传送数据帧。每一帧包括数据和必要的控制信息（如地址信息、差错控制等），可以实现相邻结点间的流量控制。

### （三）网络层
网络层负责为网络上的不同主机提供通信服务。网络层首先将传输层下发的 PDU 封装成分组，然后选择合适的路由再进行传送。为了避免转发过程中在网络上出现过多的分组而造成阻塞，网络层还可以进行拥塞控制。

### （四）传输层
传输层（运输层）负责向两台主机的多个进程提供通用的数据传输服务。“通用的”是指并不针对某个特定网络应用，而是多种应用可以使用同一个运输层服务。

### （六）表示层
表示层负责根据应用层的要求对数据格式进行处理，将二进制数转换成人们能够识别的数据，如文字、图片、音视频编码，此外表示层还可以对文件进行压缩/解压缩、加密/解密等。

OSI 模型的低三层统称通信子网，完成数据的传输功能；高三层统称资源子网，完成数据的处理等。传输层承上启下。端到端通信针对传输层而言，表示在两个进程之间进行通信；点对点通信针对数据链路层或网络层而言，表示在两个设备之间通信。

## 六、TCP/IP 模型

### （一）基本概念
OSI 体系结构是法律上的国际标准，它的七层协议体系结构的概念清楚，理论也较完整，但它既复杂又不实用。到了 20 世纪 90 年代初期，尽管 OSI 国际标准已经制订出来了，但这时因特网已在全世界流行起来了。因特网是全球覆盖范围最广、用户数量最多的互联网，它采用 TCP/IP 参考模型，即事实上的国际标准。TCP/IP 模型是一个四层协议的体系结构，自下而上依次是网络接口层、网际层、运输层以及应用层。

## 知识提要

TCP/IP 体系结构相当于将 OSI 体系结构的物理层和数据链路层合并为了网络接口层，将会话层和表示层合并到了应用层。

| OSI体系结构 | TCP/IP体系结构 |
| :--- | :--- |
| 应用层、表示层、会话层 | 应用层 |
| 传输层 | 传输层 |
| 网络层 | 网际层 |
| 数据链路层、物理层 | 网络接口层 |

TCP/IP 在网络层使用的核心协议是 IP 协议，IP 协议的中文意思是网际协议（Internet Protocol, IP），因此 TCP/IP 体系结构的网络层也常称为网际层。OSI 旨在传输层仅提供面向连接的服务，但实际应用的 TCP/IP 体系结构并不符合其设想，TCP/IP 下的传输层主要使用以下两种协议：
（1）传输控制协议 TCP（Transmission Control Protocol）提供面向连接的、可靠的数据传输服务。
（2）用户数据报协议 UDP（User Datagram Protocol）提供无连接的尽最大努力的数据传输服务（不保证数据传输的可靠性）。

**PS**：面向连接服务类似打电话，如果 A 和 B 想打电话，则 A 必须先拨打 B 的号码（发送请求），B 接到 A 的电话后接通（发出响应），A 收到 B 的响应后，便可以开始发送数据，即建立连接后双方才可开始正式电话交流（数据传输）。

无连接服务下，不需要双方建立连接才可以发送数据，省去了建立连接的时间。
倘若某个服务保证接收方能正确接收发送方的所有数据，则称为可靠服务，否则称为不可靠服务。
TCP/IP 模型中的网络层只向上层提供无连接、尽最大努力交付的数据报服务，但其中传输层则既提供面向连接的服务，也提供无连接的服务。
ISO/OSI 参考模型中的网络层既提供面向连接的服务，也提供无连接的服务，但其中的传输层只提供面向连接的服务。

| :--- | :---: | :---: |
| | **ISO/OSI 模型** | **TCP/IP 模型** |
| 传输层服务 | 面向连接 | 面向连接/无连接 |
| 网络层服务 | 面向连接/无连接 | 无连接 |

## （二）TCP/IP 协议族
由于人们几乎每天都在使用互联网，因此在个人设备的操作系统中，通常都带有完整的 TCP/IP 协议族。而因特网中的路由器的基本任务是网络互连，只包含 TCP/IP 的网络接口层和网际层即可。因此，一般认为路由器的网络体系结构的最高层为网际层（网络层）。

### 沙漏形状的 TCP/IP 协议族
还有一种方法用来表示 TCP/IP 协议族，它的特点是上下两头大而中间小：应用层和网络接口层都有多种协议，而中间的 IP 层很小，上层的各种协议都向下汇聚到一个 IP 协议中。

*   **图示结构（沙漏模型）**：
    *   **应用层**：HTTP ... SMTP，DNS ... RTP
    *   **传输层**：TCP，UDP
    *   **网络层**：IP
    *   **数据链路层**：网络1，网络2 ... 网络3

这种很像沙漏计时器形状的 TCP/IP 协议族表明：IP 层可以支持多种运输层协议（这里只画出了最主要的两种），而不同的运输层协议上面又可以有多种应用层协议，同时 IP 协议也可以在多种类型的网络上运行。正因为如此，互联网才会发展到今天的这种全球规模。

### 计算机网络分层的好处与缺点
计算机网络分层能带来很多好处，主要包括（了解即可）：
① 各层之间是相对独立的。某一层并不需要知道它的下一层是如何实现的，而仅仅需要知道该层通过层间的接口（即界面）所提供的服务。
② 灵活性好。当任何一层发生变化时，只要层间接口关系保持不变，那么在这层以上或以下各层均不受影响。
③ 能促进标准化工作。因为每一层的功能及其所提供的服务都已有了精确的说明。

分层当然也有一些缺点，例如，有些功能会在不同的层次中重复出现。
