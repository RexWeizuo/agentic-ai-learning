1.数据结构
在实现文件的基本操作时，需要用到以下数据结构。
（1）位于外存上的数据结构
① 目录文件：可以通过文件名来寻找相应的 FCB。

② 进程打开文件表（文件描述符表）：每一个进程都有这样一张表格，它描述了在进程内部所打开的所有文件，包括每个文件的打开方式、当前读写指针、该文件在系统打开文件表当中的索引等信息。

2.打开文件
任何一个文件在使用之前都必须先打开，只有在打开文件以后，才能对它进行属性访问和读写等操作。
打开一个文件的系统调用一般为如下形式：fd=open(文件路径名，打开方式);
“打开”是指系统将指定文件的属性信息从外存复制到内存中的打开文件表的一个表目中，具体流程如下。
① 文件系统会调用目录结构方面的服务功能，根据文件的路径名一层层地去查找各级目录结构，直到找到该文件所在的目录项。
【例】假设访问一个名为 example.txt 的文件，该文件位于路径/home/documents/example.txt。文件系统首先解析这个路径名，将其分解为各个组成部分：home 和 documents 目录，以及最终的 example.txt 文件。

[图片：目录树结构图，根目录/下包含home, dev, bin, user等目录，home下包含documents目录]

文件系统从根目录 / 开始查找。根目录是所有文件和目录的起点。接着，在根目录下查找名为 home 的目录项。找到 home 目录后，文件系统进入这个目录，继续查找下一个组成部分 documents。最终，在 documents 目录中查找到名为 example.txt 的目录项。至此，文件系统已经成功地根据路径名找到了文件所在的目录项。
② 根据文件的打开方式和访问权限等信息检查这次访问的合法性，如果不合法，则返回一个出错编码。
③ 若本次访问合法，则查找系统打开文件表，看看该文件是否已经被其他进程打开。
④ 如果是，就直接把它的共享计数值加 1；如果不是，说明这是第一次打开该文件，因此要把它的 FCB 从外存读入到系统打开文件表中，然后把共享计数值初始化为 1。
⑤ 在进程打开文件表中增加一项，填写该文件的访问方式、当前读写指针等。另外，在这个表项中还要记录该文件在系统打开文件表中的相应表项，用一个指针来指向它。
⑥ 返回一个指针（整数）给 fd（文件描述符），指向进程打开文件表当中的相应表项，以后的文件操作均通过该指针来完成。

3.关闭文件
文件在使用完毕后必须关闭，系统调用函数为：close(fd); fd 为打开文件时所返回的指针。
关闭一个文件的具体过程如下：
① 根据 fd 到进程打开文件表中，将该文件所对应的表项删除。
② 到系统打开文件表中，将该文件所对应的表项的共享计数值减 1。如果该值仍然大于 0，这说明还有其他进程在使用该文件，因此本次关闭文件的操作结束，直接返回；如果该值等于 0，这说明已经没有其他进程在使用该文件，如果有的文件信息被更新过，那么就要把它们写回到硬盘的目录结构中，然后把系统打开文件表中的相应表项删除。
4.读文件
读文件的系统调用函数为：int read(fd , userBuf , size);
其中，fd 是文件指针，userBuf 是用户提供的一个缓冲区，size 是需要读取的字节数。
read 函数的功能是：从文件 fd 的当前位置开始，顺序读取大小为 size 的数据块，并保存在 userBuf 中。
[图片：文件的逻辑地址空间示意图，展示了文件的起始位置、当前位置、结束位置，以及已访问部分和即将访问部分]
具体实现过程如下：
① 通过 fd 可以访问进程打开文件表中的相应表项，如文件的当前位置，并可以由此访问系统打开文件表中的相应表项，即该文件的 FCB。由此可知文件的各个逻辑块在外存上的存储位置。
② 验证本次操作的合法性，包括读取权限、数据块大小是否越界等。
③ 以块为单位来访问外存，需将用户给出的文件地址转换为逻辑块，再转换为相应的物理块，根据物理块编号去访问外存。

（一）组和用户
一台计算机往往有多个用户使用，因此 OS 往往需要为不同用户创建相应的账号，例如，我们可以通过如下命令创建 user1、user2、user3 三个用户账号。
```bash
leon@leon-virtual-machine:~$ sudo adduser user1
```

为了方便对用户的管理，我们可以把若干个用户划分进同一组。“组”是一种方便管理用户权限的方式，可以通过给组分配权限，而不需要单独为每个用户配置。
例如，我们可以通过如下命令将三个用户划分进组 group1 以及组 group2。
```bash
leon@leon-virtual-machine:~$ sudo usermod -aG group1 user1
```

（二）文件保护
为了防止未经核准的用户修改甚至破坏文件，文件系统必须拒绝来自未经授权的用户和组的访问，限制授权用户和组的访问类型。
在文件系统中，可以通过访问控制表、口令和密码等方式实现文件保护。
1.访问控制表
访问控制表的原理为：为每个文件配置一个访问控制表，表中记录每个用户的名称及其允许的访问类型，如只读、可读可写等。
当用户请求访问特定文件时，操作系统会首先检查访问控制表中当前用户的访问权限。如果合法，则允许访问，否则拒绝访问。
【例】假设为单个文件 example.txt 设置访问控制表。

| 用户 | 权限类型 |
| :--- | :--- |
| Alice | 只读 |
| David | 可读可写 |
| Bob | 只读 |

优点：可以对每个单一用户设定其所允许的访问类型，能够进行复杂的访问设定。
缺点：若系统用户量大，则维护困难，长度不可预计。
2.精简的访问控制列表
采用精简的访问列表可以解决访问控制表的问题，如下所示。

| 所有者 | 组 | 其他 |
| :--- | :--- | :--- |
| 读取 写入 执行 | 读取 写入 执行 | 读取 写入 执行 |
| 1 1 1 | 1 0 1 | 0 0 0 |

精简的访问控制列表示例
精简的访问列表只需要 9 位二进制数就能描述文件的权限。
① 创建文件时，系统将文件所有者的名字、所属组名记录在文件 FCB 中。
② 用户如果是所有者，按所有者权限访问文件，否则若用户和所有者属于同一组，按同组权限访问，否则按其他用户权限访问。
口令和密码是另外两种访问控制方法，都是防止文件被未授权用户窃取，并没有控制对文件的访问类型。
3.口令
口令是由用户在建立文件时所设定的字符串，其保存在文件元数据中。任何用户请求访问时，都必须提供正确的口令，否则无法访问文件。类似于 QQ“密码”（我们俗称的“密码”其实是口令）。
其优点是简单、便于储存，缺点是其直接存在系统内部且是静态的，易被破解。
4.密码
密码是一种对文件加密的算法，是指将文件中的数据通过一种变换手段，变为除有密钥的人以外其他人所不能读懂的信息编码。
【例】假设你使用了加密软件对一份重要文档进行加密。在这个过程中，加密软件采用了特定的加密算法文档内容中的“今天天气真好”经过加密算法加密后，可能变成一串乱七八糟的字符，比如“# % & ^ & * (YHGFDS”。当你想查看这份文档时，需要使用对应的密钥（可以理解为开启加密的“钥匙”）和解密算法（与加密算法相对应的“还原规则”），将“# % & ^ & * (YHGFDS”这样的密文还原成原来的“今天天气真好”。
其优点是比口令安全，保密性强，缺点是加密和解密需要花费时间。

② 再进行组内顺序查找。
索引顺序文件的查找类似于数据结构中的分块查找。

[图片：分块查找示意图，上方为索引表（7, 15, 23, 29 对应地址 0, 4, 8, 12，标注“块间有序”），下方为查找表（分为四块，标注“块内无序”）]

分块查找的平均查找长度由索引查找（块间查找）和块内查找两部分构成，设索引查找和块内查找的平均长度之和分别为 $L_I$ 和 $L_2$。则分块查找的平均查找长度：
$$ASL = L_I + L_2$$

索引顺序文件分组的最好情况是，N 条记录分为 $\sqrt{N}$ 组，每组有 $\sqrt{N}$ 条记录。
顺序查找索引表平均需查找 $\sqrt{N}/2$ 次，再在组内顺序查找也平均需找 $\sqrt{N}/2$ 次，即共需查找 $\sqrt{N}$ 次。
优点：极大地减少了访问单条记录的时间，同时保留了文件的顺序特性。
缺点：配置索引表同样需要增加存储开销。

【补充】提高磁盘 I/O 速度的途径（了解即可）
目前，磁盘的 I/O 速度远低于对内存的访问速度，因此，该问题已成为计算机系统的瓶颈。
提高磁盘的 I/O 速度的最主要的技术便是磁盘高速缓存。磁盘高速缓存，是指在内存中为磁盘盘块所设置的一个缓冲区，在该缓冲区中保存了某些盘块的副本。
当出现一个磁盘访问请求时，先去查看所请求的盘块内容是否已在磁盘高速缓存中，如果在，则可从磁盘高速缓存中直接获取，这样就省去了启动磁盘的操作。
【例】如果把一个大文件从 C 盘复制到 D 盘，那么该文件的很多内容就会在内存的缓冲区当中。如果以后还要访问该文件，如再把它复制一份到 E 盘，那么速度就会比刚才要快，因为有一些内容已经在内存当中了。

[图片：磁盘（C盘、D盘、E盘）与内存（缓冲区）示意图]

除了磁盘高速缓存外，可以有效提高磁盘 I/O 速度的方法还有很多，如提前读、延迟写、优化物理块的分布、虚拟盘等。下面将对这些方法进行具体介绍。
1.提前读
如果采用顺序访问方式对文件进行访问，则可以预知下一次要读的盘块。此时可采取预先读的方式，即在读当前块的同时要求将下一个盘块（提前读的盘块）中的数据也读入缓冲区。

[图片：提前读示意图，左侧为磁盘块编号0-31，中间为内存中的缓冲区内容（count, f, mail, tr, list等），右侧为内存]

这样，当要访问下一个盘块中的数据时，可直接从缓冲区中取得，而无须再去启动磁盘 I/O，从而有效地提高了磁盘 I/O 的速度。“提前读”这一方法已被广泛采用。
2.延迟写
延迟写是指缓冲区中的数据本应立即写回磁盘，但考虑到该缓冲区中的数据可能会在不久之后再次被本进程或其他进程访问，因而并不会立即将该缓冲区中的数据写回磁盘，而是会将它挂在空闲缓冲区队列

4.虚拟盘
由于访问内存的速度远高于访问磁盘的速度，于是有人试图利用内存空间去仿真磁盘，进而形成所谓的虚拟盘，又称为 RAM 盘。
该盘的设备驱动程序可以接受所有标准的磁盘操作，但这些操作的执行不是在磁盘上而是在内存中进行。它们对用户而言都是透明的，即用户不会发现这与真正的磁盘操作有何不同，仅是略微快了些而已。
