# 1.3 操作系统特性功能

‍1.3 操作系统的特性以及功能
前面介绍的多道批处理系统、分时系统和实时系统这三种基本操作系统都具有各自不同的特征，如批处理系统有较高的资源利用率和系统吞吐量；分时系统能获得及时响应；实时系统具有实时特征。除此之外，它们还共同具有并发、共享、虚拟和异步四个基本特征。

一、操作系统的基本特征（理解即可）
（一）并发性和并行性
并行性是指两个或多个事件在同一时刻发生，并发性是指两个或多个事件在同一段时间内发生。
若当前计算机系统只有一个CPU，多道程序环境下，并发性是指在一段时间内宏观上有多个程序在同时运行，但每一时刻却仅能有一道程序执行，故微观上这些程序只能是分时地交替执行。

[图片：单CPU并发执行示意图，展示主存中有甲、乙、OS程序，CPU和I/O设备交替处理甲和乙的任务，时间轴显示甲乙交替执行]
注：不考虑程序切换带来的开销
倘若在计算机系统中有多个处理机，这些程序便可被分配到多个处理机上，实现并行执行。

[图片：多CPU并行执行示意图，展示主存中有甲、乙、OS程序，两个CPU分别处理甲和乙的任务，实现并行]

（二）共享
共享指系统中的资源可供内存中多个并发执行的程序共同使用。
资源共享主要可分为互斥共享、同时访问两种方式。
1. 互斥共享方式
打印机虽然可供多个程序使用，但为使得打印结果不会出现内容不一致的问题，应规定在一段时间内只允许一个程序访问该资源，即采用互斥共享方式使用打印机。
【例】倘若当前有两个程序A和B分别负责打印408真题和考研英语真题，若不采用互斥共享方式使用打印机，则可能一张A4纸上又有408真题，又有英语真题！

[图片：打印机示意图，旁边方框显示打印结果混乱，交替出现“408真题”和“英语真题”]

为此，当程序 A 访问某个资源时，必须先提出请求，若此时该资源空闲，则系统便将之分配给 A 使用，此后有其他程序也要访问该资源时（只要 A 未用完）就必须等待。
仅当 A 访问完并释放该资源后，才允许另一个程序对该资源进行访问。

[图片：进程A和进程B请求打印机的示意图，展示了互斥访问过程]

将这种资源共享方式称为互斥共享，而将在一段时间内只允许一个程序访问的资源称为临界资源。
注：这里的“程序”用“进程”更准确，为了便于小白理解，做了简化，仅供参考
2. 同时访问方式
系统中还有另一类资源，这类资源允许在一段时间内由多个程序“同时”访问。
这里所说的“同时”通常是宏观上的，而在微观上，这些程序可能是交替地对该资源进行访问，即“分时共享”的。内存、磁盘都支持同时访问方式。例如，当程序甲和程序乙并发执行时，在一段时间内，甲和乙会交替访问内存。

[图片：主存、CPU、I/O设备示意图，展示了甲、乙程序在内存中]

并发和共享是操作系统两个最基本的特征，两者之间相互依赖。
①若系统不允许程序并发执行，则不存在资源共享问题。
②若系统不能对资源共享实施有效管理，则会影响程序之间的并发执行。
（三）虚拟（了解即可）
虚拟可以简单理解为，通过虚拟技术“增加”某个资源的数量或“扩大”其能力。
这里的“增加”和“扩大”都打了引号，即资源的真实数量或真实能力都没有变化，只是用户的感受发生了变化，他以为资源变多或者变强了！

[图片：未使用虚拟技术时用户看到的打印机（一台）与使用虚拟技术后用户看到的打印机（多台）对比，包含一个震惊的熊猫头表情包]

虚拟技术可分为：时分复用技术（如虚拟处理器）和空分复用技术（如虚拟存储器），这里仅以虚拟处理器举例说明，其他内容在后续章节中深入学习。
在分时操作系统中，处理器的运行时间分成很短的时间片，按时间片轮流将处理器分配给各作业使用。
分时 甲 乙 甲 乙 甲 乙 甲 甲 甲

（四）异步（了解即可）
若当前计算机中只有一个 CPU，则每次只允许一个程序执行，其余程序只能等待。
若正在执行的程序乙提出打印请求时，而此时打印机正在为甲程序打印，则程序乙必须等待，并将处理机让给丙执行。

直到打印机空闲，并且乙再次获得处理机时，方能继续执行。可见，由于资源限制，程序乙的执行通常不能“一气呵成”，而是以“停停走走”的方式运行。每个程序何时能获得处理机运行，何时又因提出某种资源请求而暂停，以及每道程序总共需要多少时间才能完成等等，都是不可预知的。因此，程序是以人们不可预知的速度向前推进的，即异步性。

二、用户接口（理解即可）
为了方便用户使用，操作系统必须向用户提供接口，它负责用户与操作系统之间的交互。
“插座”就是典型的“接口”，我们不必关心插座到底是如何实现供电的，需要用电的时候直接把插头插到插座上即可。
用户接口可以进一步分为命令控制接口和程序接口。（划分方式不唯一）
一方面可以利用命令控制接口控制程序的执行，另一方面可以利用程序接口请求操作系统的服务。

[图片：用户接口层级结构图，展示用户通过应用程序和操作命令分别调用系统调用和界面管理，最终到达操作系统内核和硬件]

（一）命令控制接口
命令控制接口是操作系统面向普通用户的接口，它包括命令接口和图形用户接口。
1. 命令接口
使用命令接口进行作业控制的主要方式有两种，即脱机控制方式和联机控制方式。
①联机控制方式
联机控制方式是指用户利用系统提供的命令和系统会话，交互式地控制程序的执行。其工作过程是用户通过终端输入操作命令，向系统提出各种服务要求。

```bash
leon@leon-virtual-machine:~/OS$ ./hello
hello world!

```

用户每输入一条命令，系统接收后便执行相应的指令，随后等待用户输入下一条命令。
②脱机控制方式
脱机控制方式是指用户将对作业的控制要求以作业说明书的方式提交给系统，由系统按照作业说明书的规定控制作业的执行。下面是一个简单的 Shell 脚本示例，它将执行几个基本的命令：

```bash
#!/bin/bash

# 这是一个注释，下面的脚本将打印出当前日期和时间

echo "当前日期和时间是: $(date)"

# 列出当前目录下的所有文件和文件夹
echo "当前目录下的文件和文件夹有:"
ls -l

# 创建一个名为"example_directory"的目录
mkdir example_directory

# 打印出新创建的目录的内容
echo "example_directory目录的内容:"
ls -l example_directory

# 退出脚本
exit 0
```

（二）程序接口
程序接口是操作系统面向程序员的接口，它是由一组系统调用组成。
系统调用把应用程序的请求传给内核，内核调用相应的内核函数完成所需的处理，并将处理结果返回给应用程序。

通常，一个操作系统提供的系统调用命令有几十乃至上百条之多，这些系统调用按功能大致可分为如下几类（一轮可以直接跳过，二轮再看）：
①进程管理。该类系统调用完成进程的创建、撤销、阻塞及唤醒等功能。
②进程通信。该类系统调用完成进程之间的消息传递或信号传递等功能。
③内存管理。该类系统调用完成内存的分配、回收以及获取作业占用内存区大小及起始地址等功能。
④设备管理。该类系统调用完成设备的请求或释放，以及设备启动等功能。
⑤文件管理。该类系统调用完成文件读、写、创建及删除等功能。
