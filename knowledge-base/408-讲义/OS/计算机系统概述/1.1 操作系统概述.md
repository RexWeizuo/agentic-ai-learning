# 1.1 操作系统概述

‍学完计组之后，大家就正式进入操作系统的世界了！

一、操作系统不过是个“大型程序”（理解即可）
（一）初识操作系统
我们在日常生活中经常使用操作系统（Operating System），包括 WindowsOS、LinuxOS、MacOS 等。

[图片：Windows 10、Linux、macOS 的图标]

无论是 Windows，还是 Linux，还是 mac 系统，其实都是一个大型程序，即由一行行代码构成！
例如，Linux 系统的主程序文件 init/main.c 包含如下的代码片段（不需要看懂）：

```c
void start_kernel(void)
{
    char *command_line;
    char *after_dashes;
    set_task_stack_end_magic(&init_task);
    ......
}
```

整个 Linux 系统包含上千万行代码，这些代码共同组成了一个非常庞大的程序。因此，学习操作系统，其实就是学习这个大型程序的整体代码逻辑，跟学习如何用 C 语言在显示屏上输出“hello world”无异，只是这个程序要复杂一些！

（二）程序运行流程
程序是由一条条指令和数据构成的。程序执行前，要将程序所含的指令和数据送入主存储器。

三、操作系统的发展历史（理解即可）
操作系统的发展，大致可以分为以下几个阶段。
（一）手工操作阶段（此阶段无操作系统）
第一个阶段是 20 世纪 50 年代，那时，计算机刚刚诞生不久，价格非常昂贵，主要是作为一种实验装备存在于各个学校、政府机构的实验室当中。
在这个阶段，程序设计全部采用机器语言，没有汇编语言，也没有高级程序设计语言，更谈不上操作系统。机器语言是最低级的语言，是一种由计算机硬件直接执行的指令集，由一系列二进制代码组成。每个代码都对应硬件的一个基本操作。

```
0010 1101 1011 1010
1010 0001 1001 1010
0110 1101 1111 1011
0011 1001 1011 1110
机器语言代码
```

程序的输入与输出，主要通过纸带或卡片来完成，例如，有孔表示 1，没孔表示 0。
程序员的编程，就是在纸带或卡片上编写机器语言指令，即 0/1 形式的指令，然后把写好的程序交给计算机，计算机在计算出结果后，同样以这种纸带的形式给出结果。

装入纸带 -> 纸带机 -> 读纸带 -> 计算机 -> 写纸带 -> 打印机 -> 取走纸带

在一个程序员上机期间，他总是在控制台前调试程序，需要占用整台机器的所有资源，包括主机和外设等。程序员除了写程序外，还得当操作员，手工操作计算机。
在上机过程中，一次只能完成一个功能，各个功能之间没有重叠。
例如，当程序员正在手工装入或取走纸带时，CPU 就停在那里等待，这样 CPU 的时间就浪费了。而当程序员在思考问题或编写程序时，那些输入输出设备也停在那里等待。

与计算机相比，用户（程序员）操作的速度是非常慢的。因此，这个阶段的主要问题是：由于手工操作的效率低下，造成了 CPU 等待的时间过长，从而导致了资源的浪费。

（二）批处理阶段（理解即可）
为了克服第一个阶段中的问题，提高计算机的使用效率，就必须尽可能地减少手工操作。因此，在 20 世纪 50 年代末到 60 年代中，人们设计了批处理管理程序（监督程序），来实现作业（程序）的自动转换处理，这就是操作系统的雏形。
批处理的基本思路是把程序员和操作员这两个角色分开，程序员在编写程序后，把程序提交给操作员，由操作员完成作业的输入。最后，由批处理管理程序完成作业的调入、运行以及结果的保存。
批处理系统分为单道批处理系统和多道批处理系统。

1.单道批处理
单道批处理分为联机批处理和脱机批处理两种类型。
（1）联机批处理（了解即可）
联机批处理的工作方式如下：
①若干个程序员把自己的作业（卡片或纸带）交给操作员。
②操作员把一批作业装到输入设备上，在监督程序的控制下将卡片或纸带上的信息送到外存上，例如磁带。
③监督程序从外存上将第一个作业调入内存运行，并将计算结果保存到外存上，最后再通过打印设备打印。
④当第一个作业处理完毕后，再处理第二个作业，直到该批作业全部处理完毕。

[图片：联机批处理流程图，展示了程序A/B交给操作员，通过输入设备进入外存，监督程序控制主机进行调入作业和保存结果，最后通过打印机输出]

联机批处理下，作业从输入到打印的整个过程都在 CPU 直接控制下进行。
由于 CPU 运算速度不断提高，CPU 和 I/O 设备之间的速度越来越不匹配，让 CPU 直接控制 I/O 设备已经严重地影响了系统的效率。通过两个简单的程序给大家说明计算操作和 I/O 操作的执行速度差异有多大！

(2) 脱机批处理
脱机指：输入输出操作脱离主机控制。脱机批处理系统由主机和卫星机组成，卫星机又称为外围机，它只与 I/O 设备打交道，即卫星机专门负责输入/输出，主机专门负责计算，工作方式如下。
①用户的作业通过卫星机逐个地传送到输入磁带上形成一批作业。
②输入磁带与主机相连，主机从输入磁带上将用户的作业调入内存运行
③主机将计算结果写到输出磁带上，接着处理下一个作业。
④输出磁带上的计算结果通过卫星机读出来，并交打印机输出。

[图片：脱机批处理系统工作流程图，展示了纸带机、卫星机、外存、主机、打印机之间的数据流向，以及下方的作业执行时间轴（调入作业、计算、保存结果）]

单道批处理系统的主要特征如下（理解并记忆）：
①自动性：外存上的一批作业能自动地逐个运行，无须人工干预。
②顺序性：磁带上的各道作业有序进入内存，先调入内存的作业先完成。
③单道性：内存中仅有一道程序运行，即监督程序每次从外存上只调入一道程序进入内存运行，当该程序完成或发生异常情况时，才换入后继程序进入内存。
由于内存中仅存放一道程序，在该程序运行期间，如果执行 I/O 指令，发出输入/输出请求，CPU 就得暂停执行，直到低速的 I/O 设备完成工作后继续执行后续指令。
【例】在单道批处理的环境下，作业的执行是一个接一个，先执行甲，然后再执行乙。

**2.多道批处理系统（理解并记忆）**

为了进一步提高资源的利用率和系统的吞吐量，引入了多道程序技术。**多道，即允许在内存中同时存放多个作业，CPU以切换的方式为它们服务。**

【例】假设开始时作业甲在CPU上运行，后来它需要进行输入输出操作，因此CPU转而执行另一个作业乙。此时各个作业交替地使用CPU和输入输出设备。

[图片：系统资源分配示意图。左侧显示(1)甲使用CPU；右侧显示(2)乙使用CPU，甲使用I/O设备]

在多道批处理系统中，**作业甲和作业乙可以轮流使用CPU和输入输出设备。**当一个作业在使用CPU时，另一个作业可以去使用输入输出设备，这样就充分提高了设备的利用效率。

[图片：多道批处理系统下的CPU和I/O时间轴，显示甲、乙作业交替并行使用CPU和I/O设备]

可以看出，在多道批处理系统中，这两个作业完成的总时间一般要少于单道批处理。因此，与单道程序系统相比，**多道程序系统下，CPU利用率高，I/O设备利用率高，系统吞吐量大！**

另外，为了解决多道程序下的程序切换问题，需要引入**中断技术**。**中断**指CPU暂停正在执行的程序，转去执行处理中断事件的中断服务程序，待执行完中断服务程序再返回到原被中断的程序继续执行。

多道程序设计具有并发性的特点，即某段时间内看上去有多个程序都在执行，但某个时刻 CPU 只能执行某个程序。与之对应的特性是并行性，例如具有多处理器的计算机具有并行性，此时多个 CPU 上同时执行多个程序。

（三）分时操作系统（理解并记忆）
多道批处理系统提高了系统资源的使用效率，但它也有一个缺点，它没有从用户的角度出发，解决用户对响应时间的要求。

【例】假设在一个多道批处理系统中，用户提交了一个短作业乙，乙仅需较短的CPU执行时间就能得到结果。然而，若乙不巧被卡在长作业甲后面，甲不需要I/O操作，但需要较长的CPU使用时间，那就需要较长时间后才能轮到乙。（假设各个作业不需要进行I/O操作）

[图片：多道批处理时间轴，甲长乙短]

在生活当中也有这样的例子，比如去银行存钱或取钱时，一般金额很小，真正办理起来可能只要一两分钟的时间。但是在排队的时候，如果排在前面的正好是一个大客户，需要存一大笔钱，那就麻烦了，这就会使我们的等待时间变得极其漫长，即响应时间很长！

分时技术是指将处理器的运行时间分成很短的时间片，按时间片轮流将处理器分配给各作业使用。

[图片：分时时间轴，甲乙交替短片段]

若某个作业在分配给它的时间片内不能完成其计算，则该作业暂时停止运行，将处理器让给其他作业使用，等待下一轮再继续运行。由于计算机速度很快，作业运行轮转得也很快，因此给每个用户的感觉就像是自己独占一台计算机。分时系统下，当用户在终端上键入命令时，系统应能及时接收并处理该命令，再将结果返回用户。

```bash
leon@leon-virtual-machine: ~/OS
leon@leon-virtual-machine:~/OS$ ls
hello  hello.c  IOTime  IOTime.c  noIOTime  noIOTime.c
leon@leon-virtual-machine:~/OS$ ./hello
hello world!
leon@leon-virtual-machine:~/OS$ 
```

分时系统主要特征如下：
①多路性：指系统按分时原则为多个用户服务。
②独立性：指多个用户彼此之间互不干扰，给用户的感觉就像是他一人独占主机进行操作。
③及时性：指用户的请求能在很短时间内获得响应。
④交互性：指用户可通过终端与系统进行对话，如执行某个程序等。

（四）实时操作系统（理解即可）
虽然分时操作系统较好地解决了人机交互问题，但在一些应用场合，需要系统能对外部的信息在规定的时间（比时间片的时间还短）内做出处理，比如飞机订票系统。因此，实时操作系统应运而生。
实时操作系统的主要特征有两个：一是实时性，要求系统对外部请求在严格的时间期限内做出反应；二是可靠性，要求系统高度可靠。典型的例子包括火箭的发射、核电站的控制与监测等。

【补充】按体系结构划分的操作系统（了解即可）
按体系结构来分，操作系统可分为网络操作系统、分布式操作系统、嵌入式操作系统等。

1.网络操作系统
20世纪90年代以来，随着计算机技术和通信技术的发展和融合，计算机网络得到迅猛地发展，使得在一台计算机上可以使用其他机器上的资源或与其他计算机进行通信，计算机的发展进入分布式计算阶段。多台计算机可以同时为一个用户提供服务，用户感觉就像使用一台计算机一样。
[图片：网络操作系统架构示意图，包含计算机1、计算机2、计算机3，每个计算机包含CPU和主存储器，通过云状网络连接]
为了对计算机网络进行有效地管理，在操作系统中应增加网络管理功能，因此，网络操作系统应运而生。网络操作系统是按照网络体系结构的各种协议（计网中深入学习）来完成网络的通信、资源共享、网络管理和安全管理的系统软件。

2.分布式操作系统
分布式系统（distributed system），是多个处理机通过通信线路互联而构成的系统，系统的处理和控制功能分布在各个处理机上。与传统的多处理机系统（例如多处理机）相比，分布式系统的不同之处在于：
①分布式系统中的每个节点都是一台独立的计算机，并配置有完整的外部设备。
②分布式系统中节点的耦合程度更低，地理分布区域更加广阔。
③分布式系统中的每个节点均可以运行不同的OS，每个节点均拥有自己的文件系统。
分布式OS是配置在分布式系统上的公用OS，其对分布式系统中的所有资源进行统一管理，可以直接对系统中地理位置分散的各种物理和逻辑资源进行动态分配和调度，有效地协调和控制各个任务的并行执行，并向用户提供一个统一的、方便的、透明的使用系统界面和标准接口。一个典型的例子是万维网（world wide web，WWW），在万维网中，所有的操作只通过一种界面（即网页）进行。

3.嵌入式操作系统
与通用的计算机（如便携式计算机或桌面系统等）不同，嵌入式系统是为了完成某个特定功能而设计的系统，由硬件和软件组成，是能够独立运作的器件，例如智能音响。嵌入式OS是指应用于嵌入式系统的OS，数量远超过普通的OS，而且应用非常广泛。

【补充】个人计算机操作系统（了解即可）
个人计算机操作系统是目前使用最广泛的操作系统，也被称为微机操作系统，它广泛应用于文字处理、电子表格、游戏中，常见的有Windows、Linux和MacOs等。

【补充】两种实时系统（了解即可）
1.硬实时系统
若某个动作必须绝对地在规定的时刻（或规定的时间范围）发生，则称为硬实时系统，如飞行器的飞行自动控制系统。
2.软实时系统
若能够接受偶尔违反时间规定且不会引起任何永久性的损害，则称为软实时系统，如飞机订票系统、银行系统。
