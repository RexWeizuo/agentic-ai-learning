# 4.1 I/O 管理的基本概念

计算机最核心的硬件是 CPU 和内存。但光有这两个硬件是不行的——用户根本无法直接与 CPU 和内存打交道。为了解决这个问题，需要引入输入/输出（Input/Output, I/O）设备，用户正是通过这些 I/O 设备来使用计算机。

在一个现代计算机系统中，存在着大量的 I/O 设备，它们种类繁多，差异很大。如何管理这些设备，使得各种资源能够得到充分、合理地利用？这是操作系统的一项重要任务。

> 本小节中的许多内容与计组第七章内容重合，可以根据个人情况选学。

## 一、I/O 设备和设备控制器（理解即可）

I/O 设备一般是由执行 I/O 操作的**机械部分**和执行 I/O 控制的**电子部件**组成的。

通常会将这两部分分开，执行 I/O 操作的机械部分就是一般的 I/O 设备，而执行 I/O 控制的电子部件则称为**设备控制器**或**适配器**。微机和小型计算机中的控制器常被称为控制卡、接口卡或网卡。

### （一）I/O 设备的类型（了解即可）

#### 1. 按信息交换的单位分类

| 类型 | 单位 | 可寻址 | 效率 | 示例 |
|------|------|--------|------|------|
| 字符设备 | 字符 | 不可寻址 | 较低 | 鼠标、打印机 |
| 块设备 | 数据块 | 可寻址 | 高 | 硬盘、光盘 |

#### 2. 按设备的共享属性分类

| 类型 | 特点 | 示例 |
|------|------|------|
| 独占设备 | 分配给某进程后由该进程独占，直至用完释放 | 打印机 |
| 共享设备 | 同一时间段内允许多个进程通过分时方式共享使用 | 磁盘 |

#### 3. 按使用特性分类

| 类型 | 说明 | 示例 |
|------|------|------|
| 存储设备 | 外存/辅存，速度较内存慢但容量大、价格便宜 | 硬盘 |
| 人机交互设备 | 用户与计算机直接交互 | 键盘、显示器 |
| 网络通信设备 | 计算机之间的通信 | 网卡 |

#### 4. 按传输速率分类

| 类型 | 速率 | 典型设备 |
|------|------|----------|
| 低速设备 | 每秒几字节至数百字节 | 键盘、鼠标 |
| 中速设备 | 每秒数千字节至数十万字节 | 打印机 |
| 高速设备 | 每秒数十万字节至千兆字节 | 磁盘机、光盘机 |

**练习 · 设备属性判断**

> 下列关于设备属性的叙述中，正确的是（ ）。
> A. 字符设备的基本特征是可寻址到字节，即能指定读/写操作的字节地址
> B. 共享设备必须是可寻址的和可随机访问的设备
> C. 共享设备是指同一时刻内允许多个进程同时访问的设备
> D. 在分配共享设备和独占设备时都可能引起进程死锁

答案：**B**

- A 错：字符设备不可寻址
- B 对：共享设备（如磁盘）必须可寻址、可随机访问，才能在不同进程间分时共享
- C 错：共享设备是"同一时间段内"允许访问，不是"同一时刻"
- D 错：共享设备不会引起死锁（可动态分配），独占设备才可能死锁

### （二）设备控制器（理解即可）

I/O 接口又称设备控制器、I/O 控制器。不同的外设往往对应不同的设备控制器，例如磁盘控制器、以太网卡（网络控制器）、USB 控制器、声卡等。

#### 1. I/O 接口的基本结构

I/O 接口是主机（CPU 和主存）和 I/O 设备之间进行信息传输的"桥梁"，在主机侧和外设侧各提供了一个接口：

- **内部接口**（主机侧）：通过系统 I/O 总线和 CPU、内存相连
- **外部接口**（外设侧）：通过 USB 等接口电缆与外设相连

一个 I/O 接口可以通过多个外部接口连接多个设备。

**I/O 端口**（CPU 可直接访问的寄存器）：

| 端口类型 | 寄存器 | 功能 |
|----------|--------|------|
| 数据端口 | 数据缓冲寄存器 | 暂存外设与 CPU/内存之间传送的数据 |
| 状态端口 | 状态寄存器 | 记录接口和设备的状态信息，CPU 读取 |
| 控制端口 | 控制寄存器 | 保存 CPU 对外设的控制信息，CPU 写入 |

> 状态寄存器和控制寄存器在传送方向上是相反的，由于使用时间错开，很多时候合并为状态/控制寄存器。

**I/O 控制逻辑的三项功能**：
1. 将控制寄存器中的命令字译码成控制信号送到外设
2. 在数据缓冲寄存器和外设之间发送或接收数据
3. 将外设状态收集到状态寄存器中

#### 2. I/O 接口的基本功能（了解即可）

| 功能 | 说明 |
|------|------|
| 数据缓冲 | 进行速度匹配，缓和高速主机与低速外设之间的差异 |
| 数据格式转换 | 串-并转换、并-串转换、数-模转换、模-数转换等 |
| 控制和定时 | 协调内部资源及外设间动作的先后关系 |
| 错误或状态检测 | 提供状态寄存器供 CPU 读取出错情况 |
| 与主机和设备通信 | 上述四个功能都通过此通信完成 |

## 二、I/O 端口及其编址（理解并记忆）

CPU 通过端口来发送命令、读取状态和传送数据。为了使 CPU 访问到 I/O 端口，必须为 I/O 端口编址，编址方式有**独立编址**和**统一编址**两种。

### （一）统一编址（内存映射编址，Memory-mapped I/O）

I/O 设备地址与内存地址统一编址，二者在逻辑上处于同一个地址空间，通过不同的地址区域来区分是访问内存还是 I/O 设备。

统一编址不需要设置专用的 I/O 指令，采用 Load/Store 访存指令就可以访问 I/O 设备，具体访问什么设备取决于地址。

```
lw $t0, 0x00000004    # 从 ROM 读取一个字
sb $t0, 0xff000004    # 写一个字节到内存
sb $t0, 0xffff0004    # 写一个字节到 I/O 设备
```

**优点**：
1. 不需要专门的 I/O 指令，访问灵活方便；I/O 寄存器数目只受寻址空间制约，有利于提高系统吞吐率
2. 任何访存指令都可用于 I/O 端口，意味着访存指令的寻址方式和保护机制（分段/分页）都可以被共享，不需要专门的保护机制

**缺点**：
1. I/O 端口占用了部分主存空间地址，减少了主存可用容量
2. 访问 I/O 端口时全部地址线都参与地址译码，译码复杂、寻址速度降低

### （二）独立编址（I/O 映射方式，I/O-mapped I/O）

主存地址空间和 I/O 端口地址空间分别编址，地址取值范围可以重叠。相同地址可能属于不同地址空间，需要用**专门的 I/O 指令**来表明访问的是 I/O 地址空间。

若 CPU 识别出当前指令是 I/O 指令，便会向 I/O 总线发送相应的控制信号和地址信息；若访问的是内存单元，则直接通过主存读写控制线控制数据传输。

> Intel x86 处理器普遍采用独立编址：主存地址空间 00000000H~FFFFFFFFH（4GB），I/O 地址空间 0000H~FFFFH（64KB）。两部分地址区间重叠，用 MOV 指令访问内存，用 IN/OUT 指令访问 I/O。

```
OUT DX, AL      # I/O 写：AL → DX 对应的 I/O 端口
IN  AL, DX      # I/O 读：DX 对应的 I/O 端口 → AL
MOV [BX], AL    # 内存写：AL → BX 对应的内存地址
```

**优点**：
1. I/O 端口数明显少于主存单元，I/O 接口只需少量地址线，译码简单，寻址速度快
2. 使用专用 I/O 指令，和访存指令完全区分，程序更清晰

**缺点**：
1. I/O 指令通常只支持简单的传输操作，程序设计灵活性降低；CPU 必须分别为存储器和 I/O 提供单独的读写命令，总线控制逻辑更复杂
2. 需要专门的硬件保护机制

### 统一编址 vs 独立编址 对比

| 维度 | 统一编址 | 独立编址 |
|------|----------|----------|
| 地址空间 | I/O 和内存共用 | I/O 和内存独立（可重叠） |
| 指令 | Load/Store 访存指令 | 专用 I/O 指令（IN/OUT） |
| 保护机制 | 复用分段/分页保护 | 需专门硬件保护 |
| 寻址速度 | 较慢（全部地址线译码） | 较快（少量地址线译码） |
| 灵活性 | 高（任何访存指令可用） | 低（I/O 指令功能有限） |
| 内存容量 | 被 I/O 占用部分地址 | 不受影响 |
| 典型代表 | — | Intel x86 |

## 三、I/O 通道（了解即可）

虽然在 CPU 与 I/O 设备之间增加了设备控制器后能大大减少 CPU 的干预，但当主机配置的 I/O 设备很多时，CPU 的负担仍然很重。

为此，在 CPU 和设备控制器之间又增设了**通道（Channel）**，主要目的是**建立独立的 I/O 操作**——不仅使数据的传送能独立于 CPU，而且使对 I/O 操作的组织、管理及其结束处理尽量独立，以保证 CPU 有更多时间进行数据处理。

**工作流程**：
1. CPU 向通道发送一条 I/O 指令
2. 通道从内存中取出本次要执行的**通道程序**并执行
3. 仅当通道完成了规定的 I/O 任务后，才向 CPU 发送中断信号

**I/O 通道是一种特殊的处理机**：具有执行 I/O 指令的能力，能通过执行通道程序控制 I/O 操作。但与一般处理机的区别：

| 区别点 | I/O 通道 |
|--------|---------|
| 指令类型 | 单一，主要局限于 I/O 操作相关指令 |
| 内存 | 没有自己的内存，通道程序放在主机内存中（与 CPU 共享） |
