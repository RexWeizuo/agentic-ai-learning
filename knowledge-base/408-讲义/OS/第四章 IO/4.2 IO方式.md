# 4.2 I/O 方式

CPU 与外部设备之间的信息交换随外部设备性质的不同而采用不同的控制方式，即不同的 I/O 方式。408 大纲中的 I/O 方式有**程序查询**、**程序中断**和 **DMA 方式**三种。

> 本小节内容与计组第七章内容重合，可根据个人情况选学。

---

## 一、程序查询方式（理解并记忆）

### （一）基本思想

信息交换的控制**完全由 CPU 执行程序实现**。主机进行 I/O 操作时，先读取设备状态，根据状态决定是进行数据传送还是等待。

**流程**：
1. CPU 执行初始化程序，预置传送参数（地址、计数器）
2. 向 I/O 接口发出命令字，启动 I/O 设备
3. 从外设接口读取状态信息
4. CPU 周期/持续查询设备状态，直到外设准备就绪
5. 传送一次数据
6. 修改地址和计数器参数
7. 判断传送是否结束，未结束转第 3 步

**键盘输入示例**（x86，独立编址）：

| 设备 | 寄存器 | 端口地址 | 备注 |
|------|--------|----------|------|
| 键盘 | DBR（数据缓冲寄存器） | 0x0004 | |
| 键盘 | DSR（设备状态寄存器） | 0x0000 | 最低位为 Ready 位 |
| 字符显示 | DBR | 0x000C | |
| 字符显示 | DSR | 0x0008 | 最低位为 Ready 位 |

```assembly
MOV DX, 0x0000    ; 载入键盘 DSR 端口地址
keyPoll:
IN  AL, DX        ; 读取键盘 DSR 的值
TEST AL, 1        ; 测试就绪位 Ready
JZ  keyPoll       ; Ready==0 则继续查询
MOV DX, 0x0004    ; 载入键盘 DBR 端口地址
IN  AL, DX        ; 读取键盘 DBR 中的字符
```

**优点**：设计简单、硬件量少。
**缺点**：CPU 花费大量时间查询和等待。

### （二）两种查询方式

#### 1. 独占查询

一旦设备启动，CPU **一直持续查询**接口状态，花费 100% 时间用于 I/O，外设和 CPU 完全**串行工作**。

> 若题目未明确查询方式，默认采用独占查询。

#### 2. 定时查询

不需要反复查询：
1. CPU 启动设备后，启动一个定时中断
2. 切换到程序 P2 运行
3. 定时时间到，发生中断 → 执行中断服务程序查询设备状态
   - 设备准备好 → 唤醒 P1，切换回 P1 传输数据
   - 未准备好 → 继续定时查询

定时查询避免了轮询等待浪费 CPU，但中断服务本身要占 CPU 时间。

**计算示例**：主频 500MHz，CPI=4，外设数据率 2MB/s，I/O 接口 32 位数据缓冲寄存器，采用定时查询，每次 I/O 执行 10 条指令。外设最多间隔多长时间查询一次？CPU 用于外设 I/O 的时间占比？

- 外设提供 32 位数据的时间间隔：4B ÷ 2MB/s = **2μs**（最多每隔 2μs 查询一次）
- 每秒查询次数：1s ÷ 2μs = 5×10⁵ 次
- 每秒 CPU 用于 I/O 的时钟周期：5×10⁵ × 10 × 4 = 2×10⁷
- 占 CPU 总时间：2×10⁷ ÷ 500M = **4%**

**2011 真题**：主频 50MHz，定时查询一次至少 500 时钟周期，每秒需查询至少 200 次，CPU 用于 I/O 的时间占比？

- 每秒 I/O 时钟周期：200 × 500 = 10⁵
- 占比：10⁵ ÷ 50M = **0.20%**（答案 C）

## 二、程序中断方式（理解并记忆）

### 基本思想

1. CPU 启动外设后，**继续执行程序**，不主动查询外设状态
2. 外设完成数据准备后，**主动向 CPU 发中断请求**
3. CPU 响应中断 → 暂停当前程序 → 执行中断服务程序完成一次数据传送
4. 传送完成 → CPU 返回原程序，外设和 CPU 继续并行工作

**计算示例**：主频 500MHz，CPI=4，外设数据率 40MB/s，I/O 接口 32 位数据缓冲寄存器，每次中断响应+处理至少 400 时钟周期，能否采用中断 I/O 方式？

- 中断响应处理时间：400 × (1/500M) = **0.8μs**
- 外设提供 32 位数据的时间间隔：4B ÷ 40MB/s = **0.1μs**
- 0.1μs < 0.8μs → 数据来不及处理就被覆盖 → **不适合**

**2014 真题**：中断响应和处理时间 100ns，每 400ns 一次中断请求，最长延迟 50ns，CPU 用于该设备的 I/O 时间占比？

- 每次中断占用 100ns，每 400ns 一次
- 占比：100 ÷ 400 = **25%**（答案 B）

## 三、DMA 方式（理解并记忆）

### （一）基本思想

DMA（Direct Memory Access）：外设与内存之间开辟**"直接数据通路"**，信息传送不再经过 CPU，降低 CPU 开销。

**数据传送流程**：

| 阶段 | 操作 | CPU 参与 |
|------|------|----------|
| 1. 预处理 | CPU 初始化 DMA 控制器中的寄存器 | ✅ |
| 2. DMA 请求 | I/O 设备向 DMAC 发 DMA 请求 | ❌ |
| 3. 总线请求 | DMAC 向 CPU 发总线请求（HRQ） | ❌ |
| 4. 总线响应 | CPU 响应，发 HLDA，DMAC 接管总线 | ✅（仅响应） |
| 5. 数据传送 | DMAC 控制数据传送，修改地址和计数 | ❌ |
| 6. 循环 | 重复 2-5 直至一块数据传完 | ❌ |
| 7. DMA 结束 | DMAC 向 CPU 发中断请求 | ❌ |
| 8. 后处理 | CPU 执行中断服务程序做结束处理 | ✅ |

> **HRQ**（Hold Request）：保持请求信号，请求 CPU 暂停对总线的控制。
> **HLDA**（Hold Acknowledge）：保持确认信号，表示总线控制权已被 DMA 接管。

关键结论：**整个数据块的传送过程不需要 CPU 参与**，CPU 只在最初的初始化（预处理）和最后的结束处理（后处理）时介入。适用于磁盘、显卡、声卡、网卡等高速设备大批量数据传送。

**2023 真题**：错误的叙述是？
- C 错：DMA 方式下，数据传送由 **DMA 控制器硬件**完成，不是 CPU 执行 DMA 传送程序。

### （二）DMAC 基本构造（理解即可）

| 部件 | 功能 |
|------|------|
| 主存地址计数器（AR） | 存放传送数据的主存起始地址，每传一个字 +1 |
| 传送长度计数器（WC） | 记录传送数据的总字数，每传一个字 −1，到 0 表示完毕 |
| 数据缓冲寄存器（DBR） | 暂存每次传送的数据 |
| DMA 请求触发器 | I/O 设备准备好数据后置位 |
| 控制/状态逻辑 | 指定传送方向、修改参数、协调信号 |
| 中断机构 | 一批数据传送完毕后向 CPU 发中断请求 |

### （三）DMA 的传送方式（理解即可）

由于 DMAC 和 CPU 共享主存，可能出现争用，DMA 有三种传送方式：

#### 1. 停止 CPU 访存
DMA 请求时，DMA 接口向 CPU 发停止信号，CPU 放弃总线控制权，**直到 DMA 传送完一块数据**。

- 优点：控制简单
- 缺点：CPU 可能较长时间不能访存

#### 2. DMA 与 CPU 交替访存
将存储周期分成两个时间片，一个给 CPU，一个给 DMA，**每个存储周期内轮流访存**。

- 优点：不需要总线控制权的申请/建立/归还
- 缺点：增加存储周期

#### 3. 周期挪用（理解并记忆）⭐

**仅当 DMAC 需要访问内存时**，CPU 才暂停**一个存储周期**供 DMAC 访问，一个数据传送结束后总线控制权交还 CPU。又称**周期窃取**。

> 周期挪用可能发生在指令周期下**任何一个机器周期的结束时刻**。

当 I/O 有 DMA 请求时三种情况：
1. CPU 不在访存 → 无冲突
2. CPU 正在访存 → 等存储周期结束后让出总线
3. I/O 和 CPU 同时请求访存 → **I/O 优先级高于 CPU**（I/O 不立即访存可能丢数据）

周期挪用是 DMA 传送方式的**主要方法**。

**计算示例**：主频 500MHz，CPI=4，外设数据率 40MB/s，32 位数据端口，DMA 块大小 1000B，预处理+后处理总计 500 时钟周期，CPU 用于外设 I/O 的时间占比？

- 每秒 DMA 次数：40MB/s ÷ 1000B = 40000
- CPU 仅在预处理/后处理介入：40000 × 500 = 2×10⁷ 时钟周期
- 占比：2×10⁷ ÷ 500M = **4%**

### （四）DMA 方式和中断方式的区别（理解并记忆）⭐

| | 中断方式 | DMA 方式 |
|---|---|---|
| 数据传送 | CPU 执行程序完成 | 额外硬件完成 |
| CPU 现场 | 需要保护和恢复 | 不改变 CPU 现场 |
| 请求对象 | 请求 CPU 时间 | 请求总线控制权 |
| 响应时机 | 指令执行后 | 任一机器周期结束时刻 |
| 传送过程 | 需 CPU 干预 | 不需 CPU 干预 |
| 异常处理 | 有处理异常事件的能力 | 仅限大批数据传送 |
| 适用设备 | 中低速设备 | 高速设备 |

**练习**：错误的叙述是？（答案 B：II、III、IV 全错）
- I ✅：中断过程由硬件 + 中断服务程序共同完成
- II ❌：中断请求在**每条指令执行后**检查，不是每个总线周期
- III ❌：DMA 请求检测在**每个机器周期结束**，不是每条指令末尾
- IV ❌：中断服务程序最后是**中断返回指令**（如 IRET），不是无条件转移

**练习**：DMA 传送方式中，由（外部设备）发出 DMA 请求，传送期间总线控制权由（DMA 控制器）掌握。答案 C。
