# 快表与多级页表

## 快表与多级页表

### （四）具有快表（TLB）的地址变换

页表存放在内存中，CPU 每次存取数据需**访问内存两次**：

1. 访问页表，找到物理块号
2. 访问物理地址获取数据

这使处理速度降低近 50%。

**快表（TLB，Translation Lookaside Buffer）：**  高速缓冲寄存器，存放最近使用的页表项。

**引入 TLB 后的地址变换：**

1. CPU 给出逻辑地址，将页号送入 TLB
2. TLB 命中 → 直接得到物理块号
3. TLB 未命中 → 访问内存中的页表，并更新 TLB

**EAT 计算（有效访存时间）：**

【例】快表访问 2μs，页表访问 500μs，命中率 90%：

- 串行访问：$(2+500) \times 0.9 + (2+500+500) \times 0.1 = 552\mu s$
- 并行访问：$(2+500) \times 0.9 + (500+500) \times 0.1 = 551.8\mu s$

### （五）两级页表和多级页表

**问题：**  32 位逻辑地址、页面 4KB → 页号 20 位 → $2^{20}$ 个页。每个页表项 4B → 页表 4MB → 需要 1K 个**连续**页框存放。

**解决：两级页表**

- 将页表本身再分页，用**页目录表（一级页表）** 索引
- 只调入当前需要的二级页表，其余留在磁盘

**32 位地址的两级划分：**   
| 页目录号 (10位) | 页号 (10位) | 页内偏移 (12位) |

 **【练习】**  虚拟地址 20501225H，页目录号、页号分别是？

> 20501225H = 0010 0000 0101 0000 0001 0010 0010 0101  
> 页目录号(高10位) = 081H，页号(中10位) = 401H → **B**

**多级页表：**  若逻辑地址 40 位，页面 4KB，每个页表项 4B：  
| 页目录号 (18位) | 页号 (10位) | 页内偏移 (12位) |

页目录表有 256K 个页表项，需 256 个连续页框 → 继续分级 → **三级页表**
