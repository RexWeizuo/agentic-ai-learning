# 分页存储管理基本原理

## 分页存储管理基本原理

### 从连续到离散

连续分配方式会形成许多"碎片"，虽然可以通过"紧凑"方法拼接，但开销很大。如果允许将一个进程**分散地装入**许多不相邻接的分区中，则可充分利用内存空间。

离散分配方式分为：**分页存储管理、分段存储管理、段页式存储管理**。

---

### 一、分页存储管理

#### （一）基本原理

将内存空间分为若干个**物理块（页框）** 。把进程的地址空间分为若干个固定大小的**页（页面）** 。

物理地址 = 页框号 + 页内地址  
逻辑地址 = 页号 + 页内地址

 **【例】**  页面大小 1KB，逻辑地址 3BADH：

- 1KB = $2^{10}$B，低 10 位为页内偏移，高 6 位为页号
- 页号 = 0EH，页内偏移 = 3ADH

给定逻辑地址 A，页面大小 L：

- 页号 $P = \lfloor A / L \rfloor$
- 页内地址 $d = A \bmod L$

**类比：**  类似于计组中 Cache 和主存的映射。区别：主存-Cache 交换单位是**块**（32B<sub>64B），主存-外存交换单位是</sub>​<sub>**页**</sub>​<sub>（2KB</sub>4KB），一个页包含上百个块。

---

#### （二）页表

系统为每个进程建立一张**页表**，记录逻辑页号 → 物理页号的映射。

页表本质是一个数组：下标 = 逻辑页号，值 = 物理页号。每个页表项还可以包含存取控制字段。

---

#### （三）地址映射

**分页地址变换过程：**

1. 逻辑地址分为页号 + 页内地址
2. 用页号查页表，得到物理块号
3. 物理地址 = 物理块号 + 页内地址

页表的起始地址和长度存放在进程的 **PCB** 中。进程被调度时，装入**页表寄存器**。单处理机只需一个页表寄存器。

 **【练习】**  页大小 4KB，页表：| 页号 | 块号 | | 0 | 2 | | 1 | 1 | | 2 | 6 | | 3 | 3 | | 4 | 7 |，逻辑地址 0 转换成的物理地址为？

> 逻辑地址 0 = 页号 0 + 偏移 0 → 块号 2 → 物理地址 2 × 4KB = **8192**
