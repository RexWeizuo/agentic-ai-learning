# 3.3所有寄存器汇总

## 文档中所有寄存器整理

文档中出现的寄存器分布在三种存储管理方式中：

---

### 一、分页存储管理

#### 1. 页表寄存器（PTR）

文档原文：

> "在系统中只设置一个页表寄存器（page-table register，PTR），在其中存放**页表（在内存中）的起始地址和页表长度**。"

```
┌─────────────────────────────────┐
│        页表寄存器 (PTR)          │
├──────────────┬──────────────────┤
│ 页表起始地址  │   页表长度       │
│   (物理地址)  │  (页表项个数)    │
└──────────────┴──────────────────┘
```

|属性|说明|
| ----------| ------------------------------------------------|
|存放内容|页表在内存中的起始物理地址 + 页表长度|
|所属层级|一级页表（基本分页）|
|生命周期|进程切换时从 PCB 加载，单 CPU 只需一个|
|作用|定位页表在内存中的位置，配合页号计算页表项地址|

#### 2. 有效地址寄存器 & 物理地址寄存器

文档原文（地址变换流程）：

> "将有效地址寄存器中的页内地址送入物理地址寄存器的块内地址字段中"

|寄存器|存放内容|作用|
| --------| --------------------| --------------------|
|**有效地址寄存器**|CPU 产生的逻辑地址|等待被拆分和转换|
|**物理地址寄存器**|转换完成的物理地址|送上地址总线去访存|

#### 3. 快表 / 联想寄存器（TLB）

文档原文：

> "增设一个具有并行查寻能力的高速缓冲寄存器，称为'联想寄存器'或'快表'"

|属性|说明|
| ----------| --------------------------------------|
|存放内容|最近访问的页表项（页号→物理块号）|
|本质|一组高速缓存寄存器，用于加速地址转换|
|命中时|跳过访问内存中的页表|

---

### 二、两级页表 — 页目录表寄存器

文档原文（一级页表 = 页目录表）：

> "为了方便实现地址变换，需要增设一个**页目录表寄存器**，用于存放页目录表的起始地址。"

```
页目录表寄存器
┌─────────────────────┐
│ 页目录表起始地址     │  ← 指向一级页表（页目录表）
│   (物理地址)        │
└─────────────────────┘
```

|属性|说明|
| ----------| ------------------------------------|
|存放内容|一级页表（页目录表）在内存中的起始**物理地址**|
|所属层级|**二级页表系统**|
|等同于|二级页表系统中的"页表基址寄存器"|

---

### 三、分段存储管理 — 段表寄存器

文档原文：

> "在系统中设置了一个**段表寄存器**，用于存放段表始址 F 和段表长度 M。"

```
段表寄存器
┌──────────────┬──────────┐
│ 段表始址 F   │段表长度 M │
│  (物理地址)  │          │
└──────────────┴──────────┘
```

|属性|说明|
| ----------| ---------------------------------------|
|存放内容|段表在内存中的起始物理地址 + 段表长度|
|越界检查|段号 ≥ M → 越界中断|

---

### 四、段页式存储管理 — 段表寄存器

文档原文：

> "系统中还应有一个段表寄存器，指出进程的**段表始址和段表长度**。"

```
段表寄存器
┌──────────────┬──────────┐
│ 段表始址     │段表长度   │
│  (物理地址)  │          │
└──────────────┴──────────┘
       ↓
    段表 → 页表 → 物理块
```

本质与分段管理的段表寄存器功能一致，只是段表项中不再直接存段基址，而是存**页表始址**。

---

### 五、汇总对比

|寄存器名称|出现位置|存放内容|指向什么|
| ------------| -------------| --------------------------| --------------------|
|**页表寄存器 (PTR)**|一级分页|页表始址 + 长度|页表|
|**页目录表寄存器**|两级分页|页目录表（一级页表）始址|页目录表|
|**段表寄存器**|分段管理|段表始址 + 长度|段表|
|**段表寄存器**|段页式管理|段表始址 + 长度|段表（再指向页表）|
|**有效地址寄存器**|分页 / 分段|当前逻辑地址|—|
|**物理地址寄存器**|分页 / 分段|转换后的物理地址|—|
|**快表 (TLB)**|分页|最近页表项缓存|—|

## 第12题解析

### 题目

> 在采用二级页表的分页系统中，CPU 页表基址寄存器中的内容是（ ）。

### 关键概念

这里"页表基址寄存器"在二级页表系统中，**就是文档中的"页目录表寄存器"** ：

- 二级页表中，一级页表 = **页目录表**
- 页表基址寄存器指向的是一级页表（页目录表）的起始地址
- 页表存储在**物理内存**中，所以存放的必然是**物理地址**

### 逐项排除

|选项|判断|理由|
| ---------------------------| :----: | ----------------------------------------------------|
|A. 一级页表的起始虚拟地址|❌|寄存器指向内存中的实际位置，必须用物理地址才能访存|
|**B. 一级页表的起始物理地址**|✅|页目录表在物理内存中，寄存器存其物理首地址|
|C. 二级页表的起始虚拟地址|❌|寄存器指向一级页表，不是二级页表|
|D. 二级页表的起始物理地址|❌|二级页表的地址存在一级页表的表项中，不直接放寄存器|

### 答案：B

### 为什么必须是物理地址？

地址转换的**第一步**就是读一级页表。如果寄存器里存的是虚拟地址，就陷入死循环——要转换这个虚拟地址就需要查页表，但查页表又需要物理地址……所以页表基址寄存器必须存**物理地址**，这是硬件能直接用来访问内存的地址。
