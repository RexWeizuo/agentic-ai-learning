# 段页式存储管理

## 段页式存储管理

### （一）基本原理

**分页**能有效提高内存利用率，**分段**能反映程序逻辑结构。结合两者 → **段页式存储管理**。

进程地址空间先分成若干逻辑**段**，每段再分成若干固定大小的**页**。

**逻辑地址三部分：**  | 段号 | 页号 | 页内偏移 |

**数据结构：**

- 每个进程一张**段表**（段表项：页表长度 + 页表始址）
- 每段一张**页表**（页表项：物理块号）
- 一个**段表寄存器**（段表始址 + 段表长度）

---

### （二）地址变换

1. 比较段号 S 与段表长度 → 越界检查
2. 用段表始址 + 段号 S → 得到页表始址
3. 用页号 P + 页表始址 → 得到物理块号 b
4. 物理地址 = b + 页内偏移

**代价：**  需**三次访问内存**

1. 访问段表 → 得页表始址
2. 访问页表 → 得物理块号
3. 访问物理地址 → 获取数据

**优化：**  增设高速缓冲寄存器，同时用段号+页号检索，命中则一次访问即可。
