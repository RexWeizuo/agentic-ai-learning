# 补充：物理页面表、覆盖与交换、反置页表

## 补充内容

### 物理页面表（位示图）

描述内存各物理页面的分配使用状况。

【例】内存 256 个物理页面，用字长 32 位的 8 个字构成位示图。每位对应一个物理页面：0=空闲，1=已分配。

**分配过程：**

1. 计算进程所需页面数 N
2. 查位示图中是否有 N 个空闲页
3. 若有，分配并标记为 1

---

### 覆盖技术与交换技术

**覆盖技术：**  程序太大无法全部装入时，不相互调用的模块共用同一内存分区。

- 常驻区：常用功能代码和数据
- 覆盖区：不同模块按需覆盖装入
- **发生在同一进程内部**，需程序员给出模块间的逻辑覆盖结构

**交换技术：**  将暂时不能运行的进程整体换出到外存（swap），空出内存给其他进程。

- **发生在进程之间**，以进程为单位，不需要用户干预

||覆盖|交换|
| ----------| ------------------| ------------|
|单位|进程内部模块|整个进程|
|用户参与|需要给出覆盖结构|不需要|
|透明性|不透明|对用户透明|

### 反置页表

**问题：**  64 位逻辑地址 + 4KB 页面 → $2^{52}$ 个页表项，不可能实现。

**反置页表：**  以**物理页面号**为索引（而非逻辑页号），系统只有一张页表，所有进程共享。

每个页表项记录：(pid, 逻辑页号) → 对应物理页面号。

**查找：**  根据 (pid, p) 到反置页表中顺序搜索，找到匹配项即得物理页面号 i。

**缺点：**  顺序查找很慢。需要用哈希表加速。

---

### 分段存储管理方式的特点

1. **方便编程：**  按逻辑段编写
2. **信息共享：**  以段为单位共享
3. **信息保护：**  按段设置保护权限
4. **动态增长：**  段可根据需要动态扩大
5. **动态链接：**  运行时按需链接各段
