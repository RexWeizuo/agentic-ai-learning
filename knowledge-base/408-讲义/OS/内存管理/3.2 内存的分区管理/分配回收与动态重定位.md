# 分配回收与动态重定位

## 分配回收与动态重定位

### 分区的分配与回收操作

**1. 分配内存**

系统利用某种分配算法，从空闲分区链表中找到所需大小的分区，将其分配给进程。

**2. 回收内存**

当进程运行完毕释放内存时，系统回收该区域。根据回收分区相邻区域的情况，可能需要**合并**相邻空闲分区。

---

### 四、动态重定位分区分配

当内存被分割成许多不邻接的小分区时，虽然总容量足够但因不连续而无法装入大作业。

**紧凑（紧缩/拼接）：**  通过移动内存中作业的位置，把分散的多个小分区拼接成一个大分区。

[图示：移动前 4 个小分区分散（10KB、30KB、14KB、26KB），移动后拼接成 80KB 大空闲区]

**问题：**

1. 需要移动所有进程，消耗大量 CPU 时间
2. 进程被移动后，需更新地址引用（需硬件支持动态重定位）

---

### 连续存储管理方式总结

|方式|概述|优点|缺点|
| ------------| --------------------------| ----------------------| --------------------------|
|单一连续|用户区整体分配给一个进程|无外部碎片，简单安全|有内部碎片，效率低|
|固定分区|初始化时划分静态分区|无外部碎片|有内部碎片，分区数固定|
|动态分区|按需动态创建分区|无内部碎片|有外部碎片，需紧凑|
|动态重定位|通过紧凑消除外部碎片|可拼接大空闲区|需要硬件支持，CPU 开销大|

‍

## 动态重定位 ≠ 装入时地址转换

动态重定位**不是**装入时（load time）进行地址转换，那是**静态重定位**。动态重定位是在**运行时（runtime）** 由硬件动态完成的地址映射。

---

### 三种地址转换时机对比

|类型|转换时机|能否移动|硬件支持|
| ------| ----------------| -------------------| ------------------|
|**静态重定位**|**装入时**一次性转换|❌ 装入后不能移动|不需要|
|**动态重定位**|**运行时**每次访存都转换|✅ 可以移动|需要重定位寄存器|

---

### 文档中的例子

文档提到：进程被移动（紧凑）后，需更新地址引用，必须有**硬件支持的动态重定位**。

```
移动前（内存分散）：
┌────────┐  ┌────────┐  ┌────────┐  ┌────────┐
│ 进程A   │  │ 进程B  │  │ 进程C  │  │ 空闲区 │
│ 10KB   │  │ 30KB   │  │ 14KB   │  │ 26KB   │
└────────┘  └────────┘  └────────┘  └────────┘

紧凑后（拼接成80KB大空闲区）：
┌────────┬────────┬────────┬──────────────────┐
│ 进程A   │ 进程B  │ 进程C  │    80KB 空闲区    │
│ 0~10K  │10~40K  │40~54K  │                  │
└────────┴────────┴────────┴──────────────────┘
```

关键问题：进程B原来在某个地址范围，紧凑后被搬到了 `10K~40K`。如果程序里写死了物理地址，搬移后指令全部失效。

---

### 动态重定位如何解决？

硬件用**重定位寄存器（基址寄存器）** 做运行时映射：

```
每次访存时：物理地址 = 逻辑地址 + 基址寄存器值
```

**举例：**  进程B的某条指令要访问逻辑地址 `500`

```
紧凑前：
  基址寄存器 = 50,000（进程B原来起始地址）
  物理地址 = 500 + 50,000 = 50,500

紧凑后：
  操作系统更新基址寄存器 = 10,240（进程B搬到的新起始地址）
  物理地址 = 500 + 10,240 = 10,740  ← 自动指向新位置！
```

程序内部的逻辑地址 `500`​ 完全不用改，只改基址寄存器一个值，所有地址引用自动修正。这就是**动态**的含义——地址转换在每次访存时发生，而不是装入时一次性算完。

---

### 对比：如果是静态重定位会怎样？

静态重定位在装入时就把所有地址从 `0~N`​ 换算成 `起始地址 ~ 起始地址+N`，写死在程序里。

一旦进程被搬走，里面全是旧地址 → 程序崩溃。所以静态重定位的进程**不能参与紧凑**。

---

### 一句话总结

> **静态重定位** = 装入时换地址，换完锁定，搬不了  
> **动态重定位** = 运行时硬件换算地址，随便搬，改寄存器就行

文档中"动态重定位分区分配"用的是后者，正是靠它支持**紧凑**来消除外部碎片。
