# 3.4 虚拟分页管理

前面介绍的各种存储管理技术都基于一个前提，即内存空间足够大，必须把整个程序全部装入内存才能执行。如果想在有限的内存中装入尽可能多的程序，且每一个程序又要尽可能大，可以采用虚拟存储技术。

一、程序访问的局部性原理（理解即可）  
程序局部性原理：在一段较短的时间内，程序的执行仅局限于某个部分，它所访问的存储空间也局限于某个区域。这个原理分为两种类型：时间局部性和空间局部性，下面举例说明。  
【例】假如你正在图书馆里读《哈利波特 1》，当你读了一章后发现非常有趣，因此吃完饭后你又再次阅读它。由于书里的内容太过精彩，因此你直接把这本书借了出来，接下来的几天里每天都会打开阅读。  
这体现的就是时间局部性，即某个内容被访问后可能不久后会被再次访问。  
当你看完《哈利波特 1》后，你想把哈利波特系列全部看完，于是接下来的几天里，你又去图书馆把《哈利波特 1》，边上的《哈利波特 2》《哈利波特 3》总共 6 本书依次给读完了。  
这体现的就是空间局部性，即某个内容被访问完后，其相近的内容可能被访问。

[图片：哈利波特系列书籍]

对于指令而言，因为指令中存在大量的循环结构，导致同一条指令短时间内可能会多次运行，所以指令有时间局部性；同时指令往往也是顺序执行，执行完第 x 条指令后，很可能执行第 x+1 条指令，所以指令有空间局部性。  
对于数据而言，同一个变量或某个存储单元可能会短时间内被多次访问（例如循环体里面的变量），即数据有时间局部性。许多数据常以数组等连续结构存储，它们被存放在相邻的位置里，且常在一段时间内被顺次访问，被访问的某个存储单元的邻近单元在一个较短的时间间隔内很可能也被访问，所以这类数据具有空间局部性。

【例】考虑以下代码在内存中对数组 a[4][4]的遍历顺序：

```c
void visitArray(int a[][4]) {
    int sum = 0;
    for (int i = 0; i < 4; ++i)
        for (int j = 0; j < 4; ++j)
            sum += a[i][j];
    return sum;
}
```

在这段代码中，变量 sum 会在运行过程中被反复访问，具有很好的时间局部性；  
数组 a 中的每个数据元素在程序执行中仅仅被访问 1 次，故不具备时间局部性。  
每个数据元素的邻近数据元素也会被访问，故对数组 a 的访问具有良好的空间局部性；  
for 语句代码无论是自身还是邻近的指令（求和指令）都会在短时间内被高频访问，所以 for 语句具有良好的时间局部性和空间局部性。

二、虚拟存储器（理解即可）  
（一）基本原理

[图片：磁盘（用户程序A，包含页面0-3）与物理内存（页框0-3）通过“页”进行交互的示意图]

虚拟存储技术在页式或段式存储管理的基础上实现，基本原理如下：  
① 基于程序访问的局部性原理，不必把整个程序都装入到内存中，而只要将当前需要执行的那部分页面或段装入到内存，就可以让程序开始执行。  
② 程序的执行过程中，如果需要执行的指令或需要访问的数据不在内存中，就称为缺页或缺段。缺页（缺段）时会产生一个中断，在中断处理程序中，操作系统将相应的页面或段调入到内存，然后继续执行程序。  
③ 此外，操作系统会将内存中暂时不使用的页面或段调出去，保存在外存上，从而腾出更多的空闲空间来存放将要装入的程序以及将要调入的页面或段。

（二）定义  
传统的存储管理方案需要把整个程序装入内存才能运行，即用户的逻辑地址空间必须小于实际的物理内存。

[图片：磁盘（用户程序A，包含页面0-5）与物理内存（页框0-3）通过“页”进行交互的示意图]

但在虚拟存储技术中，由于程序局部性原理，提供给用户的地址空间通常要大于实际的物理内存，从而实现了两者的分离，即两者已经没有太大关系了。  
【例】32 位的地址空间中，每一个进程都有一个 4GB 的虚拟地址空间。而实际的物理内存由于受价格等因素的制约，过去一般为 512MB，1GB 等。  
因此，逻辑地址空间一般都比较大，其大小由系统的地址结构决定。

【练习】设主存容量为 1MB，外存容量为 400MB，计算机系统的地址寄存器为 32 位，那么进程的虚拟地址空间大小为（ ）。  
A.1MB  
B.401MB  
C.1MB+2³²MB  
D.2³²B

（三）特征（了解即可）  
与传统的存储器管理方式相比，虚拟存储器具有以下3个重要特征。  
1.多次性  
多次性是相对于传统存储器管理方式的一次性而言的，是指一个作业中的程序和数据，无须一次性地全部调入内存，允许被分成多次调入内存运行。  
2.对换性  
对换性是相对于传统存储器管理方式的驻留性而言的，是指一个作业中的程序和数据无须在作业运行时常驻内存，而允许它们在作业运行时进行换入、换出，甚至还允许将暂时不运行的进程调至外存，待它们重又具备运行条件时再调入内存。  
3.虚拟性  
虚拟性是指能够从逻辑上扩大内存容量，使户所看到的内存容量远大于实际内存容量。  
虚拟性是以多次性和对换性为基础的，仅当系统允许将作业分多次调入内存，并能将内存中暂时不运行的程序和数据换至外存时，才有可能实现虚拟存储器。

三、虚拟页式存储管理（理解并记忆）  
（一）基本原理  
虚拟页式存储管理（请求分页管理）是当前主流的存储管理技术，大部分操作系统，如Windows和Linux都采用这种技术。虚拟页式存储管理在页式存储管理的基础上，增加了请求调页等功能。  
基本思路如下：  
①当一个用户程序要调入内存运行时，将该程序的部分页面装入内存，然后就可以启动该程序运行。  
②在运行过程中，如果发现要执行的指令或是要访问的数据不在内存中，就向系统发出缺页中断请求，然后系统在处理这个中断请求时，就会将保存在外存中的相应页面调入到内存，从而使该程序能继续运行。  
③当内存空间不够用时，需要把内存中的页面暂时保存在磁盘上。

[图片：逻辑地址空间（页面0-3）通过MMU映射到物理内存（页框0-3）的示意图]

进程运行时，由内存管理单元（MMU）负责地址映射。MMU是一种特殊硬件，和操作系统一起实现存储访问。为了能实现请求调页功能和页面置换功能，系统必须提供必要的硬件支持和实现请求分页的软件。主要的硬件支持包括请求页表机制、缺页中断机构、地址变换机构。实现请求分页的软件，包括用于实现请求调页的软件和实现页面置换的软件。

（二）请求页表机制  
在基本分页管理中，操作系统为每个进程都建立并维护了一张页表，每一个页表项都需要记录某个逻辑页号对应的物理页号。  
在虚拟页式存储管理中，为了满足页面换入与换出的需要，在请求页表中又增加了四个字段，分别是驻留位、保护位、修改位和访问位（不唯一）。

3.4 虚拟分页管理

[图片：页表项结构示意图，包含逻辑页号i，以及修改位、驻留位、访问位、保护位、物理页号、外存地址]

在虚拟页式存储管理中，只有一部分页面位于内存中，其余位于外存中，如何知道一个页面到底是否在内存呢？这就需要用到驻留位（有效位、状态位）：  
如果驻留位为 1，表示该页面位于内存中，即这个页表项是有效的，它存放了该页面对应的物理页面号。  
如果驻留位为 0，表示该页面不在内存中，即这个页表项是无效的，如果此时去访问它，将会导致缺页中断。  
缺页中断一般是由 MMU 引发的，当 CPU 要去访问一个内存单元时，由 MMU 负责进行地址映射。在地址映射时，它会去内存或 TLB 中读取相应的页表项，如果发现页表项驻留位为 0，就会触发一个缺页中断，这是自动进行的。  
保护位表示允许对该页面进行何种类型的访问，如只读、可读写，可执行等。例如，代码段的页面一般是只读和可执行，而数据段、堆和栈，它们的页面一般是可读、可写的。  
修改位表示该页面在内存中是否已经被修改过。如果该页面的内容已经被修改过，那么 CPU 就会自动地把这一位的值设置为 1。  
在虚拟页式存储管理中，页面需要在内存和磁盘之间来回交换。然而磁盘的读写速度是非常慢的，为了提高这种交换的效率，需要设置一个修改位。  
当系统要把一个页面保存到磁盘上时，它会先检查这个修改位，如果为 0，即没有被修改，则该页面的内容与它在磁盘上的备份完全一样，那么操作系统就可以不必再将其存回磁盘。  
如果该页面被修改过，那么操作系统就必须把它写回到硬盘，否则，那些被修改过的内容就会丢失。  
访问位表示该页面在内存中曾经被访问过。不管是读操作也好，写操作也好，只要是被访问过，该位就会被硬件自动置为 1。这个信息主要用在页面置换算法中。  
【例】假设逻辑地址为 16 位，按字节编址，即逻辑地址空间的大小为 64KB，而实际的物理内存大小只有 32KB。假设每个页面的大小均为 4KB，这样，在逻辑地址空间中总共有 16 个逻辑页面，其编号为 0<sub>15。而物理内存总共只有 8 个物理页面，其编号为 0</sub>7。

由于逻辑地址空间远远大于实际的内存空间，不可能把所有的逻辑页面都装入到内存。  
若8个逻辑页面被装入到内存，则它们的页表当中包含有它们所对应的物理页面号，且驻留位为1。而剩下的8个逻辑页面的驻留位为0，用一个×表示表示没有相对应的物理页面号。  
假设现在要去执行以下指令：

```assembly
MOV R1, [0] // 即访问逻辑地址为0的内存单元，将其值取出来，放到寄存器R1当中
```

这里需要把逻辑地址0转换为相应的物理地址。  
显然，逻辑地址0的逻辑页号为0，页内地址也为0，因此，以页号0为索引，去查找页表，相应的物理页面号为2，因此，最终的物理地址是2×4096+0=8192。  
得到物理地址以后，MMU会把它发送到地址总线上，访问相应的内存单元，访存就顺利完成了！  
假设现在要去执行另外一条指令：

```assembly
MOV R1, [32780] // 即对逻辑地址32780进行访问
```

显然，其逻辑页号为8，页内地址为12。因此，以8为索引，到页表中去查找，结果发现其驻留位的值为0，即该页面并没有在内存当中。  
这样，MMU就无法继续下去，只能引发一个缺页中断，把这个问题提交给操作系统去处理。  
（三）缺页中断  
在地址映射过程中，如果所要访问的逻辑页面p不在内存，则产生缺页中断，中断的处理过程如下。  
[图片：缺页中断处理流程图]  
① 如果在内存中有空闲的物理页面，则分配一个页，设其编号为f，然后转第③步；否则转第②步。  
② 若内存已满，则采用某种页面置换算法选择一个物理页面f，将它保存的逻辑页面q从内存中替换出去。具体来说，如果页面q被修改过，其修改位为1，需要把它的内容写回外存；如果页面q未被修改过，则什么也不做，到时它自然会被新的页面所覆盖。此外，还需对q所对应的页表项进行修改，把驻留位的值置为0，表示这个逻辑页面现在已经被替换出去了，已经不在内存中了，以后如果再要访问该页面，就会出现缺页中断。  
③ 将逻辑页面p装入到物理页面f当中。这个过程需要访问磁盘，在访问磁盘时，当前这个进程就会被阻塞起来，然后系统会调用另一个进程去运行。当I/O操作完成后，会有一个I/O中断，使得原来的进程变为就绪态。此外，还会修改P所对应的页表项的内容，把驻留位的值置为1，把物理页面号设置为f。  
④ 重新运行被中断的指令。

驻留集越大，当分配给进程的页框超过某个数目时，再为进程增加页框对缺页率的改善是不明显的，反而只能是浪费内存空间，还会导致多道程序并发度的下降。  
因此，在为进程分配内存时，需要考虑以下两个问题：  
① 为每个进程分配物理块时，应采取什么样的内存分配策略，即所分配的物理块是固定的还是可变的。  
② 在为不同进程分配物理块时，是采取平均分配算法分配，还是根据进程的大小按比例分配。

1. 内存分配策略  
   在请求分页系统中，可采取两种内存分配策略，即固定分配和可变分配。固定分配策略指驻留集的大小是固定不变的。  
   在进行置换时，也可采取两种策略，即全局置换和局部置换。  
   局部置换指若进程在运行中发生缺页，则只能从分配给该进程在内存的页面中选出一页换出。  
   全局置换指若进程在运行中发生缺页，则系统从空闲物理块队列中取出一块分配给该进程，并将所缺页调入。  
   不同的分配策略和不同的置换策略总共可以组合出以下三种适用的策略。  
   (1) 固定分配局部置换  
   为每个进程分配固定数目的物理块，在进程运行期间都不改变。  
   若进程在运行中发生缺页，则只能从分配给该进程在内存的页面中选出一页置换。  
   实现这种策略时，难以确定应为每个进程分配的物理块数目：太少会频繁出现缺页中断，太多又会降低 CPU 和其他资源的利用率。  
   (2) 可变分配全局置换  
   先为每个进程分配一定数目的物理块，在进程运行期间可根据情况适当地增加或减少。  
   若进程在运行中发生缺页，则系统从空闲物理块队列中取出一块分配给该进程，并将所缺页调入。  
   这种方法比固定分配局部置换灵活，可以动态增加进程的物理块，但也存在弊端，如它会盲目地给进程增加物理块，从而导致系统多道程序的并发能力下降。  
   (3) 可变分配局部置换  
   为每个进程分配一定数目的物理块，当某进程发生缺页时，只允许从该进程在内存的页面中选出一页置换。  
   若进程在运行中频繁地发生缺页中断，则系统再为该进程分配若干物理块，直至该进程的缺页率趋于适当程度；反之，若进程在运行中的缺页率特别低，则可适当减少分配给该进程的物理块，但不能引起其缺页率的明显增加。  
   这种方法在保证进程不会过多地调页的同时，也保持了系统的多道程序并发能力。当然，它需要更复杂的实现，也需要更大的开销，但对比频繁地换入/换出所浪费的计算机资源，这种牺牲是值得的。
2. 物理块分配算法  
   采用固定分配策略时，将系统中的空闲物理块分配给各个进程，可采用下述几种算法。  
   (1) 平均分配算法  
   该算法将系统中所有可供分配的物理块平均分配给各个进程。  
   例如，当系统中有 100 个物理块且有 5 个进程在运行时，每个进程可分得 20 个物理块。这种方式貌似公平，但由于未考虑各进程本身的大小，会造成实际上的不公平。  
   若一个进程只有 10 页，为其分配 20 个物理块后则会有 10 个物理块闲置；而另外一个进程有 200 页，也被分配了 20 个物理块，则显然不够用，且必然会有很高的缺页率。  
   (2) 按比例分配算法  
   该算法根据进程的大小按比例分配物理块，如果系统中共有 n 个进程，每个进程的页面数为 Si，则系统中各进程页面数的总和 S 为：

$$
S = \sum_{i=1}^{n} S_i
$$

再假定系统中可用的物理块总数为 m，则可由下式算得每个进程所能分到的物理块数 $b_i$ 为：

$$
b_i = \frac{S_i}{S} \times m
$$

其中，$b_i$ 应该取整，且必须保证可以让进程正常运行。

 **(3) 考虑优先权的分配算法**

在实际应用中，为了使重要的、紧迫的作业能尽快完成，应为它分配较多的内存空间。  
通常采取的方法是把内存中可供分配的所有物理块分成两部分：一部分按比例分配给各进程；另一部分则根据各进程的优先权进行分配，为高优先权进程适当地增加其相应的分配份额。

 **(六) 页面调入策略（了解即可）**

为使进程能够正常运行，必须事先将要执行的那部分程序和数据所在的页面调入内存。但问题是：  
① 系统应在何时调入所需页面？  
② 系统应从何处调入这些页面？

**1. 何时调入策略**

为确定系统将进程运行时所缺的页面调入内存的时机，可采用以下两种调页策略：

 **(1) 预调页策略**

根据局部性原理，一次调入若干个相邻的页面会比一次调入一页更高效。但若提前调入的页面中大多数都未被访问，则又是低效的。因此，可以预测不久之后可能被访问的页面，将它们预先调入内存。

[图片：进程A的页面（页0-页7）与物理内存页框（页框0-页框7）的映射关系示意图，展示了部分页面被调入内存]

 **(2) 请求调页策略**

进程在运行中需要访问的页面不在内存，便提出请求，由系统将其所需页面调入内存。由这种策略调入的页一定会被访问，且比较易于实现，因此目前的虚拟存储器大多采用此策略。  
其缺点是每次仅调入一页，增加了磁盘 I/O 开销。  
预调页实际上就是运行前的调入，请求调页实际上就是运行期间的调入。

**2. 从何处调入页面**

请求分页系统中的外存分为两部分：用于存放文件的文件区和用于存放对换页面的对换区，也称交换区。对换区的磁盘 I/O 速度比文件区的更快。

(1) 系统拥有足够的对换区空间  
可以全部从对换区调入所需页面，以提高调页速度。为此，在进程运行前，需将与该进程有关的文件从文件区复制到对换区。  
(2) 系统缺少足够的对换区空间  
这时，凡是不会被修改的文件都直接从文件区调入；而当换出这些页面时，由于它们未被修改，不必再将它们重写到磁盘（换出），以后再调入时仍从文件区直接调入。  
但对于那些可能被修改的部分，在将它们换出时，须将它们调到对换区，以后需要时再从对换区调入。  
(3) UNIX 方式  
由于与进程有关的文件都放在文件区，故凡是未运行过的页面，都应从文件区调入。  
而对于曾经运行过但又被换出的页面，由于被放在对换区，因此在下次调入时，应从对换区调入。  
由于 UNIX 系统允许页面共享，因此，某进程所请求的页面有可能已被其他进程调入内存，此时也就无须再从对换区调入。

四、请求分段系统（了解即可）  
相对于请求分段系统，请求分页系统换入与换出的基本单位都是固定大小的页面，所以在实现上要容易些。而请求分段系统换入与换出的基本单位是段，其长度是可变的，且段的分配类似于动态分区分配，它在内存分配和回收上都比较复杂。从真题和考纲来看，这块知识点考察概率不大。
