# 请求分页系统

## 请求分页系统

### 三、请求分页系统

在基本分页基础上增加**请求调页**和**页面置换**功能。

#### （一）硬件支持

**MMU（内存管理单元）：**  特殊硬件，和 OS 一起实现存储访问。

**页表新增字段：**

- **驻留位（有效位/状态位）：**  1=在内存；0=不在内存，产生缺页中断
- **修改位：**  1=被修改过，换出时需写回磁盘
- **访问位：**  1=被访问过，供置换算法参考

#### （二）缺页中断处理

当驻留位=0 时，MMU 引发**缺页中断**，交给 OS 处理：

1. 判断内存中是否还有空闲物理页面
2. 有空闲 → 分配页框，从磁盘读入所缺页面，修改页表项
3. 无空闲 → 采用**页面置换算法**选一页换出
4. 被换出页修改过？→ 写回磁盘
5. 读入所需页 → 更新页表 → 回到原指令**重新执行**

**关键特点：**

- 对进程透明——进程感知不到缺页中断
- 缺页中断在**指令执行期间**产生，中断返回后重新执行该指令（不是下一条）

 **【例】**  逻辑地址 32780 → 页号 8，驻留位=0 → 缺页中断。系统所有 8 个物理页面已全部分配，选 1 号物理页换出。若 1 号页被修改过则写回磁盘，然后读入所需页。

#### （三）含 TLB 的地址变换流程

1. 检索 TLB → 命中则得物理块号，修改访问位（写指令还需将修改位置 1）
2. TLB 未命中 → 查页表 → 找到则取出物理块号，将页表项写入 TLB（TLB 满则替换）
3. 页表中未找到（驻留位=0）→ **缺页中断处理**：调页入内存，更新 TLB 和页表
4. 重新执行原指令

 **【练习】**  TLB 命中率约 75%，使用二级页表，忽略访问 TLB 时间。每次存取平均访存次数？

> 命中 TLB：1 次访存 | 未命中：二级页表 2 次 + 访存 1 次 = 3 次  
> 平均 = 0.75 × 1 + 0.25 × 3 = **1.5 次**

### 四、请求分段系统（了解即可）

请求分段换入/换出基本单位是**段**（长度可变），类似动态分区分配，内存分配和回收更复杂。从真题和考纲看考察概率不大。
