# 程序局部性原理与虚拟存储器

## 程序局部性原理与虚拟存储器

### 一、程序访问的局部性原理

在一段较短的时间内，程序的执行仅局限于某个部分，访问的存储空间也局限于某个区域。

**时间局部性：**  某指令/数据被访问后，短时间内可能再次被访问。如循环体中同一指令多次运行、变量反复访问。

**空间局部性：**  某指令/数据被访问后，其邻近的指令/数据也可能被访问。如指令顺序执行、数组顺序访问。

 **【例】**  二维数组 `a[4][4]`​ 按行遍历 `for(i) for(j) sum += a[i][j]`：

- ​`sum`：反复访问 → 时间局部性好
- 数组 `a`：相邻元素依次访问 → 空间局部性好（但每个元素仅访问 1 次，时间局部性差）
- ​`for` 语句：高频执行 → 时间+空间局部性都好

---

### 二、虚拟存储器

#### （一）基本原理

基于程序局部性原理，**不必把整个程序装入内存**即可执行：

1. 只装入当前需要的页面/段即可开始执行
2. 执行中若指令/数据不在内存 → **缺页（缺段）中断** → OS 调入 → 继续执行
3. OS 将暂时不用的内容换出到外存

#### （二）定义

虚拟存储技术下，**逻辑地址空间 > 实际物理内存**，两者分离。

 **【例】**  32 位系统每个进程有 4GB 虚拟地址空间，但物理内存一般为 512MB~1GB。逻辑地址空间大小由系统的**地址结构**决定。

 **【练习】**  设主存容量为 1MB，外存容量为 400MB，地址寄存器为 32 位，进程虚拟地址空间大小为？

> 答案：**$2^{32}$**​**B**（由地址结构决定，与内存外存大小无关）
