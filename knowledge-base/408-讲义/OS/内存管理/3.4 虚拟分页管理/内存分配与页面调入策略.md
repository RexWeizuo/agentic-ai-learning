# 内存分配与页面调入策略

## 内存分配与页面调入策略

### 五、请求分页中的内存分配

**驻留集：**  OS 分配给进程的页框集合。

- 驻留集小 → 并发度高，但缺页率高，CPU 大量时间处理缺页
- 驻留集大 → 超过阈值后再增页框对缺页率改善不明显，反而浪费内存、降低并发度

**缺页率：**  $f = F / A$（F = 缺页次数，A = 总页面访问次数 = S + F，S = 成功次数）

#### 分配策略 × 置换策略组合

||固定分配|可变分配|
| --| -------------------------------| -------------------------------------------|
|**局部置换**|✓ 驻留集不变，从自身页面换出|✓ 缺页率高时增配，低时回收|
|**全局置换**|× 不可能|✓ 从系统空闲队列取块，可抢占其他进程页框|

#### 物理块分配算法

1. **平均分配：**  所有进程均分
2. **按比例分配：**  $b_i = (s_i / S) \times m$（$s_i$=进程大小，$S$=总大小，$m$=总物理块数），取整且保证进程能正常运行
3. **考虑优先权：**  一部分按比例 + 一部分按优先权增配

### 六、页面调入策略

#### 1. 何时调入

|策略|时机|特点|
| ------| ------------------------| -------------------------------------------------------------|
|**预调页**|运行前批量调入相邻页面|利用局部性，一次调多页更高效；但若调入后大多未访问则低效|
|**请求调页**|缺页时才调入|调入的页一定会被访问，实现简单；每次仅一页，磁盘 I/O 开销大|

预调页 = 运行前调入；请求调页 = 运行期间调入。目前虚拟存储器**大多采用请求调页**。

#### 2. 从何处调入

外存分为**文件区**（存放文件）和**对换区**（存放对换页面，磁盘 I/O 更快）。

|情况|调入来源|
| ------------------| ------------------------------------------------|
|有足够对换区空间|全部从对换区调入（运行前将文件复制到对换区）|
|缺对换区空间|不会修改的→从文件区调入；可能修改的→放对换区|

### 补充：逻辑容量 vs 最大容量

- **虚拟存储器逻辑容量：**  由**内存容量 + 外存容量**决定
- **虚拟存储器最大容量（虚拟地址空间）：**  由**地址结构**决定 = $2^{32}$B

 **【练习】**  主存 1MB，外存 400MB，地址寄存器 32 位。虚拟存储器的逻辑容量 = 1MB + 400MB = **401MB**

### Linux 进程虚拟地址空间

|区域（低→高地址）|说明|
| ---------------------| --------------------------|
|只读代码段|可执行指令，只读|
|读/写数据段|全局变量和静态变量|
|运行时堆|malloc/free 动态增长收缩|
|共享库|C 标准库等公共代码和数据|
|用户栈|局部变量、函数调用|
|OS 内核区（高地址）|内核代码和数据，PCB|
