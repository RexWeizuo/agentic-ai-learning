# 补充（程序装入内存分区后的结构 + 地址空间实例）

【补充】程序装入内存分区后的结构（理解即可）  
当我们要运行一个程序时，首先在内存当中找到一个空闲的内存分区，然后从磁盘把这个程序装入到该分区中，创建对应的进程。

[图片：磁盘到内存的示意图，显示程序从磁盘装入内存的空闲分区]

Q：当一个程序被装入到某个内存分区，变成进程以后，这个分区内部的结构是什么样的呢？  
【例】假设某个源程序文件的代码如下所示

```c
/* 全局变量 */
int global_x;
......
/* 函数*/
int f(int a)
{
    ......
    /* 局部变量*/
    int i, j;
    /* 动态申请内存空间 */
    double * d= malloc(sizeof(double) * 5);
    return i;
}
......
```

在经过编译链接之后，源文件就变成了可执行文件。对于不同的操作系统，它所支持的可执行文件的格式是不一样的。假设可执行文件由文件头、代码区、数据区和其他内容组成。

|文件头|代码区|数据区|其他|
| :-----------------------------------------| :------------------------------------------| :----------------------| :-------|
|......<br />int a_text; //代码段大小<br />......|......<br />MOV DS,AX<br />......<br />POP AX<br />......|int global_x;<br />......|......|

可执行文件的内部结构

说明如下：  
① 文件头：存放了可执行文件的一些描述信息，如每个部分的起始地址和长度。  
② 代码区，存放的是可执行指令。

--- Page 141 ---  
③ 数据区，存放的是程序中的全局变量。注意，只有全局变量才需要存放在可执行文件中，而局部变量则不需要，因为局部变量是在函数调用的时候，才在栈空间中分配相应的存储空间。  
此外，在可执行文件中还有其他一些内容，这里就不用考虑了。  
对于上面这个可执行文件，当它被装入到某个内存分区之后，就变成了下图所示的样子。

[图片：可执行文件从磁盘装入内存的示意图。左侧为磁盘上的可执行文件，包含文件头、代码区、数据区和其他；右侧为内存分区，从高地址到低地址依次为栈（SP指向）、动态堆空间、数据段、代码段（PC指向）。箭头表示代码区和数据区分别被装入内存的代码段和数据段。]

① 可执行文件中的代码区的内容被复制到该内存分区的起始地址，形成代码段。  
 可执行文件中的数据区的内容被复制到代码段的上方，构成数据段。  
③ 随后，栈指针 SP 指向这块内存分区的最高点，创建了栈这个段，并把 SP 指向栈的起始地址。  
④ 在栈和数据段之间，是堆空间。  
这样就把这个可执行文件装入到了内存，然后把 PC 指向代码段的起始地址，就能够一条指令一条指令地去执行了。在指令的执行过程中，如果要去访问全局变量，那么就去访问数据段；如果要去访问动态分配的内存空间，那么就去访问堆空间；如果发生了函数调用，那么就在栈当中分配一块栈帧，用来存放它的形参和局部变量。

【补充】地址空间实例（理解并记忆）  
linux 系统下，进程执行时的地址空间如下所示：

[图片：Linux进程地址空间示意图，从低地址0x0000 0000到高地址0xFFFF FFFF，依次划分为未使用区、只读代码段、读/写数据段、动态生成的堆（用户空间）、共享库、用户栈、操作系统内核区（内核空间）]

下面按照虚拟地址递增顺序依次进行介绍。  
（一）只读代码段  
存储可执行代码的指令。只读，通常不可修改。  
（二）数据段  
存储全局变量和静态变量。  
（三）运行时堆  
由进程在运行时动态创建，该区域可以动态增长和收缩，例如 C 语言的 malloc()可以申请该区域的内存，free()可以释放该区域的内存。  
（四）共享库  
该区域用于存放公共的共享库代码和数据，例如 C 语言的标准库函数。  
（五）用户栈  
该区域用来存放局部变量、返回地址，函数调用参数等信息，可以利用该区域实现函数调用。和堆一样，该区域也可以动态增长和收缩。  
（六）操作系统内核区  
操作系统内核使用的地址空间，包括内核程序、内核数据等信息，这些信息对用户程序不可见。  
一个进程映像（实体）主要包括三个部分：程序段、相关的数据段和 PCB。  
【例】有代码如下所示。

```c
......
char *ptr;
void main() {
    int length;
    ptr=(char*)malloc(10);
    length=sizeof(ptr);
    printf("%d",length);
    free(ptr);
}
```

(1) 代码执行时，PCB 位于哪个区域？执行 scanf()函数时，进程处于什么状态？  
(2) main()函数代码、ptr 指针、length 变量、malloc()函数以及 ptr 执行的字符串各位于哪个区域？  
【答案】(1) PCB 位于内核区，执行 scanf()时，进程处于阻塞态。  
(2) main()函数的代码位于只读代码段，ptr 位于读/写数据段，length 位于用户栈，malloc()函数的代码位于共享库，ptr 指向的字符串位于运行时堆。
