# 补充、地址空间实例

【补充】地址空间实例（理解并记忆）  
linux 系统下，进程执行时的地址空间如下所示：

[图片：Linux进程地址空间示意图，从低地址0x0000 0000到高地址0xFFFF FFFF，依次划分为未使用区、只读代码段、读/写数据段、动态生成的堆（用户空间）、共享库、用户栈、操作系统内核区（内核空间）]

下面按照虚拟地址递增顺序依次进行介绍。  
（一）只读代码段  
存储可执行代码的指令。只读，通常不可修改。  
（二）数据段  
存储全局变量和静态变量。  
（三）运行时堆  
由进程在运行时动态创建，该区域可以动态增长和收缩，例如 C 语言的 malloc()可以申请该区域的内存，free()可以释放该区域的内存。  
（四）共享库  
该区域用于存放公共的共享库代码和数据，例如 C 语言的标准库函数。  
（五）用户栈  
该区域用来存放局部变量、返回地址，函数调用参数等信息，可以利用该区域实现函数调用。和堆一样，该区域也可以动态增长和收缩。  
（六）操作系统内核区  
操作系统内核使用的地址空间，包括内核程序、内核数据等信息，这些信息对用户程序不可见。  
一个进程映像（实体）主要包括三个部分：程序段、相关的数据段和 PCB。  
【例】有代码如下所示。

```c
......
char *ptr;
void main() {
    int length;
    ptr=(char*)malloc(10);
    length=sizeof(ptr);
    printf("%d",length);
    free(ptr);
}
```

(1) 代码执行时，PCB 位于哪个区域？执行 scanf()函数时，进程处于什么状态？  
(2) main()函数代码、ptr 指针、length 变量、malloc()函数以及 ptr 执行的字符串各位于哪个区域？  
【答案】(1) PCB 位于内核区，执行 scanf()时，进程处于阻塞态。  
(2) main()函数的代码位于只读代码段，ptr 位于读/写数据段，length 位于用户栈，malloc()函数的代码位于共享库，ptr 指向的字符串位于运行时堆。
