# 四、程序的装入

四、程序的装入（理解并记忆）  
在将一个装入模块装入内存时，可以有如下3种装入方式。  
（一）绝对装入方式  
当计算机系统很小，仅能运行单道程序时，完全有可能知道程序将驻留在内存的什么位置。此时可以采用绝对装入方式。

[图片：应用程序装入内存示意图，左侧为应用程序，右侧为内存（包含系统区和用户区）]

用户程序经编译后，将产生绝对地址（物理地址）的目标代码。

```
1000    ...             1000    ...
1100    str 5, [1200]   1100    str 5, [1200]
        ldr R1, [1200]          ldr R1, [1200]
        add R2, R1,3            add R2, R1,3
        str R2, [1204]          str R2, [1204]
1200    X               1200    X
1204    y               1204    y
1300    ...             1300    ...
      可执行文件                内存
```

绝对装入程序便可按照装入模块（可执行文件）中的地址，将程序和数据装入内存。装入模块被装入内存后，由于程序中的逻辑地址与内存地址完全相同，故无须对程序和数据的地址进行修改。  
（二）可重定位装入方式（静态重定位）  
在多道程序环境下，编译程序不可能预知装入模块应放在内存的何处，因此采用可重定位装入方式。  
若当用户程序被装入内存时，直接对指令代码进行修改，一次性实现逻辑地址到物理地址的转换，以后不再转换，该方式则称为静态重定位。（刚刚已经讲过静态重定位了，只是从装入的角度来解释）

```
0       ...             1000    ...
100     str 5, [200]    1100    str 5, [1200]
        ldr R1, [200]           ldr R1, [1200]
        add R2, R1,3            add R2, R1,3
        str R2, [204]           str R2, [1204]
200     X               1200    X
204     y               1204    y
300     ...             1300    ...
      可执行文件                内存
```

可重定位装入方式克服了绝对装入方式的缺点，不再需要预先知道装入位置，但该方式不允许程序在运行时移动位置。  
（三）动态运行时装入方式（动态重定位）  
当用户程序被装入到内存时，若不对指令代码做任何的修改，而是在程序的运行过程中，当它需要访问内存单元时，再来进行地址转换，则称为动态重定位。  
为了提高效率，通常会设置一个基地址寄存器（重定位寄存器），即物理地址=基址寄存器中的程序起始地址+逻辑地址。装入模块在装入后有可能还会调动位置，动态运行装入方式可以很好地应对这一情况。
