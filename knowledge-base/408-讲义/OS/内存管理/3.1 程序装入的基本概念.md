# 3.1 程序装入的基本概念

多道程序环境下，多个进程并发执行时的指令和数据被保存在内存中，因此对于内存资源的管理（就是管理指令和数据）非常重要。本章重点研究操作系统是如何管理内存的 Y(<sup>_</sup>)Y。

其中，预处理、编译、汇编的过程也可被通称为“编译”。  
因此，要在系统中运行用户程序，一般要经过以下3个步骤，分别是编译、链接、装入，下面通过一个具体的例子讲解整个过程。

【例】假设使用如下的用户程序文件完成一个加法计算的任务，代码如下所示。

```c
#ifndef ADD_H
#define ADD_H

int add(int a, int b);

#endif
```

add.h

```c
#include "add.h"

int add(int a, int b) {
    return a + b;
}
```

add.c

```c
#include <stdio.h>
#include "add.h"

int main() {
    int num1 = 5;
    int num2 = 3;
    int result = add(num1, num2);
    printf("%d + %d = %d\n", num1, num2, result);
    return 0;
}
```

main.c

  
步骤 1：编译（Compiler）  
由编译程序将用户源程序 add.c 和 main.c 分别编译成目标模块。

```bash
leon@leon-virtual-machine:~/OS/example$ gcc -c add.c -o add.o
leon@leon-virtual-machine:~/OS/example$ gcc -c main.c -o main.o
```

[图片：add.c 编译为 add.o，main.c 编译为 main.o 的示意图]  
不妨使用 nm 指令查看两个目标模块的构造。

```bash
leon@leon-virtual-machine:~/OS/example$ nm add.o
0000000000000000 T add
```

T 表示 Text section，即代码段，这说明 add.o 模块包含了 add 函数，其起始地址为 0000000000000000H。

```bash
leon@leon-virtual-machine:~/OS/example$ nm main.o
                 U add
0000000000000000 T main
                 U printf
```

U 表示 Undefined reference，这说明 main.o 中引用了 add 函数以及 printf 函数，但这两个函数未在 main.o 中定义。main 函数的起始地址为 0000000000000000H。  
步骤 2：链接（Linker）  
显然，main.o 模块中并没有加法操作以及输出操作的具体实现，因此需要使用链接程序将若干个目标模块以及它们所需要的库函数链接在一起，形成一个完整的装入模块。

```bash
leon@leon-virtual-machine:~/OS/example$ gcc add.o main.o -o add_program
```

[图片：add.o, main.o 和 printf 库链接生成 add_program 的示意图]  
步骤 3：装入（Loader）和运行  
接着由装入程序将装入模块装入内存，设置好程序的执行环境，随后执行该程序。

```bash
leon@leon-virtual-machine:~/OS/example$ ./add_program
5 + 3 = 8
```

二、逻辑地址和静态链接（理解并记忆）  
经过编译后，便得到了一组目标模块，这些模块可以看作是包含了指令和数据的二进制文件，所有的高级语言代码或者汇编指令都转换为了机器语言形式，如下所示：

显然，main 函数如果要调用 add 函数，需要通过一条 call 指令实现，即 call add，在编译时，需要将 add 这个符号转换成具体的地址。  
Q：问题来了，这个时候只有多个目标模块，它们需要经过链接和装入后才可以放入内存中，确定物理地址。然而，各个模块内的指令和数据都需要地址，否则就没法说清楚各个指令之间的相对位置，也没法根据某个地址去访问某个数据，或者实现程序跳转。该怎么办呢？  
其实，最简单的办法就是，各个模块都直接从地址 0 开始不断向下编址，这个地址就叫做逻辑地址或相对地址。

[图片：add.o和main.o两个目标模块加上从0开始的地址编号示意图]

main.o  
有了目标模块之后，接着将各个模块链接成一个装入模块。  
【例】下图中给出了经过编译后所得的3个目标模块A、B、C，它们的长度分别为L、M、N。  
[图片：模块A、模块B、模块C的结构图，显示了地址范围0到L-1/M-1/N-1以及内部指令CALL和Return]  
在模块 A 中有一条语句 CALL B，用于调用模块 B。  
在模块 B 中有一条语句 CALL C，用于调用模块 C。  
显然，同一个可执行文件中，肯定不允许出现两个相同的地址。例如，若有一条 JMP 100 的指令，如果有两个地址为 100 的指令，那接下来该跳转到哪里继续执行呢？  
因此，在将多个目标模块合并成一个可执行文件的过程中，必须重新给各个模块编址，使得他们形成一个整体，这个过程就叫链接。通过链接形成的完整的装入模块又可以称为可执行文件。

接下来，就要开始运行这个程序了。但现在有一个问题，由于这段程序现在已经在内存中，当前的地址都是实际的物理地址，如 1100、1200。但是指令中所访问的地址还是逻辑地址，如 200 和 204，如果按照 200 和 204 这些地址去访问内存的话会出错，无法正确取到数据。

因此，需要将逻辑地址转换为物理地址，这个过程称为地址映射（地址重定位），主要有两种映射方

式，分别是静态重定位和动态重定位。  
（一）静态重定位  
静态重定位也叫做静态地址映射。  
基本思路：当用户程序被装入内存时，直接对指令代码进行修改，一次性实现逻辑地址到物理地址的转换，以后不再转换。  
【例】如果采用静态地址映射方式，那么在把该程序装入到内存后，如下图所示。

[图片：静态重定位示意图。左侧为“可执行文件”，包含地址0, 100, 200, 204, 300及对应指令和数据（如str 5, [200]等）。中间箭头指向右侧“内存”，标注“装入分区”。右侧内存起始地址为1000，对应指令中的地址变为物理地址（如str 5, [1200]）。]

具体来说，由于内存分区的起始地址为 1000，因此，每一个逻辑地址只要加上 1000，就得到了相应的物理地址。经过这样的修改以后，所有的逻辑地址都转换成了物理地址。  
由于指令中使用的地址都是物理地址，因此不能把这段程序再移动到内存的其他地方，否则会造成内存访问错误。  
优点：实现起来很简单，不需要任何硬件方面的支持。  
缺点：程序一旦装入到内存以后，就不能再移动了。  
（二）动态重定位  
动态重定位也叫动态地址映射。其基本思路为，当用户程序被装入到内存时，不对指令代码做任何的修改，而是在程序的运行过程中，当它需要访问内存单元时，再来进行地址转换。为了提高效率，通常设置一个基地址寄存器（重定位寄存器）来实现动态重定位。  
【例】如果采用动态地址映射方式，那么在把该程序装入到内存后如下图所示，指令中的每一个逻辑地址都原封不动，不做任何修改。

[图片：动态重定位示意图。左侧为“可执行文件”，指令保持原样（如str 5, [200]）。中间箭头指向右侧“内存”，标注“装入”。右侧内存中指令地址未变。右上方有一个“基地址寄存器”值为1000。图示展示了相对地址200与基地址1000相加得到物理地址1200的过程。]

程序执行时，操作系统会把起始地址 1000 装入到基地址寄存器中。  
当这个程序运行时，如果在指令执行过程中需要访问内存，硬件装置就会自动地把其中的相对地址取出来，如 200，然后把它和基地址寄存器做一个加法，从而得到实际的物理地址，即 1200，然后再用这个新的地址来访问内存单元。

四、程序的装入（理解并记忆）  
在将一个装入模块装入内存时，可以有如下3种装入方式。  
（一）绝对装入方式  
当计算机系统很小，仅能运行单道程序时，完全有可能知道程序将驻留在内存的什么位置。此时可以采用绝对装入方式。

[图片：应用程序装入内存示意图，左侧为应用程序，右侧为内存（包含系统区和用户区）]

用户程序经编译后，将产生绝对地址（物理地址）的目标代码。

```
1000    ...             1000    ...
1100    str 5, [1200]   1100    str 5, [1200]
        ldr R1, [1200]          ldr R1, [1200]
        add R2, R1,3            add R2, R1,3
        str R2, [1204]          str R2, [1204]
1200    X               1200    X
1204    y               1204    y
1300    ...             1300    ...
      可执行文件                内存
```

绝对装入程序便可按照装入模块（可执行文件）中的地址，将程序和数据装入内存。装入模块被装入内存后，由于程序中的逻辑地址与内存地址完全相同，故无须对程序和数据的地址进行修改。  
（二）可重定位装入方式（静态重定位）  
在多道程序环境下，编译程序不可能预知装入模块应放在内存的何处，因此采用可重定位装入方式。  
若当用户程序被装入内存时，直接对指令代码进行修改，一次性实现逻辑地址到物理地址的转换，以后不再转换，该方式则称为静态重定位。（刚刚已经讲过静态重定位了，只是从装入的角度来解释）

```
0       ...             1000    ...
100     str 5, [200]    1100    str 5, [1200]
        ldr R1, [200]           ldr R1, [1200]
        add R2, R1,3            add R2, R1,3
        str R2, [204]           str R2, [1204]
200     X               1200    X
204     y               1204    y
300     ...             1300    ...
      可执行文件                内存
```

可重定位装入方式克服了绝对装入方式的缺点，不再需要预先知道装入位置，但该方式不允许程序在运行时移动位置。  
（三）动态运行时装入方式（动态重定位）  
当用户程序被装入到内存时，若不对指令代码做任何的修改，而是在程序的运行过程中，当它需要访问内存单元时，再来进行地址转换，则称为动态重定位。  
为了提高效率，通常会设置一个基地址寄存器（重定位寄存器），即物理地址=基址寄存器中的程序起始地址+逻辑地址。装入模块在装入后有可能还会调动位置，动态运行装入方式可以很好地应对这一情况。

五、程序的链接（理解并记忆）  
（一）静态链接方式  
前面已经学习了静态链接方式，它指将若干个目标模块在装入前链接成一个完整的装配模块后，再将其装入内存。

```text
0       模块A
L-1     CALL B;
        Return;

0       模块B
M-1     CALL C;
        Return;

0       模块C
N-1     Return;

目标模块
```

[图片：展示了静态链接过程。左侧为三个独立的目标模块（模块A、B、C），中间通过“链接”箭头指向“装入模块”。装入模块中，模块A的CALL B变为JSR "L"，模块B的CALL C变为JSR "L+M"，各模块按顺序拼接，地址从0到L+M+N-1。最后通过“装入”箭头指向“内存”。]

（二）装入时动态链接  
装入时动态链接是指，在装入一个目标模块时，若发生一个外部模块调用事件，则将引起装入程序找出相应的外部目标模块，并将它装入内存，修改目标模块中的相对地址。

```text
0       模块A
L-1     CALL B;
        Return;

0       模块B
M-1     CALL C;
        Return;

0       模块C
N-1     Return;
```

[图片：展示了边装入边链接的过程。左侧为目标模块。中间箭头指向右侧“内存”。内存中依次装入了模块A（JSR "L"）、模块B（JSR "L+M"）、模块C。]

装入时动态链接方式有以下优点。

1. 便于修改和更新  
   对于经静态链接装配在一起的装入模块，如果要修改或更新其中的某个目标模块，则要求重新打开装入模块。这不仅是低效的，而且有时是不可能的。若采用动态链接方式，则由于各目标模块是分开存放的，因此要修改或更新各目标模块是一件非常容易的事。
2. 便于实现对目标模块的共享  
   在采用静态链接方式时，每个应用模块都必须含有其目标模块的复制版本，而无法实现对目标模块的共享。

六、内存保护（理解即可）  
在多道程序并发运行的环境下，操作系统位于内存中，多个用户程序也位于内存中。为了防止一个用户程序去访问其他用户程序的内存区域，保护操作系统免受破坏，就必须进行内存保护。  
【例】假设在内存中有三个分区，按字节编址，每个分区的大小均为 100KB，分别存放了操作系统、用户程序 A 和用户程序 B。

① 在用户程序 A 执行前，把它所在的内存分区的起始地址，即 100K，放入到基地址寄存器中。  
② 同时把这个分区的长度，即 100K，放入到限长寄存器中。  
③ 然后，在用户程序 A 的运行过程中，当它需要访问内存单元时，硬件就会自动地把相应的逻辑地址与限长寄存器中的值进行比较，以防止它越界访问。另外，也可以通过判断“基地址≤物理地址<（基地址+界限地址）”是否成立来防止越界访问。

当在用户态下执行的程序试图访问 OS 内存或其他用户内存时，会陷入 OS 内核，而 OS 内核则会将其作为致命错误来处理，进而防止用户程序修改 OS 以及其他用户的代码或数据。  
加载基地址寄存器和界限寄存器时必须使用特权指令，由于特权指令只能在内核态下执行，因此只有 OS 内核才可以加载基地址寄存器和界限寄存器。

【补充】地址空间实例（理解并记忆）  
linux 系统下，进程执行时的地址空间如下所示：

[图片：Linux进程地址空间示意图，从低地址0x0000 0000到高地址0xFFFF FFFF，依次划分为未使用区、只读代码段、读/写数据段、动态生成的堆（用户空间）、共享库、用户栈、操作系统内核区（内核空间）]

下面按照虚拟地址递增顺序依次进行介绍。  
（一）只读代码段  
存储可执行代码的指令。只读，通常不可修改。  
（二）数据段  
存储全局变量和静态变量。  
（三）运行时堆  
由进程在运行时动态创建，该区域可以动态增长和收缩，例如 C 语言的 malloc()可以申请该区域的内存，free()可以释放该区域的内存。  
（四）共享库  
该区域用于存放公共的共享库代码和数据，例如 C 语言的标准库函数。  
（五）用户栈  
该区域用来存放局部变量、返回地址，函数调用参数等信息，可以利用该区域实现函数调用。和堆一样，该区域也可以动态增长和收缩。  
（六）操作系统内核区  
操作系统内核使用的地址空间，包括内核程序、内核数据等信息，这些信息对用户程序不可见。  
一个进程映像（实体）主要包括三个部分：程序段、相关的数据段和 PCB。  
【例】有代码如下所示。

```c
......
char *ptr;
void main() {
    int length;
    ptr=(char*)malloc(10);
    length=sizeof(ptr);
    printf("%d",length);
    free(ptr);
}
```

(1) 代码执行时，PCB 位于哪个区域？执行 scanf()函数时，进程处于什么状态？  
(2) main()函数代码、ptr 指针、length 变量、malloc()函数以及 ptr 执行的字符串各位于哪个区域？  
【答案】(1) PCB 位于内核区，执行 scanf()时，进程处于阻塞态。  
(2) main()函数的代码位于只读代码段，ptr 位于读/写数据段，length 位于用户栈，malloc()函数的代码位于共享库，ptr 指向的字符串位于运行时堆。
