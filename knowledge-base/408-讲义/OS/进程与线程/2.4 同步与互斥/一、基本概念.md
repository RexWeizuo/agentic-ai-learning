# 一、基本概念

## （一）互斥

### 定义

当进程 A 访问某个资源时，必须先提出请求，若此时该资源空闲，则系统便将之分配给 A 使用，此后有其他进程也要访问该资源时（只要 A 未用完）就必须等待。仅当 A 访问完并释放该资源后，才允许另一个进程对该资源进行访问。

这种资源共享方式称为**互斥共享**，进程 A 和进程 B 的关系为**互斥关系**，也叫**间接制约关系**。

## （二）同步

### 定义

当 A 进程完成数据输入后，B 进程才可运行，B 进程计算出结果后，C 进程才可输出结果。此时，进程之间的这种制约关系称作**同步关系**，又叫**直接制约关系**。

## （三）临界资源与临界区

### 临界资源的含义

**临界资源**指一次仅允许一个进程使用的资源，既可以是硬件资源，也可以是软件资源，比如打印机、变量、队列等。

### 为什么需要临界区

由于进程的并发执行可能会导致错误的运行结果，因此需要相应的规则来加以约束进程对共享资源的使用。因此，我们引入了临界资源和临界区的概念。

### 并发执行流分析

【例】有两个进程 P1 和 P2，它们共享一个变量 n，程序代码如下。当两个进程并发执行时的输出结果是怎么样的？

```c
int n=0;//n 为共享变量
P1(){
    while (n<5)
    {
        n++;
    }
    printf("A=%d\n",n);
}
P2(){
    while (n<5)
    {
        n++;
    }
    printf("B=%d\n",n);
}
```

解：由于两个进程并发执行时，很可能 P1 执行了某个语句后跳转到 P2 执行，因此处理这种问题时我们最好先给各个语句标号，这样方便后续描述。

```c
int n=0;//n 为共享变量
P1(){
    ①while (n<5)
    {
        ②n++;
    }
    ③printf("A=%d\n",n);
}
P2(){
    ④while (n<5)
    {
        ⑤n++;
    }
    ⑥printf("B=%d\n",n);
}
```

1. 倘若 P1 执行完语句③后切换到 P2 执行，则输出结果为：
   A=5
   B=5
2. 倘若 P2 执行完语句⑥后切换到 P1 执行，则输出结果为：
   B=5
   A=5
3. 倘若按照①②①②①②①②的顺序执行，此时 n=4；
   接着执行①，由于 n<5 成立，接下来将要执行②；
   若即将执行②时发生进程切换，执行④，由于 n<5 成立，接下来执行⑤，此时 n=5；
   接着执行⑥后切换到 P1，继续执行②，此时 n=6；
   接着执行①，由于 n<5 不成立，接下来执行③，最终输出结果为：
   B=5
   A=6

显然，并发执行下，输出结果有多种可能性。

【练习】两个进程并发时，可能的输出结果有几种？分别有哪些结果？（不考虑 A、B 的输出顺序）

解：这两个进程并发时可能的输出结果是下列 4 组结果之一：

1. P1 输出 A=5，P2 输出 B=5。
2. P1 输出 A=5，P2 输出 B=6。
3. P1 输出 A=6，P2 输出 B=5。
4. P1 输出 A=6，P2 输出 B=6。

【练习】有两个并发进程 P1 和 P2，其程序代码如下：

```c
P1(){
    x=1;    //A1
    y=2;
    z=x+y;
    print z;    //A2
}
P2(){
    x=-3;   //B1
    c=x*x;
    print c;    //B2
}
```

> 可能打印出的 z 值有（ ），可能打印出的 c 值有（ ）（其中 x 为 P1，P2 的共享变量）
> A. z=1,-3; c=-1;9      B. z=-1,3; c=1,9      C. z=-1,3,1; c=9      D. z=3; c=1,9
> 答案：B

### 临界区的四个组成部分

若能保证一次仅允许一个进程使用临界资源，即保证只有一个进程进入临界区，就不会产生错误的结果。因此，为了保证临界资源的正确使用，可将临界资源的访问过程分成 4 个部分：

```c
while (true) {
    进入区
    临界区
    退出区
    剩余区
}
```

**① 进入区（entry section）** 
每个进程在进入临界区之前，应先对欲访问的临界资源进行检查，看它是否正在被访问。如果此刻该临界资源未被访问，进程便可进入临界区对该资源进行访问，并将访问标志设置为"正被访问"；如果此刻该临界资源正在被某进程访问，则本进程不能进入临界区。

**② 临界区（critical section）** 
进程中访问临界资源的那段代码，又称临界段。

**③ 退出区（exit section）** 
将正在访问临界区的标志清除。

**④ 剩余区（remainder section）** 
代码中的其余部分。

> A. 磁盘存储介质      B. 公用队列      C. 私用数据      D. 可重入的程序代码
