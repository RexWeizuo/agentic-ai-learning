# 六、PV 大题

408 真题往往以**大题形式**考察信号量相关的经典问题，大题并非一轮重点，因此仅以"生产者-消费者问题"讲解 PV 大题的基本思路，"读者写者"问题见 **七、管程.md** 补充部分，"哲学家进餐"问题放二轮学习。

## （一）生产者-消费者问题

### 场景描述

一组生产者和一群消费者一起工作，通过一个大小为 n 的有限缓冲区进行生产和消费。一个缓冲区可以容纳 n 个产品，其中生产者负责投放产品，消费者负责消费产品。

由于生产者和消费者共享的缓冲区的大小为 n，缓冲区最多可以容纳 n 个产品：

- 如果缓冲区**已满**，生产者不能继续生产，否则会使产品溢出
- 如果缓冲区**为空**，消费者不能继续消费，否则会导致从空缓冲区拿取产品

### 解题五步法

#### ① 找出各类进程和各类资源

两类进程：生产者进程和消费者进程。
一类资源：大小为 n 的缓冲区。

#### ② 分析同步、互斥关系

- **互斥关系**：生产者和消费者均需要互斥访问缓冲区
- **同步关系**：
  - 缓冲区有产品，消费者才能取
  - 缓冲区有空位，生产者才能生产

#### ③ 定义信号量，确定信号量初值

```c
semaphore mutex = 1;  // 互斥使用缓冲区
semaphore full = 0;   // 表示缓冲区的产品数量（也可以看作非空闲区数量）
semaphore empty = n;  // 表示缓冲区的空闲区数量
```

#### ④ 写出伪代码

```c
semaphore mutex = 1, empty = n, full = 0;

void producer () {
    while (TRUE) {
        生产一个新的产品;
        P(empty);         // 判断是否有空闲的缓冲区，若无则加入等待队列
        P(mutex);         // 对缓冲区上锁
        将产品放入空闲缓冲区;
        V(mutex);         // 对缓冲区解锁
        V(full);          // 产品数量加 1，并唤醒等待队列中的消费者
    }
}

void consumer () {
    while (TRUE) {
        P(full);          // 判断是否有产品，若无则加入等待队列
        P(mutex);         // 对缓冲区上锁
        从缓冲区取出一个产品;
        V(mutex);         // 对缓冲区解锁
        V(empty);         // 空闲区数量加 1，并唤醒等待队列中的生产者
        生产该产品;
    }
}
```

#### ⑤ 检查是否 PV 成对，检查是否有死锁

PV 成对，无死锁，符合题意。

### 经典问题：为什么 P(empty) 在 P(mutex) 之前？

关键在于避免**死锁**。如果交换顺序：

```c
// 错误顺序
P(mutex);     // 先上锁
P(empty);     // 再检查空位 → 若缓冲区已满，当前进程阻塞等待空位
// 但锁在自己手里！其他进程也无法进入缓冲区消费产品释放空位
// → 死锁！
```

正确的顺序是：**先检查资源是否可用（P(empty/P(full)），再上锁（P(mutex)）** 。

## （二）解题原则总结

1. **找进程类型 &amp; 资源类型**：明确有几类进程在竞争/协作
2. **分析关系**：互斥？同步？还是两者都有？
3. **定义信号量**：互斥信号量初值 1，同步信号量初值 0（或 n）
4. **写伪代码**：先 P 再 V，注意 P 的顺序防止死锁
5. **验证**：
   - 检查 PV 是否成对
   - 检查是否有死锁风险
   - 检查是否符合题意

**核心技巧**："找同步关系"的核心是**找事件之间的关系**，某个事件导致某个事件，而不是找"某个进程导致某个进程"，直接找进程之间的关系往往无从下手。
