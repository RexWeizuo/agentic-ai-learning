# 四、信号量

前文所讲的软件硬件方法都无法满足"让权等待"准则。

信号灯是人类社会中应用于交通管理等领域的一种设备，人们可以利用信号灯的状态（颜色）来规范相关活动。**信号量机制**类似于信号灯，通过在几个进程之间使用简单的变量来实现同步。

## （一）信号量的类型

信号量机制在长期应用中得到了很大的发展，从**整型信号量**、**记录型信号量**、**AND 型信号量**，最终发展为**信号量集**。本小节重点学习整型信号量和记录型信号量。

### 信号量的两种操作：P/V 原语

除初始化外，信号量仅能通过两个标准的原子操作来访问，即 **wait(S)**  和 **signal(S)**  操作。这两个操作又分别被称为 **P 操作** 和 **V 操作**。

## （二）整型信号量

### 定义

整型信号量代表一个用于表示资源数目的整数 S，S 指代的是操作系统中某资源的数量。

### wait(S) 操作（P 操作）

```c
wait ( S ) {
    while (S <= 0); // 不符合条件则陷入"忙等"
    S--;
}
```

wait(S) 原语实现 S 值的减 1，减 1 代表申请并获取一个资源。如果 S≤0，操作系统无法为其分配资源，该进程就会陷入 while 循环，直到 S>0，方可结束等待。

### signal(S) 操作（V 操作）

```c
signal (S) {
    S++;
}
```

signal(S) 原语实现 S 数值的加 1，加 1 相当于使用结束并释放一个资源。

### 问题：依然忙等

整型信号量中，如果 S≤0 时，wait(S) 操作会陷入 **&quot;忙等&quot;** 状态，违反了"让权等待"准则。

## （三）记录型信号量

### 结构体定义

为了实现"让权等待"准则，记录型信号量在整型信号量的基础上，增加了一个**被阻塞进程的队列**，用于链接所有等待相应资源的进程。

```c
typedef struct semaphore {
    int s;               // 整型信号量
    struct process *queue; // 用于记录被阻塞进程的队列
} semaphore ;
```

### wait(S) 操作

```c
wait(semaphore S) {
    S.s--;
    if(S.s < 0) 阻塞该进程并将其放入阻塞队列中;
}
```

### signal(S) 操作

```c
signal (semaphore S) {
    S.s++;
    if(S.s <= 0) 从阻塞队列中选择并唤醒一个进程;
}
```

### 信号量值的意义

- 若信号量的值**小于 0**，说明有进程在等待这种资源，信号量的值的**绝对值**就是等待该资源的进程数
- 若信号量的值**等于 0**，说明该资源恰好被分配完

### 创作信号量的方式

一般来说，若要创建一个记录型信号量，按照整型信号量进行声明和定义即可。

```c
semaphore printer = 8; // 声明一个记录系统中打印机资源的信号量
semaphore mutex = 1;   // 声明一个互斥信号量
```

其中，一个信号量对应一种资源，信号量的值等于该资源的剩余数量。互斥信号量的取值一般为 1。

## （四）整型 vs 记录型对比

|对比维度|整型信号量|记录型信号量|
| :-------| :------------------------| :--------------------------|
|**是否让权**|❌ 忙等，不满足"让权等待"|✅ 阻塞入队，满足"让权等待"|
|**是否有阻塞队列**|❌ 无|✅ 有 queue 队列|
|**S &lt; 0 的含义**|无意义（无法进入）|绝对值 = 等待进程数|
|**wait 操作**|while 忙等 + S--|S.s--，<0 则阻塞入队|
|**signal 操作**|S++|S.s++，≤0 则唤醒队列|

**关键区别**：记录型信号量的队列，相当于给等待的进程安排了一个"候车室"——没资源了就去候车室睡觉，等别人用完再叫你。而整型信号量就像让等待的人在门口**原地蹦迪**（空转），既累又没用。

## 🧠 深度理解

### wait 和 signal 为什么这样命名？（来自 DS 问答）

你可能觉得"wait 是申请资源"听起来像是"发信号给系统要资源"，而"signal 是释放资源"像是"等待系统回收资源"。但实际上：

**wait(S) = 申请资源（P 操作）= S--**

```c
wait(S) {
    while (S <= 0);   // 没资源了？那就在这"等待"
    S--;               // 拿到一个资源，资源数减1
}
```

- 为什么叫 wait？因为如果资源不够，进程就要**等待**，直到有资源可用

**signal(S) = 释放资源（V 操作）= S++**

```c
signal(S) {
    S++;               // 归还一个资源，资源数加1
}
```

- 为什么叫 signal？因为你归还资源后，要**发个信号**告诉正在等待的人："嘿！有资源了，快来！"

**借书类比：**

```
图书馆有 5 本书（S=5）

你去借书（执行 wait）：
  wait(S):
    while (S <= 0);   // 没书了？等着吧
    S--;               // 借走一本，剩余数减1

你还书（执行 signal）：
  signal(S):
    S++;               // 还回一本书，S从0变成1
  此时正在等待的人收到"信号"，就能借到书了！
```

**记忆技巧：**

- **wait（等待）→ 想要东西 → 等别人给 → 拿到后资源数减1**
- **signal（信号）→ 还东西 → 发信号通知别人 → 资源数加1**

### P(mutex) 和 wait(mutex) 的关系

`P(mutex)` 完全等价于 `wait(mutex)`。

|名称|全称来源|含义|
| :---| :--------------| :--------------------------|
|**P(S)**|荷兰语 **Proberen**（尝试）|= wait(S) = 申请资源，S--|
|**V(S)**|荷兰语 **Verhogen**（增加）|= signal(S) = 释放资源，S++|

```c
semaphore mutex = 1;   // 资源数 = 1，相当于一把锁

进程 A:
    P(mutex);          // = wait(mutex): mutex-- → mutex=0，拿到了锁 ✅
    临界区;
    V(mutex);          // = signal(mutex): mutex++ → mutex=1，释放了锁

进程 B:
    P(mutex);          // = wait(mutex): mutex-- → mutex=-1 < 0，被阻塞 ⛔
    // 等A执行V(mutex)后，被唤醒才能进来
```
