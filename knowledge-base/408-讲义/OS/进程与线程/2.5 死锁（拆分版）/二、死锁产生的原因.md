# 二、死锁产生的原因

二、死锁产生的原因（理解即可）

死锁的起因通常包含以下三个要素，分别是对不可抢占资源进行争夺、对可消耗资源进行争夺以及进程推进顺序不当。

## （一）竞争不可抢占资源引起死锁

不可抢占资源是指，一旦系统把这类资源分配给某进程后，就不能将它强行收回，而只能在进程用完后等待其自行释放。

【例】当一个进程已开始刻录光盘时，如果突然将刻录机分配给另一个进程，则结果必然会损坏正在刻录的光盘，因此只能等刻录好光盘后，由进程自己释放刻录机。

[图片：光驱]

另外，磁带机、打印机等也都属于不可抢占资源。

--- Page 118 ---

[图片：打印机图片]

通常，系统中所拥有的不可抢占资源的数量不足以满足多个进程运行的需要，这使得进程在运行过程中，会因争夺资源而陷入僵局。

【例】系统中有两个进程 P1 和 P2，他们都准备写两个文件 F1 和 F2，这两个文件都属于不可抢占资源。进程 P1 先打开文件 F1，后打开文件 F2；进程 P2 先打开文件 F2，后打开文件 F1，下面给出了对应的代码。

```

P1                  P2

......              ......

open (F1,w);        open (F2, w);

open (F2, w);       open (F1,w);

```

[图片：进程资源分配图，左侧显示P1请求F1和F2；右侧显示P1持有F1请求F2，P2持有F2请求F1，形成死锁]

两个进程 P1 和 P2 在并发执行时，如果 P1 先打开 F1 和 F2，然后 P2 才去打开 F2，则由于文件 F2 已被 P1 打开，故 P2 会被阻塞。

当 P1 写完文件 F1、F2 并将其关闭时，P2 会由阻塞状态转为就绪状态，并被调度执行后实现打开文件 F1、F2。在这种情况下，P1 和 P2 都能正常运行下去。

但如果在 P1 打开 F1 的同时，P2 去打开 F2，每个进程都占有一个打开的文件，此时就可能会出现问题。因为当 P1 试图去打开 F2，而 P2 试图去打开 F1 时，这两个进程都会因文件已被打开而阻塞，它们都希望对方关闭自己所需要的文件，但谁都无法进行该操作，因此这两个进程将会无限期地等待下去，进而形成死锁。

## （二）竞争可消耗资源引起死锁

可消耗资源指进程运行期间动态创建和消耗的资源，通常是由生产者进程创建、由消费者进程消耗的，最典型的可消耗资源就是用于进程间通信的消息。

【例】假设 P1、P2、P3 三个进程通过消息机制通信，如下所示。

[图片：进程通信循环图，P1发送m1给P2，P2发送m2给P3，P3发送m3给P1]

P1 产生消息 m1，利用 send (P2, m1) 原语将它发送给 P2。

P2 产生消息 m2，利用 send (P3, m2) 原语将它发送给 P3。

P3 产生消息 m3，利用 send (P1, m3) 原语将它发送给 P1。

三个进程分别利用 receive(Pi,mj)原语接收消息。

如果这 3 个进程间的消息通信按下列步骤进行：

P1 : ...send (P2, m1); receive (P3, m3); ...

P2 : ...send (P3, m2); receive (P1, m1); ...

P3 : ...send (P1, m3); receive (P2, m2); ...

那么，这3个进程都可以先将消息发送给下一个进程，也都能够接收到从上一个进程发来的消息。因此，这3个进程可以顺利地运行下去，而不会发生死锁。

但若改成3个进程都先执行 receive 操作，后执行 send 操作，即按下列步骤进行：

P1 : ...receive (P3, m3); send (P2, m1); ...

P2 : ...receive (P1, m1); send (P3, m2); ...

P3 : ...receive (P2, m2); send (P1, m3); ...

那么，这3个进程就会永远阻塞在它们的 receive 操作上，等待一条永远不会发出的消息，于是发生死锁。

## （三）进程推进顺序不当引起死锁

不难发现，进程推进顺序不当，也是产生死锁的一个重要因素，上述的两个例子中都体现了该因素。

【练习】假设系统中有 m (m≥1) 个进程，每个进程都需要 k (k≥1) 个相同类型的临界资源，那么确保系统不出现死锁的最小资源数是多少？

解：考虑极端情况：假设每个进程被分配了(k-1)个资源，若此时系统资源耗尽，将会发生死锁。

但只需要一个额外资源，就可以帮助其中某个进程完成任务，并释放其占有的资源，以便其他进程运行所需。

因此，确保系统不出现死锁的最小资源数是(k-1)m+1。

### 一、单选题

**1. 根据文档，死锁的起因通常不包括以下哪个要素？**​**IV**  
A. 竞争不可抢占资源  
B. 竞争可消耗资源  
C. 进程推进顺序不当  
D. 进程优先级错误

**答案：D**  
**详解：**   
文档明确列出死锁的起因包含三个要素：“分别是对不可抢占资源进行争夺、对可消耗资源进行争夺以及进程推进顺序不当”。优先级错误虽然可能导致饥饿，但并非文档指出的死锁起因三要素之一。

根据文档《二、死锁产生的原因》中的定义：

---

### 可消耗资源

**定义：**  可消耗资源是指进程运行期间**动态创建和消耗**的资源，通常由**生产者进程创建、由消费者进程消耗**。

**最典型的例子：**  用于进程间通信的**消息**。

---

### 文档中的具体例子

三个进程 P1、P2、P3 通过消息机制通信：

- P1 产生消息 m1，发送给 P2
- P2 产生消息 m2，发送给 P3
- P3 产生消息 m3，发送给 P1

消息 m1、m2、m3 就是可消耗资源——它们由发送方进程创建（生产者），由接收方进程消耗（消费者），一旦被接收消费就不复存在。

---

### 与不可抢占资源的区别

|对比维度|不可抢占资源|可消耗资源|
| ----------| ----------------------------------| --------------------------------|
|存在方式|系统中固定存在的设备/数据|运行期间动态创建和消耗|
|典型例子|打印机、刻录机、磁带机、文件|消息、信号量等|
|释放方式|进程用完后自行释放|被消费者接收后消耗消失|
|死锁场景|进程持有并等待对方占有的设备资源|进程等待一条永远不会发出的消息|

---

**简单来说：**  可消耗资源不是系统里固定存在的硬件设备，而是进程在运行过程中临时创建、供其他进程使用的"一次性"资源，用完了就消失了。消息就是最常见的可消耗资源。

**2. 以下哪个设备属于不可抢占资源？**   
A. CPU  
B. 内存  
C. 打印机  
D. 高速缓存

**答案：C**  
**详解：**   
文档举例：刻录机、磁带机、打印机等都属于不可抢占资源。CPU 和内存可被操作系统剥夺和重新分配，属于可抢占资源（或至少可被重新调度和换出）。打印机在被一个进程占用期间如果强行分配给另一个进程会造成输出混乱，因此属于不可抢占资源。

---

**3. 进程 P1 和 P2 按以下顺序执行时，哪个顺序会导致死锁？ II**  
代码逻辑：  
P1 先打开 F1，后打开 F2  
P2 先打开 F2，后打开 F1  
A. P1 打开 F1 和 F2 完成后，P2 才开始打开 F2  
B. P1 打开 F1 的同时，P2 打开 F2，然后两者继续请求另一个文件  
C. P2 打开 F2 和 F1 完成后，P1 才开始打开 F1  
D. P1 放弃打开 F2，只打开 F1

**答案：B**  
**详解：**   
文档分析：若 P1 和 P2 顺序执行（A、C 情况），由于先执行的进程释放资源后另一个再执行，不会死锁。若同时 P1 占有 F1、P2 占有 F2，然后 P1 请求 F2、P2 请求 F1，则形成循环等待，两者均阻塞，导致死锁。D 情况放弃资源也不符合死锁的“持有并等待”必要条件。

---

**4. 关于可消耗资源，以下描述正确的是：II**  
A. 可消耗资源是永久存在的设备资源  
B. 消息是可消耗资源的一个典型例子  
C. 打印机属于可消耗资源  
D. 可消耗资源不能被动态创建

**答案：B**  
**详解：**   
文档定义：“可消耗资源指进程运行期间动态创建和消耗的资源，通常是由生产者进程创建、由消费者进程消耗的，最典型的可消耗资源就是用于进程间通信的消息。” 打印机是不可抢占资源（设备），属于持久资源；可消耗资源可以动态创建和消耗。

---

**5. 三个进程 P1、P2、P3 通过消息通信，以下哪种执行顺序会发生死锁？**​**II**  
A. 每个进程先 send 后 receive  
B. 每个进程先 receive 后 send  
C. P1 先 send，P2 先 receive，P3 先 send  
D. 交替执行，无固定顺序

**答案：B**  
**详解：**   
文档给出两个例子：先 send 后 receive 可顺利运行；但若三个进程都先执行 receive 操作，再执行 send 操作，每个进程都在等待其他进程发来消息，而其他进程也在等待接收消息，不会执行 send，导致所有进程永远阻塞在 receive 上，形成死锁。

**6. 系统中有 3 个进程，每个进程需要 4 个同类临界资源，保证系统不死锁的最小资源数是：I**  
A. 10  
B. 11  
C. 12  
D. 13

**答案：A**  
**详解：**   
文档中给出公式：确保系统不出现死锁的最小资源数为  **(k-1)m + 1**。  
此处 m=3, k=4，代入得 (4-1)×3 + 1 = 3×3 + 1 = 10。  
推导思路：最坏情况每个进程分配到 (k-1)=3 个资源，此时还需要 1 个额外资源使某个进程完成任务并释放全部资源。

### 二、判断题

**7. 不可抢占资源被分配给进程后，操作系统可以在需要时强制收回，以便分配给其他进程。 （ ）**   
**答案：×**   
**详解：**   
文档定义：“不可抢占资源是指，一旦系统把这类资源分配给某进程后，就不能将它强行收回，而只能在进程用完后等待其自行释放。” 因此操作系统不能强制收回不可抢占资源。

---

**8. 如果系统中所有进程都先申请资源、再运行，执行完毕后才释放资源，一定不会发生死锁。 （ ）**   
**答案：×**   
**详解：**   
这其实就是打破“持有并等待”条件的一种方法，如果一次性申请所有资源，理论上不会产生死锁。但文档中的例子显示：如果进程申请资源的顺序不当（如 P1 先开 F1 再开 F2，P2 先开 F2 再开 F1），即使都是申请，也可能在持有部分资源后请求被其他进程占用的资源，从而死锁。所以“先申请资源”并不能保证安全，除非是一次性申请全部所需资源。

---

**9. 进程推进顺序不当是引起死锁的一个重要因素，文档中的两个例子都涉及该因素。 （ ）**   
**答案：√**   
**详解：**   
文档原文：“进程推进顺序不当，也是产生死锁的一个重要因素，上述的两个例子中都体现了该因素。” 无论是不可抢占资源的竞争例子，还是可消耗资源的消息通信例子，都是因为并发进程的推进顺序（交错执行时机）不当而导致了死锁。

### 三、简答题

**10. 简述不可抢占资源与可消耗资源在引起死锁时的不同机制。**

  
**答案要点：**

- **不可抢占资源**：每个进程占有某些资源后，又去申请其他进程已占有的资源，而进程不能强行夺取对方资源，只能等待对方释放；由于对方也在等待，形成循环等待而死锁。典型例子：P1 占 F1 请求 F2，P2 占 F2 请求 F1。
- **可消耗资源**：资源是动态创建和消耗的（如消息）。进程在等待一条尚未产生且永远不会产生的消息时阻塞，而能够产生消息的进程又因为先执行了 receive 而陷入等待，所有进程都无法推进，形成死锁。

两者核心都是循环等待，但不可抢占资源竞争的是静态存在的设备或数据，可消耗资源竞争的是动态消息的发送与接收顺序。

---

**11. 用文档中 P1、P2 打开文件的例子，解释进程推进顺序如何影响死锁的产生。**   
**答案要点：**   
如果进程按顺序执行（P1 全部完成后再运行 P2），不会发生死锁，因为 P1 会释放文件，P2 随后可以获取。但如果 P1 打开 F1 的同时 P2 打开 F2，然后两者分别请求对方已占有的文件，由于两个进程都阻塞等待对方释放，而谁也无法释放，从而形成死锁。这体现了并发进程交替执行的推进顺序（交错）是导致死锁的关键因素之一。

### 四、计算题

**12. 系统中有 m 个进程，每个进程需要 k 个同类资源，每个进程采用“申请一部分资源、运行、再申请剩余资源”的方式。若系统总资源数为 R，请推导系统不死锁所需的最小 R 值，并写出推导过程。**   
**答案详解：**   
考虑最极端情况：所有进程都获得了 (k-1) 个资源，而系统已经没有多余资源可用。此时每个进程都还需要 1 个资源才能完成任务，但谁也无法得到，形成死锁。  
为了避免死锁，系统至少要多出 1 个资源，使其中一个进程能得到第 k 个资源，从而运行完毕并释放所占用的 k 个资源，随后这些资源可以分配给其他进程继续运行。  
因此不死锁的最小资源数 = m × (k-1) + 1。  
即 **R_min = m(k-1) + 1**。
