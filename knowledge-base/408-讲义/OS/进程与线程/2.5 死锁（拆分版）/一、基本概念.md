# 一、基本概念

一、基本概念（理解即可）

## （一）死锁的定义

死锁指一组并发进程中的每一个进程都等待其余进程中的某个进程拥有的资源，且这些并发进程在得到自己需要的资源之前不会释放自己拥有的资源，从而造成所有并发进程都想得到资源而又都得不到资源，均不能继续向前推进的状态。

[图片：两只猫分别拿着筷子和碗，对话框显示“我要吃饭！”]

死锁、饥饿、死循环都是进程无法顺利向前推进的现象，但有以下区别：

## （二）死锁与饥饿、死循环的区别

① 死锁是 n 个进程 (n≥2) 循环等待对方手里的资源，导致 n 个进程都被阻塞，均无法向前推进的现象。

② 饥饿是进程由于长期未获得想要的资源，从而无法向前推进的现象。

“资源”可以是 CPU——饥饿进程长期处于就绪态，也可以是 I/O 设备——饥饿进程长期处于阻塞态。

③ 死循环是进程在执行过程中，一直进行某个循环，而不跳出的现象。

④ 死锁和饥饿是由于资源的不合理分配引起，而死循环是由于程序的设计错误引起。

   - ​**破坏“持有且等待”** ：要求进程在开始执行前一次性申请所有所需资源，如果申请不到则等待，但不持有部分资源。或者当新资源申请不到时，主动释放已占资源，再重新尝试。

### 一、单选题

**1. 死锁发生时，至少涉及几个进程？**   
**答案：B. 2个**  
**详解：**   
根据文档定义：“死锁指一组并发进程中的每一个进程都等待其余进程中的某个进程拥有的资源”，并且“死锁是 n 个进程 (n≥2) 循环等待对方手里的资源”。死锁的本质是多个进程之间形成资源等待的闭环，最少需要两个进程互相等待对方持有的资源，因此 n 至少为 2。只涉及一个进程的情况不可能形成循环等待，不属于死锁（可能是饥饿或死循环）。

---

**2. 以下哪种现象是由于资源分配不合理造成的？**   
**答案：B. 死锁**  
**详解：**   
文档明确指出：死锁和饥饿是由于资源的不合理分配引起，而死循环是由于程序的设计错误引起。选项中的 A（死循环）和 C（程序逻辑错误）、D（数组越界）均属于程序的逻辑/编码错误，只有死锁属于资源分配不合理导致的现象。饥饿也同样由资源分配不合理引起，但选项中未出现。

---

**3. 一个进程长期处于就绪态，但一直得不到 CPU，这属于：**   
**答案：B. 饥饿**  
**详解：**   
文档解释：饥饿是进程由于长期未获得想要的资源，从而无法向前推进的现象。资源可以是 CPU —— 饥饿进程长期处于就绪态（等待 CPU 调度）。而死锁要求多个进程互相等待且都被阻塞，不会只是单个进程处于就绪态；死循环是程序一直在循环体内执行，通常占用 CPU 并不处于就绪等待。

---

**4. 关于死锁、饥饿、死循环，下列说法正确的是：**   
**答案：C. 死锁是所有进程都想得到资源但又都得不到资源的状态**  
**详解：**

- A 错误：死循环是程序设计错误引起，不是资源分配不合理。
- B 错误：饥饿是资源分配不合理引起，不是程序设计错误。
- C 正确：文档定义“从而造成所有并发进程都想得到资源而又都得不到资源，均不能继续向前推进的状态”正是对死锁的描述。
- D 错误：饥饿不一定永久，例如低优先级进程长期得不到 CPU，但一旦高优先级结束它仍可能运行；而文档并未断言饥饿一定无法恢复。死锁中的进程如果外界不干预可能永远阻塞，但该选项说法太绝对且混为一谈。

### 二、判断题

**5. 死锁中的每个进程都不愿意释放自己已获得的资源，必须等待其他进程释放资源后自己才可能继续运行。 （√）**   
**详解：**   
文档指出：“这些并发进程在得到自己需要的资源之前不会释放自己拥有的资源”，这导致进程都占有部分资源并等待其它资源，人为无法推进。因此进程确实不会主动释放资源，必须等待其他进程释放，但死锁状态下谁都无法释放，所以大家都无法继续运行。陈述符合定义。

---

**6. 进程因为等待 I/O 设备而长期阻塞，这种情况一定是死锁。 （×）**   
**详解：**   
文档说明：饥饿的“资源”可以是 I/O 设备，饥饿进程长期处于阻塞态。单个进程等待 I/O 设备长期得不到响应，可能是由于资源分配不合理造成的饥饿，但不构成死锁。死锁必须涉及多个进程之间循环等待彼此的资源。因此不一定是死锁。

### 三、简答题

**7. 请用自己的话解释死锁的定义，并说明死锁、饥饿、死循环三者的主要区别是什么？**   
**参考答案要点：**   
死锁定义：多个进程在执行过程中，各自持有一些资源，并互相等待对方持有的资源释放，且每个进程在获得所需全部资源前都不会释放已占资源，导致所有涉及的进程都被阻塞，无法继续执行。

主要区别：

- **死锁**：发生在两个或以上进程之间，循环等待对方占有的资源，所有进程都被阻塞。
- **饥饿**：单个进程长期得不到所需资源（如 CPU、I/O 等），可能由于调度策略或资源分配不公平造成，进程本身可能仍处于就绪或阻塞，但不涉及循环等待。
- **死循环**：程序内部逻辑错误导致一直在某段代码循环执行，不涉及资源争夺或多个进程等待，该进程可能一直占用 CPU。
- 根源不同：死锁和饥饿通常由资源分配不当造成，死循环源自程序设计错误。

---

**8. 文档中用两只猫吃饭的图片形象地比喻了死锁，请简要描述这幅图是如何体现死锁特征的。**   
**参考答案要点：**   
图中每只猫都占用了一部分“吃饭”所需要的资源：比如一只猫拿着筷子，另一只猫拿着碗。要吃饭必须同时拥有筷子和碗。每只猫都在等待对方释放手中的资源，但都不会主动放弃自己已有的资源，于是两只猫都无法开始吃饭，陷入僵持状态。这形象地表现了死锁的四个关键要素：互斥（资源一次只能被一只猫使用）、持有并等待（占有一部分资源并等待其他）、不剥夺（猫不会强行抢对方资源）、循环等待（猫A等猫B释放碗，猫B等猫A释放筷子），完全符合死锁特征。

**9. 如果让你设计一个简单的资源分配系统，你会在哪些环节特别留意以避免死锁的发生？请结合死锁的定义谈谈你的想法。**   
**参考思路：**   
结合死锁产生的必要条件（互斥、持有等待、不可剥夺、循环等待），可以从以下几个方面入手：

- **破坏“持有且等待”** ：要求进程在开始执行前一次性申请所有所需资源，如果申请不到则等待，但不持有部分资源。或者当新资源申请不到时，主动释放已占资源，再重新尝试。

- **死锁检测与恢复**：如果无法完全预防，可定时检测资源分配图是否存在环，若有则通过终止进程或回滚操作来解除死锁。
