# 3.4 虚拟分页管理 · 题7

7. 为判断一个请求分页存储管理系统的运行性能，对系统中各设备的利用率进行测试，结果如下 CPU 利用率为 $13\%$，磁盘交换区利用率为 $99.8\%$，其他 IO 设备利用率为 $4\%$。以下措施中可以改进 CPU 利用率的是（）。I

I . 使用容量更大的内存  
II . 减少系统多道程序度数  
III . 增加多道程序度数  
IV . 增大磁盘交换区的容量  
V . 使用存取速度更快的磁盘交换区  
VI . 使用处理速度更快的 CPU

A. I、II  
B. I、III  
C. II、III、V、VI  
D. I、II、III、IV

I III IV V(VI V对CPU利用率的改善不大？)

### 第7题

**题目：**  CPU 利用率 13%，磁盘交换区利用率 99.8%，其他 I/O 设备 4%。可改进 CPU 利用率的是（ ）。

**答案：A. I、II**

**解析：**

磁盘交换区利用率 99.8%，CPU 只有 13% → 这是典型的\*\*抖动（Thrashing）\*\*现象：

> 内存中进程过多，每个进程分到的物理页框太少，频繁缺页 → CPU 大部分时间在等待磁盘 I/O

**I. 使用容量更大的内存 ✅**  — 更多页框分配给进程，降低缺页率

**II. 减少多道程序度数 ✅**  — 减少并发进程数，每个进程分到更多页框，缺页率下降

|措施|判断|理由|
| -----------------| :----: | ----------------------------------------------|
|I. 加大内存|✅|更多物理页框 → 驻留集可增大 → 缺页率降低|
|II. 减少并发度|✅|进程数减少 → 每个进程分到更多页框|
|III. 增加并发度|❌|火上浇油，抖动更严重|
|IV. 增大交换区|❌|是内存不够导致频繁换页，加大交换区不解决问题|
|V. 更快的磁盘|❌|治标不治本，缺页次数没减少|
|VI. 更快的 CPU|❌|CPU 本来就闲置，再快也是等着|

### 第8题（2014统考真题）

**题目：**  能加快虚实地址转换的是（ ）。

**答案：C. 仅 I、II**

**解析：**

**I. 增大快表（TLB）容量 ✅**

TLB 越大 → 命中率越高 → 更多地址转换在 TLB 中完成，不用访问内存中的页表。

**II. 让页表常驻内存 ✅**

如果页表本身还可能被换出，访问页表时先要缺页中断把页表页调入 → **页表缺页**。页表常驻内存避免这种情况，加快转换。

**III. 增大交换区（swap）❌**

交换区影响的是**缺页时从何处调入页面**的速度，不影响**虚实地址转换**这一步骤。

|措施|影响环节|是否加速地址转换|
| ------------------| :--------------------: | :----------------: |
|I. 增大 TLB|地址映射（第一步）|✅|
|II. 页表常驻内存|访问页表|✅|
|III. 增大交换区|缺页处理（换入换出）|❌ 不相关|

### 第9题（2015统考真题）

**题目：**  页面分配策略与页面置换策略不能组合使用的是（ ）。

**答案：C. 固定分配 + 全局置换**

**解析：**

文档给出了三种有效组合：

|组合|是否可行|理由|
| ---------------------| :--------: | --------------------------------------------------------------------|
|固定分配 + 局部置换|✅|驻留集固定，缺页时从自己的页框内置换|
|可变分配 + 全局置换|✅|缺页时从系统空闲队列拿新框，驻留集可变|
|可变分配 + 局部置换|✅|缺页率调整驻留集大小，置换在自己内部|
|**固定分配 + 全局置换**|❌|矛盾！固定分配下每个进程页框数是锁定的，不可能从全局拿空闲框给进程|

 **"全局置换"意味着进程可以从其他进程那里"抢"页框，但"固定分配"说每个进程的页框数死板不变 → 逻辑矛盾，无法并存。**

### 第10题（2023统考真题）

**题目：**  关于进程虚拟地址空间，错误的是（ ）。

**答案：D. 虚拟地址空间的大小由内存和硬盘的大小决定**

**解析：**

|选项|判断|理由|
| -------------------------------| :----: | --------------------------------|
|A. 每个进程有独立虚拟地址空间|✅|各进程地址空间隔离|
|B. malloc() 返回虚拟地址|✅|用户态程序永远只看到虚拟地址|
|C. 数据段和代码段可不同权限|✅|代码段只读可执行，数据段可读写|
|**D. 由内存+硬盘决定**|❌|由**地址结构（CPU 位数）** 决定|

这题和第 2 题本质相同——虚拟地址空间大小 = 2^地址位数，与硬件配置无关。32 位系统不管配多大内存+硬盘，每个进程的虚拟地址空间始终是 4GB。
