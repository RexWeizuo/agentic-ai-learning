# 3.2 内存的分区管理 · 题5

5. 某个计算机采用动态分区分配内存, 经过一段时间的运行, 现在内存中按地址从小到大存在以下空闲分区: $100 \mathrm{~KB} 、 250 \mathrm{~KB} 、 150 \mathrm{~KB} 、 200 \mathrm{~KB}$ 和 $390 \mathrm{~KB}$ 。分配指针指向地址起始点。接下来, 系统会收到以下进程的内存申请: $120 \mathrm{~KB} 、 180 \mathrm{~KB} 、 110 \mathrm{~KB}$ 和 $280 \mathrm{~KB}$ 。那么, 不能够完全完成所有内存分配任务的算法是 ()。III  
   A. 首次适应算法  
   B. 最佳适应算法  
   C. 最坏适应算法  
   D. 邻近适应算法

   I 250-120; 200-180; 150-110; 390-280  
   II 150-120; 200-180; 150-110; 390-280  
   III  
   IV 邻近适应算法是什么鬼（循环首次适应算法）

## 解析

**题目：**  空闲分区 100KB、250KB、150KB、200KB、390KB。申请序列 120KB、180KB、110KB、280KB。不能完全完成所有分配任务的算法是（）。

**答案：C. 最坏适应算法**

**解析：**  逐一模拟四种算法：

**首次适应算法 ✅ 能完成**

```
空闲: 100, 250, 150, 200, 390
120KB → 选250，剩130  空闲: 100, 130, 150, 200, 390
180KB → 选200，剩20   空闲: 100, 130, 150, 20, 390
110KB → 选130，剩20   空闲: 100, 20, 150, 20, 390
280KB → 选390，剩110  全部完成 ✓
```

**最佳适应算法 ✅ 能完成**

```
空闲: 100, 150, 200, 250, 390（按大小排）
120KB → ≥120最小是150，剩30   空闲: 100, 30, 200, 250, 390
180KB → ≥180最小是200，剩20   空闲: 100, 30, 20, 250, 390
110KB → ≥110最小是250，剩140  空闲: 100, 30, 20, 140, 390
280KB → ≥280最小是390，剩110  全部完成 ✓
```

**最坏适应算法 ❌ 不能完成**

```
空闲: 390, 250, 200, 150, 100（从大到小）
120KB → 选最大390，剩270  空闲: 270, 250, 200, 150, 100
180KB → 选最大270，剩90   空闲: 90, 250, 200, 150, 100
110KB → 选最大250，剩140  空闲: 90, 140, 200, 150, 100
280KB → 最大200 < 280 → 无法分配！ ✗
```

最坏适应算法每次都挑最大的切，把大块提前"碎掉"，导致后面大请求无法满足。

**邻近适应算法 ✅ 能完成**

```
120KB → 250，指针移到130
180KB → 200（从130找），指针移到20
110KB → 390（从20循环找），指针移到280
280KB → 280刚好，全部完成 ✓
```

### 第6题（2017统考真题）

**题目：**  回收始址 60K、大小 140KB 的分区后，系统中空闲分区的数量、第一个分区的始址和大小分别是（）。

**答案：B. 3, 500K, 80KB**

**解析：**

**第一步：判断相邻关系**

回收分区始址 60K，大小 140KB → 范围 [60K, 200K)

现有空闲分区：

- 始址 20K，大小 40KB → 范围 [20K, 60K) → **是回收分区的上邻！**
- 始址 200K，大小 200KB → 范围 [200K, 400K) → **是回收分区的下邻！**

**第二步：三区合并**

上邻 40KB + 回收 140KB + 下邻 200KB = **380KB**，始址 **20K**

**第三步：整理空闲分区**

合并后空闲分区为：

|始址|大小|
| -------| -------|
|20K|380KB|
|500K|80KB|
|1000K|100KB|

数量：**3个**（原来4个减为3个）

**第四步：排序（最佳适应，按大小升序）**

|始址|大小|
| -------| -------|
|500K|80KB|
|1000K|100KB|
|20K|380KB|

空闲分区链**第一个分区**：始址 **500K**，大小 **80KB**。

### 第7题（2019统考真题）

**题目：**  最容易产生内存碎片的是（）。

**答案：C. 最佳适应算法**

**解析：**

- **最佳适应算法**：每次选最接近请求大小的空闲分区，导致切完后剩下**极小的碎片**，这些小碎片很难再被利用，长此以往积累大量外部碎片。
- **最坏适应算法**：每次切最大的，剩下来的依然较大，碎片相对少。
- **首次适应/循环首次适应**：碎片情况介于两者之间。

这也是文档中哈希算法尝试解决的问题——按大小快速定位合适分区，实现最佳分配。

### 第8题（2024统考真题）

**题目：**  每次回收分区时仅合并大小相等的空闲分区的是（）。

**答案：A. 伙伴算法**

**解析：**  伙伴系统（Buddy System）的核心规则：

> 只有**互为伙伴**的两个块才能合并，而互为伙伴的前提是**大小相等**且地址相邻（由分裂关系决定）。

例如：两个 4KB 的伙伴块合并成一个 8KB 块，两个 8KB 伙伴块合并成 16KB……大小不相等的块即使地址相邻也不能合并。

其余三种算法（最佳、最坏、首次适应）在回收时只要地址相邻就合并，**不要求大小相等**。
