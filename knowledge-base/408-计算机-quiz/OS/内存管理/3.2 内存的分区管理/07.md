# 3.2 内存的分区管理 · 题7

7. 【2019】在下列动态分区分配算法中，最容易产生内存碎片的是（）。I III  
   A. 首次适应算法  
   B. 最坏适应算法  
   C. 最佳适应算法  
   D. 循环首次适应算法

## 解析

**最佳适应算法**每次选择最接近请求大小的空闲分区，切割后剩下的残余块极小，几乎无法再被利用，长期积累形成大量**外部碎片**。选 C。

   内存碎片-外部碎片

   # 两种碎片

   ---

   ## 一、核心定义

   |类型|定义|通俗理解|
   | ------| ----------------------------------------| ----------------------------|
   |**内部碎片**|已分配给进程的内存中，进程用不到的部分|"给多了，吃不完"|
   |**外部碎片**|空闲内存总量够，但因不连续而无法分配|"零钱够，但凑不成整张整钱"|

   ---

   ## 二、各算法的碎片产生情况

   根据文档中的总结表：

   |分配方式|内部碎片|外部碎片|原因|
   | ----------| :--------: | :--------: | --------------------------------------------|
   |**单一连续分配**|✅ 有|❌ 无|整个用户区给一个进程，进程通常占不满|
   |**固定分区分配**|✅ 有|❌ 无|分区大小预先固定，进程很难刚好填满|
   |**动态分区分配**|❌ 无|✅ 有|按需分配，不浪费；但回收后产生不连续小洞|
   |**伙伴系统**|✅ **较多**|✅ 较少|分配单位必须是 2^k，进程需求往往不是 2的幂|
   |**动态重定位**|❌ 无|可消除|通过紧凑拼接外部碎片|

   ---

   ## 三、容易产生内部碎片的情况

   **根本原因：分配单位 > 实际需求**

   ### ① 固定分区分配（最典型）

   ```
   内存被预划分为：
   ┌────────┐
   │  4KB   │ ← 进程只要 2.5KB，剩下 1.5KB 就是内部碎片
   ├────────┤    这 1.5KB 已经分给进程，但进程用不到
   │  8KB   │
   ├────────┤
   │ 16KB   │
   └────────┘
   ```

   分区大小固定不变，进程只能"对号入座"，几乎不可能刚好填满。

   ### ② 伙伴系统（文档明确指出"内部碎片较多"）

   ```
   进程申请 9KB → 伙伴系统分配 16KB（2^4）
                            ↑
                       多出 7KB 就是内部碎片
   ```

   原因：伙伴系统只能分配 2 的幂次大小（2KB、4KB、8KB、16KB…），实际需求 9KB 得拿 16KB，**最坏情况浪费近一半**。

   ### ③ 单一连续分配

   整个用户区给一个进程，进程很少正好用满。

   ---

   ## 四、容易产生外部碎片的情况

   **根本原因：分配和回收导致空闲空间被打散**

   ### ① 动态分区分配（最典型的外碎片来源）

   ```
   初始状态：一块完整的 200KB 空闲区

   分配 P1(30KB) → 回收 P1：
   ┌───┐──────────┐
   │空闲│  空闲    │  变成两个小空闲区
   │30K │  170K   │
   └───┘──────────┘

   反复分配回收后：
   ┌──┬──┬──┬──┬──┬──┐
   │用│空│用│空│用│空│  虽然空闲总量 120KB 够
   └──┴──┴──┴──┴──┴──┘  但每个空闲块都不够 100KB
                         大进程无法装入 → 外部碎片
   ```

   ### ② 最佳适应算法（文档第7题结论）

   > 最容易产生内存碎片的是 → **最佳适应算法**
   >

   ```
   空闲: 100KB, 150KB, 200KB, 250KB
   申请 98KB → 选 100KB（最接近）

   结果：100KB → 2KB + 98KB（分配）
                   ↑
           剩下 2KB 小碎片，几乎永远用不上
   ```

   每次"精准匹配"，切完后剩下极小残留，累积成大量**用不了的小碎片**。这是外部碎片的最严重形态。

   ---

   ## 五、本质矛盾

   ```
   内部碎片 ←→ 外部碎片

   内碎片：在"分配块内部"浪费      → 固定分配、粗粒度管理
   外碎片：在"分配块之间"浪费      → 动态分配、细粒度管理

   减少内碎片 → 分配更精准 → 产生更多不规则小空闲区 → 增加外碎片
   减少外碎片 → 固定块大小保持规整 → 分配不够精准 → 增加内碎片
   ```

   **不可能两者都为零，只能在不同算法间做权衡。**  文档中提到的"紧凑"技术是少数能**消除外部碎片**而不引入内部碎片的方法，但代价是 CPU 开销大、需要硬件支持动态重定位。
