# 3.3 内存的基本分页-分段管理 · 题9

9. 【2016】某进程的段表内容如下所示。

|段号|段长|内存起始地址|权限|状态|
| ------| ------| --------------| ------| ----------|
|0|100|6000|只读|在内存|
|1|200|-|读写|不在内存|
|2|300|4000|读写|在内存|

访问段号为 2、段内地址为 400 的逻辑地址时, 进行地址转换的结果是 ()。IV

A. 段缺失异常  
B. 得到内存地址 4400  
C. 越权异常  
D. 越界异常

## 解析

**题目：**  访问段号为 2、段内地址为 400 的逻辑地址时，地址转换的结果是（ ）。

**答案：D. 越界异常**

**解析：**

|段号|段长|内存起始地址|权限|状态|
| :----: | :----: | :------------: | :----: | :------: |
|2|**300**|4000|读写|在内存|

```
段内偏移 W = 400
段长 C = 300
W ≥ C → 400 ≥ 300 → 越界中断！
```

文档原文流程："若 W ≥ C，则产生越界中断"。虽然段 2 在内存中且权限允许，但请求的地址超出了该段的长度范围。

### 第10题（2019统考真题）

**题目：**  关于分段共享，错误的是（ ）。

**答案：B. 段 S 在 P₁ 和 P₂ 中应该具有相同的段号**

**解析：**

文档原文明确指出："P1 和 P2 共享的代码段在各自的段表中可以分配不同的段号，即段 S 的物理地址是唯一的，但其在 P1 和 P2 中的逻辑地址可以是不同的。"

|选项|判断|依据|
| -----------------------| :----: | --------------------------------------------|
|A. 物理内存仅一份 S|✅|共享就是多对一映射|
|**B. 必须相同段号**|❌|段号只在本进程内有效，不同进程可不同|
|C. 共享段表中同一表项|✅|共享段表是全局的，记录该共享段的 count 等|
|D. 都不再用才回收|✅|count 字段控制，最后一个进程释放才真正回收|

### 第11题（2019统考真题）

**题目：**  虚拟地址 2050 1225H 对应的页目录号、页号分别是（ ）。

**答案：A. 081H, 101H**

**解析：**

地址结构：`页目录号(10位) | 页号(10位) | 页内偏移(12位)`

```
2050 1225H → 二进制展开：

2=0010, 0=0000, 5=0101, 0=0000, 1=0001, 2=0010, 2=0010, 5=0101

完整 32 位：
0010 0000 0101 0000 0001 0010 0010 0101

按 10+10+12 划分：
高 10 位(31~22)：0010 0000 01 → 00 1000 0001 → 081H
中 10 位(21~12)：01 0000 0001 → 01 0000 0001 → 101H
低 12 位(11~0)： 0010 0010 0101 → 225H
```

|字段|二进制|十六进制|
| ----------| ----------------| :--------: |
|页目录号|00 1000 0001|**081H**|
|页号|01 0000 0001|**101H**|
|页内偏移|0010 0010 0101|225H|

### 第12题（2021统考真题）

**题目：**  二级页表系统中，CPU 页表基址寄存器中的内容是（ ）。

**答案：B. 当前进程的一级页表的起始物理地址**

**解析：**

文档原文："为了方便实现地址变换，需要增设一个页目录表寄存器，用于存放页目录表的起始地址。"

- 一级页表（页目录表）存储在物理内存中，地址是**物理地址**。
- 页表基址寄存器（PTR / 页目录表寄存器）指向一级页表在内存中的位置 → 存放的是**物理地址**。
- 进程切换时，会从新进程的 PCB 中加载该进程的一级页表物理地址到寄存器。

### 第13题（2023统考真题）

**题目：**  进程 R 和 S 共享数据 data，页号分别为 p1、p2，页框号分别为 f1、f2。正确的是（ ）。

**答案：C. p1 和 p2 不一定相等，f1 和 f2 一定相等**

**解析：**

这是前面第 10 题的"分页版"：

```
物理内存中数据只有一份 → 物理页框号必须相同 → f1 = f2

但在不同进程的逻辑地址空间中：
  - 进程 R 把该数据映射到自己的第 p1 页
  - 进程 S 把该数据映射到自己的第 p2 页
  - p1 和 p2 可以相同也可以不同，取决于各进程中页面编号
```

|组合|判断|
| :----------------------------: | :----------------: |
|p1=p2, f1=f2|❌ p1=p2 不一定|
|p1=p2, f1≠f2|❌ f1≠f2 不可能|
|**p1≠p2 不一定, f1=f2 一定**|✅|
|p1≠p2 不一定, f1≠f2 不一定|❌ f1≠f2 不可能|

**本质：**  共享数据在物理上同一份 → 框号唯一；逻辑页号是各进程私有的索引 → 可以不同。
