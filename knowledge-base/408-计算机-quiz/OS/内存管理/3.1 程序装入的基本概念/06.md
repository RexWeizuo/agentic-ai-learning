# 3.1 程序装入的基本概念 · 题6

6. 装入时动态链接的优点不包括（）。III  
   A. 方便单独修改其中一个或几个目标模块  
   B. 方便对其中一个或几个目标模块进行重用  
   C. 可以加快程序装入过程  
   D. 不需要重新打开整个装入模块

下面是这六道题的详细解析，结合我们学过的“程序装入基本概念”逐一说明。

## 解析

**题目**：装入时动态链接的优点不包括（）。  
A. 方便单独修改其中一个或几个目标模块  
B. 方便对其中一个或几个目标模块进行重用  
C. 可以加快程序装入过程  
D. 不需要重新打开整个装入模块

**答案**：C  
**解析**：

- 装入时动态链接的两个典型优点（文档明确列出）：

  1. **便于修改和更新**（对应A、D：修改某个模块不用重新打完整包）
  2. **便于实现对目标模块的共享**（对应B：多个应用可共享同一个模块）
- 而**加快程序装入过程**（C）并不是装入时动态链接的优点，反而因为需要边装入边解析外部调用，可能比静态链接的一次性装入更慢。另外，装入时动态链接还会装入可能根本不运行的模块（如错误处理模块），并不节省时间。
- 所以，**不包含**的选项是C。
