## 参考答案

**1. A**。编译（源代码→目标模块）→ 链接（多目标模块→一个装入模块）→ 装入（装入模块→内存）→ 执行。

**2. C**。静态重定位实现简单、不需要硬件支持，动态重定位才需要硬件支持。

**3.** (1) 运行时动态链接——库在使用时才加载，多个进程共享同一份物理副本。(2) 装入时动态链接——修改某个模块后只重装该模块。(3) 静态链接——简单可靠，运行时无额外开销。

**4.** 基址寄存器和限长寄存器。基址寄存器存起始物理地址，限长寄存器存储逻辑地址范围。CPU 每次访存时硬件检查"逻辑地址 ≤ 限长"，越界则触发**地址越界中断（陷阱）**，操作系统终止该进程。

**5. B**。动态分区按需分配，不产生内部碎片；但回收后产生不连续空洞 → 外部碎片。A 错（最后一页可能填不满），C 错（固定分区必有内部碎片），D 错（紧凑消外部碎片，不能消内部碎片）。

**6.** (1) 首次适应：20KB（第一个≥45的）；最佳适应：50KB（最接近45的）；最坏适应：120KB（最大的）。(2) 最佳适应留下剩余碎片最小（5KB）。这是**坏事**——5KB 太小很难再利用，长期累积成大量外部碎片。

**7.** 紧凑：将内存中所有已分配分区**移动**到一起，使分散的空闲区合并为一个大空闲区。解决问题：外部碎片。前提：需要**动态重定位**支持（程序在内存中移动后，通过重定位寄存器修正地址）。

**8.** 伙伴系统分配 2 的幂次大小。70KB → 最接近的 2^7 = **128KB**。内部碎片 = 128 - 70 = **58KB**。

**9.** (1) 分页对用户**不可见**（系统行为）；分段对用户**可见**（程序员按逻辑单元组织）。(2) 分页地址空间**一维**（所有地址连续）；分段**二维**（段名+段内偏移）。(3) 分页产生**内部碎片**（最后一页）；分段产生**外部碎片**（段长不一）。

**10.** (1) 段数 = 2^8 = 256；每段页数 = 2^10 = 1024；页面大小 = 2^14 = **16KB**。(2) **3 次**：查段表→查页表→访问数据。(3) 比纯分段：不会产生外部碎片；比纯分页：保留了段的逻辑意义，便于共享和保护。

**11.** 普通页表：每个进程一张，以虚拟页号为索引，条目数 = 虚拟页数。反置页表：**整个系统一张**，以物理页框号为索引，条目数 = 物理页框数。优势：**节省内存**——当虚拟地址空间远大于物理内存时（如 64 位系统），反置页表大小仅取决于物理内存大小。

**12.** (1) 对 `sum` 的重复访问 → **时间局部性**。(2) 对 `a[i]` 的顺序访问 → **空间局部性**。(3) 递归反复调用同一函数 → **时间局部性**。

**13.** (1) 页表大小：页面越小 → 页数越多 → 页表越大。(2) 内部碎片：页面越小 → 碎片越小。(3) I/O 效率：页面越大 → 每次 I/O 传输效率越高。(4) 缺页率：页面太小会导致频繁缺页；太大则装入很多不需要的数据（浪费），也会降低局部性利用。现代系统通常 **4KB** 的折中。

**14.** 写时复制：`fork()` 后父子进程共享同一物理页，**标记为只读**。当任一进程尝试写该页时，触发保护异常，OS 才为该进程**复制一份独立的物理页**。这避免了完全拷贝进程地址空间的开销，尤其当 `fork()` 后紧跟 `exec()` 时几乎不需要拷贝。

**15.** LFU 淘汰**访问次数最少**的页（考虑整个历史），LRU 淘汰**最久未被访问**的页（只看最近一次访问时间）。区别：LFU 可能淘汰一个很久以前频繁访问但最近没用的页；LRU 只看"最近"。

模拟（3框，初始空）：
LRU：1×[1], 2×[1,2], 1✓, 3×[1,2,3], 2✓, 1✓, 2✓, 3✓, 1✓, 4×淘汰2→[1,3,4], 4✓, 1✓ → **4 次缺页**（不含初始3次装入）。
LFU：初始装入同上，访问串中 1 出现5次最多、2 出现3次、3 出现2次、4 出现2次。缺页发生在 1,2,3（初始）以及 4 时淘汰计数最少的 2 或 3 → 也约 4-5 次缺页。两者结果接近，但 LFU 在多遍循环访问中可能比 LRU 表现更差（因为保留了过去高频但已不再使用的页）。

**16. B**。抖动 = 多道程序度过高 → 每个进程分配的页框数**小于其工作集大小** → 频繁缺页 → 系统大部分时间在换入换出 → CPU 利用率急剧下降。

**17.** 工作集模型：监控每个进程最近 $\Delta$ 次访问中使用的页面集合。OS 确保每个进程分配的页框数 ≥ 工作集大小。$\Delta$ 太小：工作集不完整，进程仍会频繁缺页。$\Delta$ 太大：工作集过大，OS 会拒绝加载新进程，降低多道程序度。

**18.** t=5（最近5次访问：3,1,4,2,5）：工作集 = {3,1,4,2,5}，5 个页面。t=10（最近5次：5,3,2,1,5,4）：注意访问序列是 3,1,4,2,5,3,2,1,5,4。从第6次开始往回数5个：第6次=3, 第7次=2, 第8次=1, 第9次=5, 第10次=4。工作集 = {3,2,1,5,4}，5 个页面。

**19.** 页缓冲队列：存放**被淘汰页**和**预读页**的缓冲区。被淘汰页先写入缓冲队列（而不是直接写磁盘），进程可快速获取新页框；系统空闲时再批量写回。好处：(1) 减少等待磁盘写入的时间；(2) 被淘汰页在写回前若被再次访问，可直接从缓冲恢复，避免重新从磁盘读入。

**20.** (1) 虚拟地址空间 = 2^32 = **4GB**。物理地址空间同样 = 4GB。(2) 页目录 = 2^10 = 1024 项；每个页表 = 2^10 = 1024 项；每项 4 字节。(3) 页目录大小 = 1024×4B = 4KB。完整二级页表 = 4KB + 1024×4KB = 4KB + 4MB = **4MB+4KB**。(4) 多级页表的核心优势：**不需要为未使用的虚拟地址空间分配页表**。一级页表常驻内存（仅 4KB），二级页表按需分配，极大节省内存。

**21.**
| 策略 | 页框数 | 置换范围 | 实现难度 | 灵活性 |
|---|---|---|---|---|
| 固定+局部 | 固定 | 仅本进程 | 最简单 | 低 |
| 可变+全局 | 可变 | 全部进程 | 中等 | 最高 |
| 可变+局部 | 可变 | 仅本进程 | 最复杂 | 中等 |

**22.** OPT（4框）：初始装入 0,1,2,3（4次缺页），访问 0✓,1✓，4×淘汰 3（未来最远），0✓,1✓，2×淘汰 4，3×淘汰？→ 共 **7 次缺页**。FIFO（4框）：0×,1×,2×,3×,0✓,1✓,4×淘汰0,0×淘汰1,1×淘汰2,2×淘汰3,3×淘汰4,4×淘汰0 → 共 **10 次缺页**。LRU（4框）：同 OPT 策略但基于历史 → **8 次缺页**。增到 5 框后 FIFO 需要重新模拟。FIFO 可能出现 Belady 异常（页框数增加但缺页反而增加），需具体数据验证。

**23.**
- 第一轮（找 (0,0)）：扫描 (1,0)→跳过，(0,1)→跳过，(1,1)→跳过，(0,0)→**找到！淘汰该页**。
- 如果第一轮未找到，第二轮（找 (0,1)，同时将经过页的 A 清零），第三轮重复。

**24.** OPT 需要预知未来页面访问序列，这在实际系统中**不可能**做到。研究意义：(1) 作为理论**下界**，衡量其他算法的优劣；(2) 指导其他算法设计（如 LRU 利用"最近过去 ≈ 不远的将来"的经验规律模仿 OPT）。

**25.** 完整过程：
1. CPU 发出虚拟地址 → **TLB** 查找（用虚拟页号查）
2. TLB 命中 → 直接获物理页框号 → 拼接偏移 → 访存 ✓
3. TLB 未命中 → 查页表（可能在内存中多级页表遍历）
4. 页表项有效 → 物理页框号加载到 TLB → 访存 ✓
5. 页表项**无效**（存在位=0）→ **缺页中断**
6. OS 缺页处理：(a) 检查有无空闲页框；(b) 若无，选 victim 淘汰；(c) 从磁盘交换区读入缺失页；(d) 更新页表；(e) 重新执行被中断指令
