## 核心错误

### 1. 导入后从不验证文件数
任何数据迁移（siyuan→knowledge-base、GitHub→knowledge-base、zip→quiz）完成后，必须做 `源文件数 vs 目标文件数` 对比。今天三次出错都因跳过此步骤。

**How to apply**: 每次导入后立即 `ls | wc -l` 或 `rglob('*.md')` 对比源和目标。差一个文件就要查原因。

### 2. 正则假设格式统一
`\d+\.\s`（点后有空格）漏掉了 `\d+\.`（点后无空格），导致 3.5 漏了 5 道题。不同 FlowUs 页面导出格式不同，不能假设一致。

**How to apply**: 写正则前先抽样 10+ 个文件看实际格式。用 `\d+\.\s*` 而非 `\d+\.\s`。

### 3. pathlib `name` 在 `／` 文件名上的行为
SiYuan GitHub 导出用全角 `／` (U+FF0F) 做路径分隔。`Path.name` 返回的可能是完整路径而非最后一段。`startswith` 失效。应用 `in` 替代。

**How to apply**: 处理 siyuan-data 文件时，先 `print(repr(f.name))` 确认实际值。

### 4. 删除操作无备份
删空文件时误杀了 quiz 的短注释文件（"极快"、"也可以举反例验证"）。阈值 15 字符太激进。

**How to apply**: 删除前先存列表到文件，用户确认后再执行。永远不要基于启发式规则直接删除。

### 5. 自定义解析器不如用现成导出
花了数小时写 `.sy` JSON 解析器处理 40 种节点类型，最终发现 WSL 上的 Claude session 已经完整导出到 GitHub。应该**先搜索是否有现成导出**，再决定是否自己造轮子。

**How to apply**: 接到数据迁移任务时，先问：有无官方导出功能？有无 API？有无他人做好的导出？三者都没有才自己解析。

## 冗余操作

| 操作 | 耗时 | 最终用途 |
|---|---|---|
| CDP 连接 FlowUs 提取 IndexedDB | ~1h | 未使用 |
| FlowUs REST API 递归提取 | ~2h | 仅用于理解结构，未产出最终数据 |
| 自定义 .sy JSON 解析器 | ~3h | 被 GitHub 导出替代 |
| 多次重导 quiz（误删→恢复→再误删→再恢复） | ~1h | 最终手动导出才是正确数据源 |
| 内存管理拆分脚本修了 4 版 | ~1h | 正则 `\d+\.\s*` 一版就该对 |
| 空文件删除→恢复→再删除 | ~0.5h | 应该先列表确认 |

**总冗余时间**: ~8.5h
