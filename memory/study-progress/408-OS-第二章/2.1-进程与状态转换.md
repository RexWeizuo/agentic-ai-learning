# 2.1 进程与状态转换 · Socratic 问答

## Q1 · 程序 vs 进程

> 程序和一个进程的区别是什么？进程比程序多了哪些东西？PCB 的全称是什么，为什么需要它？

**学生回答**：进程除了包含程序本身还多了它的运行状态、运行环境（上下文）。进程控制块内部存放运行状态和环境信息，相当于档案。

**AI 批改**：✅ 全对。程序 = 静态代码+数据（菜谱），进程 = 动态运行（做饭过程）。PCB = Process Control Block，存 pid、CPU 现场、状态、资源清单。

---

## Q2 · 进程五状态模型

> 画出五状态图。哪四种转换合法？哪种不可能？

**学生回答**：运行态、就绪态、阻塞态、终止态、创建态。阻塞态不可能转换成执行态。追问：就绪态会变成阻塞态吗？

**AI 批改**：✅ 五状态全对，阻塞→运行不可能 ✅。回答追问：就绪态→阻塞态也不行。阻塞是进程的主动行为（调用 block 原语），只有正在运行的进程才能做。就绪态连 CPU 都没有，无法执行任何指令——包括阻塞自己。

合法转换：就绪→运行、运行→就绪、运行→阻塞、阻塞→就绪、运行→终止。
不可能：阻塞→运行 ❌、就绪→阻塞 ❌。

## Q3 · 状态转换触发条件

> 三种情况触发运行→阻塞，两种情况触发运行→就绪？

**学生中途回答**：中断、异常、还有一个什么；时间片到、还有不知道。

**AI 批改**：⚠️ "中断""异常"是机制不是具体条件。

**运行→阻塞**（三种）：
1. I/O 请求——scanf()等系统调用等用户输入/等磁盘读写
2. 等待临界资源——wait()操作，资源被占
3. 等待某事件发生——如等待子进程结束

**运行→就绪**（两种）：
1. 时间片用完——CPU 调度程序强制切换
2. 被更高优先级进程抢占

## Q4 · PCB 四类信息

> PCB 包含哪四类信息？

**学生回答**：PID、运行环境、状态字寄存器

**AI 批改**：⚠️ 说对了两类，差两类。

| 类别 | 内容 |
|------|------|
| ① 进程标识符 | PID（内部）+ 外部标识符 |
| ② 处理机状态 | 通用寄存器、PC、PSW——进程切换时保存/恢复的核心 |
| ③ 进程调度信息 | 状态、优先级、等待时间、阻塞原因 |
| ④ 进程控制信息 | 代码段/数据段地址、资源清单、链接指针 |

## Q5 · 原语 block/wakeup

> 为什么 block 和 wakeup 必须配对？进程唤醒后直接运行吗？

**学生回答**：不配合阻塞的话就永远不会被唤醒了。意味着进入就绪态。

**AI 批改**：✅ 全对。唤醒 = 入就绪队列排队重新竞争 CPU，不是直接运行。

---

## Q6 · 进程切换

> 一定发生在什么态？P0→P1 完整流程？

**学生回答**：用户态？系统调用的时候？P0 不用 CPU，OS 保存 P0 状态到 PCB0，然后找到就绪队列队首 P1 运行。

**AI 批改**：⚠️ 核心错误——进程切换一定发生在**内核态**，不是用户态。

完整流程：
```
P0 运行（用户态）
→ 中断/系统调用 → 陷入内核态
→ 内核保存 P0 的 CPU 现场到 PCB0
→ 修改 P0 状态为就绪/阻塞，放相应队列
→ 调度程序选中 P1
→ 从 PCB1 恢复 P1 的 CPU 现场
→ 返回用户态，P1 继续运行
```

## Q7 · 父子进程

> 【讲义练习】叙述错误的是？
> A.可并发 B.共享虚拟地址空间 C.不同PCB D.不同时用同一临界资源

**学生回答**：不知道。追问父子进程和线程有什么区别，感觉一样。

**AI 批改**：选 **B**。父子进程不共享虚拟地址空间——fork() 后子进程复制父进程地址空间（写时拷贝），两者独立。

| | 父子进程（fork） | 线程 |
|---|---|---|
| 地址空间 | 独立（子复制一份） | 共享同一个 |
| PCB | 各有一个 | 共享一个 PCB |
| 创建代价 | 大 | 小 |
| 通信 | 管道/IPC | 直接读写共享变量 |
| 崩溃影响 | 父崩子不死（变孤儿被 init 收养） | 一个线程崩，整个进程死 |

## Q8 · 虚拟地址空间 + 内核映射

> Linux 32 位下 4GB 地址空间怎么分？为什么每个进程都要包含内核？

**学生回答**：1GB 内核 3GB 用户 ✅。追问：因为进程切换必须要处于内核态所以要把进程切换那部分保留是吗？

**学生反思**：所以其实不是复制一份到每一个进程，只是告诉进程内核部分的地址是吗？那也不是每个进程空间都有内核啊。

**AI 批改**：✅ 追问 → 核心理解到位。

**正确理解**：不是复制，是**映射**。每个进程虚拟地址空间的最高 1GB 表面上"有自己的内核"，但页表把这片区域指向**同一块物理内存**——内核真只有一份。类比：每人的通讯录里都有"110"这个号码，打过去是同一个派出所。

## Q9 · 三态资源表

> 三态模型下各状态拥有什么资源？

**学生回答**：只有运行态有处理器资源和共享资源吧

**AI 批改**：❌

| 状态 | CPU | 其他所需资源 |
|------|-----|------------|
| 运行态 | 拥有 | 拥有 |
| 就绪态 | 等待 | **拥有**（万事俱备只欠 CPU） |
| 阻塞态 | 等待 | 等待（等 I/O、等资源——什么都没有） |

就绪态已经分配了内存、文件、I/O 设备，就差 CPU 没分到。
