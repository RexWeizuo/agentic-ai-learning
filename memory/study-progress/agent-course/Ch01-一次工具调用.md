## 核心概念掌握

| 概念 | 得分 | 备注 |
|------|------|------|
| 工具调用四步循环 | ★★★★★ | 模型→发请求→代码执行→结果返回，理解准确 |
| Schema 是合同 | ★★★★ | 名称+描述+输入schema。描述要写"何时用+何时别用"的安全边界 |
| 错误不抛异常 | ★★★★★ | throw 杀进程模型看不到→return error 模型可修正 |
| 模型视野 | ★★★★★ | 只看到 tool_result，看不到源码和中间变量 |
| input vs arguments | ★★★★★ | 都是工具参数，非用户输入；不同 provider 不同名 |
| Adapter 模式 | ★★★★★ | 薄封装统一 provider 差异，换模型只改一行 |
| OpenCode Tool.define | ★★★★ | 类型化生命周期追踪每次调用 |
| Hermes ToolEntry | ★★★★★ | schema+handler+大小限制+错误分类(可恢复/致命) |
| 一切皆工具 | ★★★★★ | channel adapter、工作流步骤、shell命令、子agent都是工具 |

## 代码实战

**文件**：`workspace/quant-agent/src/quant_agent/tools/prices.py`

**实现**：
- `GET_PRICE_SCHEMA`：工具定义（名称+描述含边界+参数schema）
- `_detect_market()`：自动识别市场（6位数字→A股，含/→加密货币，字母→美股）
- `_fetch_a_stock()`：新浪 API（`hq.sinajs.cn`），GBK 编码
- `_fetch_crypto()`：ccxt 币安（国内需 VPN）
- `_fetch_us_stock()`：yfinance（可能限速）
- `get_price()`：统一 dispatch，错误返回 dict 不抛异常

**踩坑**：东方财富 API TCP 拒连接（频繁请求被封 IP）→ 切新浪 API
