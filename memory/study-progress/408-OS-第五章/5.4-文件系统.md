# 5.4 文件系统 · Socratic 问答

## Q1 · 四种空闲空间管理方法

> 四种方法各是什么？数据结构是什么？

**学生回答**：空闲区表法（起始地址+长度）、空闲链表法（指针指向下一个）、位示图法（二维数组）、成组链接法（没看懂）

**AI 批改**：前三个 ✅，第四留到 Q4。

| 方法 | 数据结构 | 要点 |
|------|---------|------|
| 空闲区表法 | 表（起始块号+空闲块数） | 类似内存动态分区 |
| 空闲链表法 | 链（每块/每区→下一块/区） | 分盘块链和盘区链 |
| 位示图法 | m×n 位数组（0=空闲,1=占用） | 常驻内存，快速 |
| 成组链接法 | 空闲盘块号栈（分组+索引+栈） | UNIX 采用 |

## Q2 · 位示图——盘块号 ↔ 行列转换 ⭐

> 字长 32 位，8 个字，行/列从 1 开始。
> (1) 归还块号 100 → 第几行第几列？
> (2) 第 3 行第 5 列 → 盘块号？

**学生回答**：(1) 第 4 行第 4 列 ✅ (2) 3×32+5=101 ❌

**AI 批改**：⚠️ (1)对，(2)错了。

(1) 100 = 32×3 + 4 → 行号=3+1=4, 列号=4 ✅ 公式：i=(b-1) DIV 32 +1, j=(b-1) MOD 32 +1

(2) **正确**：第 3 行 = 块号 65~96。第 5 列 → b = 32×(3-1) + 5 = **69**。
错误原因：你用了 3×32，把第 1 行等价于第 0 行。

> 公式：b = n×(i-1) + j，不是 n×i + j。

## Q3 · 位示图空间计算

> 位示图 512 字节，盘块 4KB → 最大容量？

**学生回答**：4KB × 512 × 512 = 2^28B ❌

**AI 批改**：❌ 漏了字节→位转换。

- 512 字节 = 512×8 = **4096 位**（1 位管 1 个盘块，不是存地址）
- 4096 × 4KB = 4096 × 4096B = 2^24 = **16MB**

你的 512×512 把字节直接当位用了，差了 8 倍。

**学生追问**："4096 位不是可以表示 4096×4096 个位置吗？"
**答**：位示图中 1 bit 是 0 或 1（空闲/占用），不能用 1 bit 存块号——那是索引表的概念。

## Q4 · 成组链接法 ⭐

> 解决了什么问题？栈底盘块号（S.free[0]）分配时需什么特殊操作？

**学生回答**：解决表太长。把它指向的盘块指针指向的栈作为新的可分配空闲栈，然后分配出去。

**AI 批改**：✅ 方向完全对。

**解决什么**：空闲链表太长 → 分配效率低。通过**分组 + 索引块 + 栈**把长链切成短组。

**栈底特殊操作**：
```
① 读 S.free[0] 指向的盘块内容 → 作为新的空闲盘块号栈
② N 刷新为 100（新栈的空闲块数）
③ 原 S.free[0] 这块 → 分配出去给用户
```

**核心理解**：索引块 = 从空闲块征用来的盘块，S.free[0]~S.free[N-1] 全存空闲块号。栈顶正常分配，栈底指向下一组索引块。

## Q5 · 四种方法对比

**学生回答**：

| | 空闲区表法 | 空闲盘块链 | 空闲盘区链 | 位示图法 |
|---|---|---|---|---|
| 分配速度 | 慢 | 快 | 较快 | 快 |
| 合并 | 是 | 否 | ❌ 否（错） | 否 |
| 长度 | 无链表 | 长 | 较短 | 无链表 |
| 连续分配 | 支持 | ❌ | ❌ | ✅ |

**AI 批改**：⚠️ 两处纠正。

**空闲盘区链回收**：盘区链和内存动态分区一样——回收时需检查与前后空闲区是否相邻，相邻就合并。不是"否"。

| | 空闲区表法 | 空闲盘块链 | 空闲盘区链 | 位示图法 |
|---|---|---|---|---|
| 分配速度 | 慢（顺序搜索） | 快（摘一块） | 较快（摘一个区） | 快（常驻内存） |
| 回收需合并 | 是 | 否 | **是**（相邻区合并） | 否 |
| 链表长度 | 无 | 长 | 较短 | 无 |
| 支持连续分配 | 是 | 否 | 是 | 是 |

## Q6 · FAT 文件系统结构

> FAT 磁盘卷分为哪 5 个区域？FAT1 和 FAT2 为什么两份？

**学生回答**（初答）：引导块、控制块、I结点、普通目录和文件 ❌——混成了通用布局

**AI 批改**：FAT 卷 5 区域：

```
引导区 → FAT1 → FAT2 → 根目录区(FDT) → 数据存储区(簇)
```

| 通用布局（UNIX） | FAT 卷 |
|---|---|
| 引导块 | 引导区 |
| 分区控制块 | ❌ FAT 没有单独控制块 |
| 空闲空间管理 | FAT1+FAT2 本身就管空闲空间 |
| I 结点 | ❌ FAT 用 FDT 取代 inode（老式设计，FCB 嵌目录） |
| 根目录 | 根目录区(FDT) |

**FAT2 = FAT1 的完整副本**。FAT 表损坏 → 文件全丢，所以存两份用于恢复。

## Q7 · VFS 虚拟文件系统

> 为什么 Linux 需要 VFS？四种对象？

**学生回答**：VFS 提供虚拟接口统一不同文件系统。对象：inode、文件对象、另外不知道。

**AI 批改**：⚠️ 概念对，缺两个。

| 对象 | 代表什么 | 类比 |
|------|---------|------|
| 超级块 (superblock) | 整个文件系统（类型、大小） | 图书馆总览 |
| 索引节点 (inode) | 一个文件（权限、物理位置） | 书的档案卡 |
| 文件对象 (file) | 一个已打开的文件（读写位置） | 正在看的那本书的翻页位置 |
| 目录条目 (dentry) | 文件名→inode 的映射 | 索书号标签 |

VFS 的作用：向上提供统一接口（open/read/write），向下调用具体文件系统（ext4/FAT/NFS 各自实现）。

## Q8 · 文件系统布局 + 挂载

> 磁盘分区从引导块开始包含哪些部分？注册和挂载做了什么？

**学生回答**：引导块、控制块、inode、根目录、普通文件和目录；注册=初始化，挂载=虚拟路径映射到真实路径。

**AI 批改**：⚠️ 漏了空闲空间管理。

```
引导块 → 分区控制块 → 空闲空间管理 → I结点 → 根目录 → 普通文件/目录
```

**注册** ✅：文件系统向内核登记（填 `file_system_type` 结构，声明"我是 ext4"）
**挂载** ✅：附着到目录树节点（`mount -t ext4 /dev/sda1 /home` → `/home` 成为挂载点）

## Q9 · 外核（Exokernel）+ 逐选项详解

> 【讲义练习】关于外核的说法，**错误**的是（ ）

**学生回答**：只知道外核就是 Ubuntu 桌面，内核是 Linux ❌

**AI 批改**：

**概念纠正**：Ubuntu 桌面 ≠ 外核。Ubuntu 内核 = Linux（宏内核 monolithic kernel）。外核是 MIT 1995 年的研究项目，核心哲学：传统 OS 做太多抽象 → 应用（数据库、Web 服务器）比 OS 更懂硬件 → 外核只做**保护 + 资源复用**，其余丢给用户态"库 OS"。

| 选项 | 对错 | 原因 |
|------|------|------|
| A. 外核可以分配未经抽象的硬件资源 | ✅ | 核心特点——允许应用申请特定物理内存、磁盘块 |
| B. 用户进程通过"库"请求外核服务 | ✅ | 外核只给底层，上层功能由 LibOS 提供 |
| C. 外核负责完成进程调度 | ❌ | 外核只管保护和复用，调度不属于它的职责 |
| D. 外核可减少"映射"开销 | ✅ | 绕开虚拟→物理映射，直接访问硬件 |

**答案：C**

## Q10-真题 · 文件系统说法 + 逐选项详解

> 【讲义练习】关于文件系统的说法，**正确**的是（ ）

| 选项 | 对错 | 原因 |
|------|------|------|
| A. 文件数量受限于 FCB 数量 | ✅ | 每个文件必须有 FCB/inode，FCB 数组大小定了，文件数就定了 |
| B. 文件系统容量 = 磁盘容量 | ❌ | 可跨多盘（RAID/LVM），也可小于磁盘（只分一个区） |
| C. 单文件大小只受剩余空间限制 | ❌ | 还受文件系统设计限制——FAT32 单文件最大 4GB，即使磁盘还有 1TB |
| D. 不能跨磁盘存放数据 | ❌ | 逻辑卷、RAID 均可跨盘 |

**答案：A**
