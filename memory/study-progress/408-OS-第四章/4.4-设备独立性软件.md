### Q1 · 单缓冲公式

> 单缓冲区下，处理每块数据的时间通用公式是 Max(C, T) + M。C、T、M 分别是什么？
>
> T=80μs, M=40μs, C=30μs 时，处理一块数据的总时间是多少？

**学生回答**：C是CPU处理一块数据的时间，T是I/O设备把一块数据传送到缓冲区的时间，M是这块数据从缓冲区传送到工作区进程的时间。因为只有块缓冲区读和写不能同时操作 所以T和M必须串行C可以和下一个T并行但是必须等T和M结束才能拿到数据 max(C, T)+M = 80+40=120微秒

**AI 批改** ✅ 全对

C/T/M 定义准确。120μs 计算正确。你说的"T和M必须串行"是关键——单缓冲只有一个缓冲区，设备往里写的时候不能同时往外拷，必须等 T 完成才能开始 M。Max(C,T) 的含义是：C 和下一个块的 T 可以重叠，但当前块必须等 T 和 M 都走完。

### Q2 · 双缓冲公式 + 为什么快

> 双缓冲公式是 Max(C+M, T)。和单缓冲的 Max(C, T)+M 相比，为什么双缓冲能省时间？省在哪里？ 
>
> T=80μs, M=40μs, C=30μs 时，双缓冲处理一块的时间是多少？比单缓冲快了多少？

**学生回答**：两块缓冲区第一块写完写第二块 第二块在写的时候第一块就可以读了 流水线的情况下M和T也并行了 时间省在哪串行传输变并行传输吗？ M+C < T 的情况处理时间是80微秒

**AI 批改** ✅ 理解对了，补一个数字

"流水线让 M 和下一个 T 并行"抓住了核心。画成时间线：

```
单缓冲：| T(80) | M(40)+C(30) | T(80) | M+C | ...     = 120/块
双缓冲：| T(80)               | T(80)               | ...  = 80/块
         | M(40)+C(30) |       | M+C |               
```

双缓冲比单缓冲快 120-80 = **40μs**。省的就是 M 那段——单缓冲的 M 必须等 T 完成，双缓冲的 M 和下一个 T 同时跑。

你说 M+C<T 时处理时间 80μs 是对的（Max(70,80)=80）。

**Q2.5 追问**：缓冲池不就是缓冲区链表+操作函数吗 不还是很多缓冲区很占内存空间吗

答：占，但值。每个进程自己开缓冲区 vs 共享一个池——100 个进程各 4KB = 400KB vs 一个池 40KB 就够了。池的共享让总量大幅减少，而且 OS 能动态调大小。

### Q3 · 设备分配四表

> 设备分配涉及四种数据结构：SDT、DCT、COCT、CHCT。各自的全称是什么？哪个是"整个系统唯一一张"的？哪个是"每设备一张"的？

**学生回答**：SDT系统设备表 整个系统一张 每个物理设备对应表中一条；DCT设备控制表 每个设备一张表；COCT控制器控制表 每个控制器一张表；CHCT通道控制表 每个通道一张表。

**AI 批改** ✅ 全对

SDT=系统设备表（唯一）、DCT=设备控制表（每设备）、COCT=控制器控制表（每控制器）、CHCT=通道控制表（每通道）。四个全称和数量关系都准确。

### Q4 · 设备分配三步顺序

> 设备分配的三步顺序是什么？（2013 南京理工 877 真题考点）
> 如果用了逻辑设备名，分配流程有什么不同？

**学生回答**：分配设备 分配控制器 分配通道 设备无关性是什么直接用物理设备名访问导致没法排队吗？逻辑设备名就可以查询所有同类型的设备 都忙的情况下才会去排队

**AI 批改** ✅ 三步顺序全对

先设备 → 再控制器 → 最后通道。三者都分配成功才算完成。

你的追问"设备无关性是什么"——就是你理解的那个意思。物理设备名 = 指定唯一设备，那个设备忙就挂了。逻辑设备名 = 说"我要一台打印机"，OS 找同类型里空闲的给你，全忙才排队。"设备无关性"和"设备独立性"是一个概念。

### Q5 · 逻辑设备名 vs 物理设备名

> 为什么说用物理设备名"不灵活"？逻辑设备名解决了什么问题？逻辑设备表（LUT）放在哪里？

**学生回答**：因为物理设备名对应唯一的物理设备可能正在被使用但进程只需要同类型设备；放在内核或者PCB中

**AI 批改** ✅ 全对

物理设备名不灵活的原因你抓住了——绑定唯一设备，忙了就等。逻辑设备名让 OS 帮你选。

LUT 位置也对了：单用户系统放内核全局，多用户系统放每个进程的 PCB 里。补充一个考点——PCB 里的 LUT 在进程切换时需要保存/恢复。

### Q6 · 安全分配 vs 不安全分配

> 安全分配方式下，进程发出 I/O 请求后会怎样？不安全分配呢？两者的死锁风险和 CPU/I/O 并行性有什么区别？

**学生回答**：阻塞，获得时不在请求 没获得就阻塞也不保持资源不会死锁 不知道并行性区别；不阻塞 可以再发I/O请求 可能死锁

**AI 批改** ⚠️ 对了一半，补并行性

| | 安全分配 | 不安全分配 |
|---|---|---|
| I/O 请求后进程状态 | **阻塞**，等 I/O 完成 | **不阻塞**，继续运行 |
| 死锁风险 | 无（请求时不保持资源） | 有（可能边保持资源边请求新资源） |
| CPU 与 I/O | **串行**（进程阻塞，CPU 空闲等 I/O） | **可并行**（进程不阻塞，CPU 继续干活） |

你没答出来的并行性区别就是这道题的第二个考点——安全和效率的 trade-off。安全=没死锁但慢，不安全=快但可能死锁。

### Q7 · 阻塞 vs 非阻塞 I/O

> 阻塞 I/O 和非阻塞 I/O 的区别是什么？各自适合什么场景？

**学生回答**：键盘是阻塞I/O吗 我只知道运行程序的时候会卡在scanf那一步阻塞等待输入 非阻塞就是进程启动I/O以后继续执行原有程序 不知道适合什么场景

**AI 批改** ⚠️ 定义对，补场景

scanf 的例子很好——那就是阻塞 I/O。非阻塞定义也对：启动 I/O 后立即返回，不等结果。

场景：
- **阻塞**：简单、小并发。如命令行程序、单用户场景。容易写，不会乱。
- **非阻塞**：高并发服务器。如 Web 服务器同时处理 1000 个请求，不能每个都卡住等磁盘。需要配合多线程或异步机制，写起来复杂。

### Q8 · 四种缓冲机制对比

> 单缓冲、双缓冲、循环缓冲、缓冲池——一句话概括每种的核心特征。缓冲池的三个队列叫什么？四种工作方式分别是什么？

**学生回答**：单缓冲：用户进程I/O请求的时候OS为它分配一个缓冲区I/O的数据先输入到缓冲区 这样生产者生产的先放到缓冲区再继续生产就行不用等到消费者消费了再生产；双缓冲：两个缓冲区 一个缓冲区满了消费者还没消费生产者又生产了一块数据可以放到第二个缓冲区生产者再继续生产 流水线的情况下读写可以并行；循环缓冲：缓冲区变成环形；缓冲池：放很多不同进程可以共享的缓冲区，还包含操作函数管理缓冲区；三个队列：空白缓冲队列 输入队列 输出队列；收容输入（输入一个空缓冲区） 提取输入（一个有数据的缓冲区数据提取到用户程序）收容输出（程序提取完的空缓冲区重新放回缓冲池）提取输出（用户写满的缓冲区提取到I/O中）

**AI 批改** ✅ 几乎全对，两处调一下

四种缓冲概括、三个队列名称——全对。

四种工作方式里有两处定义反了：

| 方式 | 正确含义 |
|---|---|
| 收容输入 | 从空队列取缓冲区 → 装入**输入**数据 → 挂到输入队列 |
| 提取输入 | 从输入队列取缓冲区 → 提取数据给进程 → 挂回空队列 |
| 收容输出 | 从空队列取缓冲区 → 装入**输出**数据 → 挂到输出队列 |
| 提取输出 | 从输出队列取缓冲区 → 数据送**设备** → 挂回空队列 |

你说的"收容输出=空缓冲区放回缓冲池"应该是提取输入或提取输出做的事情。收容=往里装数据，提取=往外拿数据。输入/输出区分的是数据方向（从设备来还是往设备去）。
