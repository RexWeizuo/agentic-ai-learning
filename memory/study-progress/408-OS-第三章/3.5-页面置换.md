## 概念问答记录

**Q1**: 哪些是栈算法？和 Belady 的关系？
**A1**: 知道 FIFO 不是（会 Belady）。纠正：栈算法定义 = 页框 N 时的页面集合是 N+1 时集合的子集。LRU/OPT 是，FIFO 不是。

**Q2**: 改进型 CLOCK 淘汰顺序？
**A2**: (0,0)→(0,1)→(1,0)→(1,1)。纠正："优先考虑是否被访问过"不够精确——改进型是四轮扫描，第一轮找(0,0)，第二轮找(0,1)并清 A，第三轮重复。淘汰(0,1)先于(1,0)因为即使要写回磁盘，也比淘汰即将被访问的页划算。

**Q3**: 什么是抖动？
**A3**: 物理页不够导致频繁换入换出。纠正：CPU 低+磁盘高=经典抖动症状。验证方法：减少进程数 → CPU 回升则确认。

**Q4**: 工作集？t=5 和 t=10？
**A4**: 工作集概念正确，两个时刻都是 {1,2,3,4,5}（5页）。说明该进程可能需要 5 页内存。

**Q5**: LRU vs LFU 区别？LFU 何时更差？
**A5**: LRU=最久未用，LFU=使用次数最少。纠正：LFU 更差的场景——早期高频访问后永不再用的页面，LFU 舍不得淘汰，LRU 正确淘汰（如初始化函数）。

## Quiz 错题

**Q1**（AI 也错了）: LRU 步 5 淘汰 3 而非 2。纠正：7 次缺页(C)。

**Q3**: 缺页次数下限 = n（不同页号数，每个首次必缺）。已理解。

**Q5**: OPT 模拟需逐次看"未来最远"。已理解。

**Q11**: 02H = 十进制 2 即页号 2。改进 CLOCK 淘汰页 3 → 页 2 装入 60H → 60A01H。

## 补充问答：工作集详解

**追问**: 工作集到底是什么？举例子展开。

**回答**:
- 工作集 = 最近 Δ 次访问中用到的不同页面集合（不是"单位时间内需要的所有页面"）
- 举例 Δ=5，序列 3,1,4,2,5,3,2,1,5,4：
  - t=5 工作集 {1,2,3,4,5} (5页)；t=7 时页面 1 被挤出 → {2,3,4,5} (4页)
- 与抖动的联系：物理页框 < 工作集 → 频繁缺页 → 抖动
- Δ 太大 → OS 高估需求，拒绝新进程；Δ 太小 → 给太少页框 → 缺页
- OS 策略：确保每个进程分配的页框数 ≥ 工作集大小
